# 5.5.playtest.v1.i1: keyrack fill byhand verification

## prerequisites

1. run from rhachet repo root (this repo)
2. terminal with tty (interactive prompts)
3. clear daemon state: `rhx keyrack relock`

## happy paths

### [h1] fresh fill single key

```bash
# from rhachet repo root
rhx keyrack fill --env all --key MY_API_KEY
```

**verify:**
- [ ] prompts for key value
- [ ] shows set output in treebucket
- [ ] shows unlock verification line
- [ ] shows get verification line
- [ ] exits 0

### [h2] skip behavior (key already set)

```bash
# first, set a key
rhx keyrack set --key REGULAR_KEY --env all --vault os.secure

# then fill
rhx keyrack fill --env all --key REGULAR_KEY
```

**verify:**
- [ ] shows "found vaulted under" with slug
- [ ] does not prompt for value
- [ ] exits 0

### [h3] refresh forces re-prompt

```bash
# with key already set from [h2]
rhx keyrack fill --env all --key REGULAR_KEY --refresh
```

**verify:**
- [ ] prompts for value despite key extant
- [ ] overwrites extant key
- [ ] verifies roundtrip
- [ ] exits 0

### [h4] --help shows usage

```bash
rhx keyrack fill --help
```

**verify:**
- [ ] shows --env option (required)
- [ ] shows --owner option (repeatable)
- [ ] shows --prikey option (repeatable)
- [ ] shows --key option (filter)
- [ ] shows --refresh option
- [ ] shows --repair option
- [ ] shows --allow-dangerous option

### [h5] without --env fails fast

```bash
rhx keyrack fill
```

**verify:**
- [ ] error: --env is required
- [ ] exits non-zero

## edge paths

### [e1] no keyrack.yml in repo

```bash
cd /tmp
mkdir test-no-manifest && cd test-no-manifest
git init
rhx keyrack fill --env test
```

**verify:**
- [ ] error: no keyrack.yml found
- [ ] exits non-zero

### [e2] key not found in manifest

```bash
# from rhachet repo root
rhx keyrack fill --env all --key NONEXISTENT_KEY
```

**verify:**
- [ ] error: key NONEXISTENT_KEY not found in manifest for env=all
- [ ] exits non-zero

### [e3] no keys for env

```bash
# rhachet keyrack.yml has env.prod: [] (empty)
rhx keyrack fill --env prod
```

**verify:**
- [ ] shows "no keys found"
- [ ] exits 0 (not an error)

### [e4] nonexistent owner fails fast

```bash
# from rhachet repo root
rhx keyrack fill --env all --owner nonexistent
```

**verify:**
- [ ] error: no available prikey for owner=nonexistent
- [ ] exits non-zero

### [e5] env=all key satisfies env=test

```bash
# first, set a key with env=all (already done in [h1] with MY_API_KEY)
# test with a different key to avoid state from [h1]
rhx keyrack set --key RELOCK_REGULAR_KEY --env all --vault os.secure

# then fill for env=test
rhx keyrack fill --env test --key RELOCK_REGULAR_KEY
```

**verify:**
- [ ] shows "found vaulted under" with .all. in slug
- [ ] does not prompt for value
- [ ] exits 0

## pass criteria

- all happy paths pass
- all edge paths show expected errors

## fail criteria

- any happy path fails verification
- error messages are unclear or absent
- roundtrip verification fails silently

## cleanup

```bash
# relock to clear daemon state
rhx keyrack relock
```
