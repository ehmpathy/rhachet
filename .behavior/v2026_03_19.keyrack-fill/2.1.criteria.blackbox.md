# 2.1.criteria.blackbox: keyrack fill

## episode experience

### usecase.1 = fill all keys for default owner

```
given(repo with keyrack.yml that declares env.test keys)
  when(user runs `rhx keyrack fill --env test`)
    then(prompts user for each key value)
      sothat(user can enter credentials interactively)
    then(sets each key for owner=default)
    then(unlocks and gets each key to verify roundtrip)
    then(exits 0 when all keys verified)
      sothat(user knows all credentials are configured)
```

### usecase.2 = fill all keys for multiple owners

```
given(repo with keyrack.yml that declares env.test keys)
  when(user runs `rhx keyrack fill --env test --owner default --owner ehmpath --prikey ~/.ssh/ehmpath`)
    then(for each key, prompts user for value per owner)
      sothat(user can enter potentially different values per owner)
    then(sets, unlocks, gets each key for each owner)
    then(inner loop is owners)
      sothat(user can repeat same action for same key across owners before next key)
    then(exits 0 when all keys verified for all owners)
```

### usecase.3 = partial fill (some keys already set)

```
given(repo with keyrack.yml that declares 3 keys)
  given(2 keys already set for owner=default)
    when(user runs `rhx keyrack fill --env test`)
      then(skips already-set keys with "already set, skip")
        sothat(user is not prompted for credentials they already configured)
      then(prompts only for the 1 absent key)
      then(exits 0 when all keys verified or skipped)
```

### usecase.4 = refresh all keys

```
given(repo with keyrack.yml that declares 3 keys)
  given(all keys already set)
    when(user runs `rhx keyrack fill --env test --refresh`)
      then(re-prompts for all keys despite already set)
        sothat(user can rotate or update credentials)
      then(overwrites extant values)
      then(exits 0 when all keys re-verified)
```

### usecase.5 = fill specific key only

```
given(repo with keyrack.yml that declares 3 keys)
  when(user runs `rhx keyrack fill --env test --key CLOUDFLARE_API_TOKEN`)
    then(prompts only for CLOUDFLARE_API_TOKEN)
      sothat(user can fill or refresh a single credential)
    then(ignores other keys in manifest)
    then(exits 0 when specified key verified)
```

### usecase.6 = fill skips keys satisfied by env=all

```
given(repo with keyrack.yml that declares API_KEY in env.test)
  given(API_KEY already set as env=all: `org.all.API_KEY`)
    when(user runs `rhx keyrack fill --env test`)
      then(unlocks key to check if already set)
        sothat(fallback resolution finds env=all key)
      then(gets key to verify roundtrip)
      then(recognizes env=all key satisfies env=test requirement)
      then(skips with "already set (org.all.API_KEY), skip")
        sothat(user is not prompted for credentials already configured under env=all)
      then(slug in skip message shows .all. segment)
        sothat(user knows which key satisfied the requirement)
```

note: fill uses unlock → get to check if a key is already set. unlock does env=all fallback, so `org.all.API_KEY` satisfies `org.test.API_KEY`. the skip message shows the actual slug that was found.

### usecase.7 = fill fails fast on blocked keys

```
given(repo with keyrack.yml that declares GITHUB_TOKEN)
  given(GITHUB_TOKEN is set but blocked (dangerous token detected))
    when(user runs `rhx keyrack fill --env test`)
      then(fail-fast with error that explains blocked status)
      then(message suggests --repair or --allow-dangerous)
        sothat(user opts into knowledge that fill has blocked dangerous keys)
      then(exits 1)
```

### usecase.8 = repair blocked keys

```
given(repo with keyrack.yml that declares GITHUB_TOKEN)
  given(GITHUB_TOKEN is set but blocked (dangerous token detected))
    when(user runs `rhx keyrack fill --env test --repair`)
      then(prompts user for new value)
        sothat(user can rotate the dangerous token)
      then(overwrites the blocked key)
      then(verifies roundtrip)
      then(exits 0 when key re-verified)
```

### usecase.9 = allow dangerous keys

```
given(repo with keyrack.yml that declares GITHUB_TOKEN)
  given(GITHUB_TOKEN is set but blocked (dangerous token detected))
    when(user runs `rhx keyrack fill --env test --allow-dangerous`)
      then(skips the blocked key with "blocked (dangerous), allowed" message)
        sothat(user acknowledges the dangerous token but accepts it)
      then(continues to next key)
      then(exits 0 when all keys verified or allowed)
```

## exchange experience

### inputs

```
given(any invocation of keyrack fill)
  then(--env is required)
    sothat(user explicitly chooses which environment to fill)
  then(--env accepts test, prod, or all)

given(--owner not specified)
  then(defaults to owner=default)
    sothat(single-owner is the common case)

given(--owner specified multiple times)
  then(fills keys for each specified owner)
    sothat(user can configure multiple keyracks in one run)

given(--prikey specified)
  then(adds to pool of prikeys to consider for owner manifest decryption)
  then(does not imply owner association)
    sothat(system tries each prikey against each owner until one works)

given(--key specified)
  then(filters to only that key)
  then(ignores other keys in manifest)

given(--refresh specified)
  then(re-prompts even for already-configured keys)

given(--repair specified)
  then(overwrites blocked keys instead of fail-fast)
    sothat(user can rotate dangerous tokens)

given(--allow-dangerous specified)
  then(skips blocked keys instead of fail-fast)
    sothat(user can accept dangerous tokens as-is)
```

### outputs

```
given(all keys successfully set, unlocked, and gotten)
  then(exits 0)
    sothat(scripts can depend on success)

given(any key fails roundtrip verification)
  then(exits 1)
    sothat(scripts can detect failure)

given(successful fill)
  then(outputs tree with each key × owner result)
  then(shows keyrack set output nested within treebucket)
  then(shows unlock and get verification steps)
```

### errors

```
given(no prikey can decrypt owner's host manifest)
  then(fail-fast with "no available prikey for owner=X")
    sothat(user knows which owner's prikey is absent)

given(no keys match specified --env)
  then(warns "no keys found for env=test")
  then(exits 0)
    sothat(empty manifest is not an error)

given(user enters empty value when prompted)
  then(fail-fast with "value cannot be empty")
    sothat(user cannot accidentally set empty credentials)

given(keyrack.yml has extends cycle)
  then(fail-fast at manifest expansion)
    sothat(user knows manifest is malformed)

given(specified --key not found in manifest)
  then(fail-fast with "key X not found in manifest for env=Y")
    sothat(user knows they specified an invalid key)

given(key is blocked (dangerous token detected))
  given(neither --repair nor --allow-dangerous specified)
    then(fail-fast with error message)
    then(message includes "use --repair to overwrite or --allow-dangerous to accept as-is")
      sothat(user opts into knowledge of blocked dangerous keys)
```

### boundary conditions

```
given(--env all)
  then(fills keys from both env.test and env.prod)
    sothat(user can configure all environments at once)

given(key has prescribed vault in manifest)
  then(uses prescribed vault)

given(key has no prescribed vault)
  then(infers vault from key characteristics if possible)
  then(falls back to os.secure when not inferrable)
    sothat(fill always has a vault to use)

given(owner's prikey is in ssh-agent)
  then(discovers and uses it without --prikey flag)
    sothat(user doesn't need to specify prikeys already unlocked)
```
