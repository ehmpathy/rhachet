# 3.2.distill.domain._.v1.i1

## usecases

| usecase | contract |
|---------|----------|
| fill all keys for default owner | `rhx keyrack fill --env test` |
| fill all keys for multiple owners | `rhx keyrack fill --env test --owner default --owner ehmpath --prikey ~/.ssh/ehmpath` |
| fill specific key only | `rhx keyrack fill --env test --key CLOUDFLARE_API_TOKEN` |
| refresh all keys | `rhx keyrack fill --env test --refresh` |

## domain operations

### extant operations [REUSE]

| operation | file | role in fill |
|-----------|------|--------------|
| `setKeyrackKey` | setKeyrackKey.ts | set key for one owner |
| `unlockKeyrackKeys` | unlockKeyrackKeys.ts | unlock key after set |
| `getKeyrackKeyGrant` | getKeyrackKeyGrant.ts | verify roundtrip after unlock |
| `getAllKeyrackSlugsForEnv` | getAllKeyrackSlugsForEnv.ts | filter keys by env |
| `inferKeyrackVaultFromKey` | inferKeyrackVaultFromKey.ts | infer vault when not prescribed |
| `genContextKeyrackGrantUnlock` | genContextKeyrackGrantUnlock.ts | generate unlock context per owner |

### new operations

| operation | type | input | output |
|-----------|------|-------|--------|
| `fillKeyrackKeys` | orchestrator | `{ env, owners, prikeys, key?, refresh? }` | `{ results: FillKeyResult[] }` |

**fillKeyrackKeys pseudocode:**
```
1. load repoManifest
2. get all keys for env via getAllKeyrackSlugsForEnv
3. if --key specified, filter to that key (fail-fast if not found)
4. for each key:
   5. for each owner:
      6. load hostManifest for owner (with prikey pool)
      7. check if key already set in hostManifest
      8. if already set and not --refresh: skip with "already set, skip"
      9. infer vault if not prescribed (fallback to os.secure)
      10. prompt for secret via promptHiddenInput
      11. call setKeyrackKey
      12. call unlockKeyrackKeys
      13. call getKeyrackKeyGrant to verify roundtrip
      14. if verification fails: report error, continue
5. exit 0 if all verified, exit 1 if any failed
```

## access layer

### extant DAOs [REUSE]

| DAO | role in fill |
|-----|--------------|
| `daoKeyrackRepoManifest.get()` | load repo manifest with extends resolution |
| `daoKeyrackHostManifest.get()` | load host manifest (with prikey discovery) |
| `daoKeyrackHostManifest.set()` | write key to host manifest (via setKeyrackKey) |

### prikey pool extension

the `--prikey` flags extend the pool of prikeys to consider. the mechanism:
1. `daoKeyrackHostManifest.setSuppliedPrikeys([...paths])`
2. trial-decryption considers supplied prikeys alongside discovered ones

## relationships

```
fillKeyrackKeys (orchestrator)
├── daoKeyrackRepoManifest.get() — load manifest
├── getAllKeyrackSlugsForEnv() — filter keys by env
├── for each key:
│   └── for each owner:
│       ├── daoKeyrackHostManifest.get() — load host manifest for owner
│       ├── inferKeyrackVaultFromKey() — infer vault if needed
│       ├── promptHiddenInput() — get secret from user
│       ├── setKeyrackKey() — set key
│       ├── unlockKeyrackKeys() — unlock key
│       └── getKeyrackKeyGrant() — verify roundtrip
└── aggregate results and report
```

## composition

| layer | what fill uses |
|-------|----------------|
| contract/cli | add `keyrack fill` subcommand to invokeKeyrack.ts |
| domain.operations | new fillKeyrackKeys orchestrator |
| access/daos | extant daoKeyrackRepoManifest, daoKeyrackHostManifest |

**key insight:** fill is pure orchestration. no new primitives, no new DAOs, no new domain objects. it composes extant pieces in the standard loop: for each key × for each owner → set → unlock → get.
