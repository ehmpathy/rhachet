## matrix 1: fill scope

which keys get filled based on input flags

| ind: --env | ind: --key | dep: keys filled |
|------------|------------|------------------|
| test | (none) | all env.test keys |
| prod | (none) | all env.prod keys |
| all | (none) | all env.test + env.prod keys |
| test | CLOUDFLARE_API_TOKEN | only CLOUDFLARE_API_TOKEN from env.test |

| all | CLOUDFLARE_API_TOKEN | CLOUDFLARE_API_TOKEN from both envs |

## matrix 2: owner iteration

which owners get filled based on --owner flags

| ind: --owner flags | dep: owners filled | dep: iteration order |
|--------------------|--------------------|-----------------------|
| (none) | default only | key → default |
| --owner default | default only | key → default |
| --owner ehmpath | ehmpath only | key → ehmpath |
| --owner default --owner ehmpath | both | key → default → ehmpath (inner loop on owners) |

## matrix 3: key state behavior

what happens for each key × owner based on state and --refresh

| ind: key state | ind: --refresh | dep: behavior |
|----------------|----------------|---------------|
| not set | false | prompt, set, verify |
| not set | true | prompt, set, verify |
| already set | false | skip with "already set, skip" |
| already set | true | prompt, set, verify (overwrite) |

## matrix 4: prikey resolution

|--------------------------|------------------------|------------------------------|--------------|
| yes | no | ssh-agent prikey | use discovered prikey |
| no | yes | supplied prikey | use supplied prikey |
| yes | yes | ssh-agent prikey | use discovered prikey |
| yes | yes | supplied prikey | use supplied prikey |
| no | no | — | fail-fast: no available prikey |
| yes | yes | neither | fail-fast: no available prikey |
