# blocker.1


### 1.3. interface asymmetry: config.get vs config.set — cognitive load

**location**: `BrainCli` interface

```typescript
config: {
  get(input: { scope: string }, context): Promise<{ uri: string; scope: string } | null>;
  set(input: { roles: RoleSlug[]; scope: string }, context): Promise<void>;
}
```

**problem**: 
- `get()` returns `{ uri, scope }` — no `roles`
- `set()` takes `{ roles, scope }` — produces no `uri`

caller cannot answer: "what roles does this config have?" without re-computing from scope hash. this is a **lossy roundtrip**.

**fix**: persist roles in config (e.g., `enrollment.json` or header in CLAUDE.md)

```typescript
get(): Promise<{ uri: string; scope: string; roles: RoleSlug[] } | null>;
```

**note**: vision.yield.md line 181 shows `get()` returning `roles`, but blueprint drops it. restore it.

# blocker.7

### 3.3. asScopeHash has no version/namespace

**location**: `asScopeHash`

```typescript
return createHash('sha256').update(sorted.join(',')).digest('hex').slice(0, 8);
```

**problem**: if we change the hash algorithm or the join format, existing scoped configs are orphaned (different hash, same roles).

**fix**: include version prefix

```typescript
const sorted = [...input.roles].sort();
const payload = `v1:${sorted.join(',')}`;
return createHash('sha256').update(payload).digest('hex').slice(0, 8);
```

**note**: 8 chars = 32 bits = 4B combinations. sufficient for now, but document collision risk.
