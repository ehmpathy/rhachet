### 1. **Slug Asymmetry Breaks Discovery Pattern**

| BrainAtom/BrainRepl | BrainCli (proposed) |
|---------------------|---------------------|
| `repo: string` + `slug: string` | `slug: string` only |
| `{ repo: 'anthropic', slug: 'claude-opus-4.5' }` | `{ slug: 'claude/code' }` |

**Problem:** Discovery via `getBrainAtomsBy*` / `getBrainReplsBy*` / `getBrainClisBy*` uses `repo` to group by supplier. The `repo/slug` concatenation loses this grouping and breaks deduplication.

**Fix:** Add `repo: string` field to BrainCli interface:
```typescript
interface BrainCli {
  repo: string;  // 'anthropic'
  slug: string;  // 'claude-code'
  // ...
}
```

### 2. **TContext Generic Missing**

BrainAtom and BrainRepl have `<TContext = Empty>` for typed context injection. BrainCli does not.

**Problem:** Suppliers can't type their required context. Callers lose type safety at config.set() and spawn() calls.

**Fix:**
```typescript
interface BrainCli<TContext = Empty> {
  config: {
    set(input: {...}, context: TContext & ContextCli): Promise<void>;
  };
  spawn(input: {...}, context: TContext & ContextCli): ChildProcess;
}
```

### 3. **config.set() Creates Adapters — Wrong Temporal Coupling**

```typescript
// CURRENT: adapters created inside config.set
config: {
  set: async ({ roles, scope }, context) => {
    const adapters = {
      boots: genClaudeCodeBootsAdapter({ configUri }),  // created every call
      hooks: genClaudeCodeHooksAdapter({ configUri }),
      // ...
    };
    await setBrainCliConfig({ roles, scope }, { ...context, adapters });
  }
}
```

**Problem:** 
- Adapters recreated on every `config.set()` call (inefficient)
- configUri computation duplicated in `get()`, `set()`, `spawn()`
- Adapter creation coupled to invocation rather than factory construction

**Fix:** Create adapters at factory time, pass configUri resolver:
```typescript
export const genBrainCliForClaudeCode = (input: { slug: string }): BrainCli => {
  const getConfigUri = (scope: string, context: ContextCli) => 
    asConfigUri({ gitroot: context.gitroot, brain: 'claude-code', scope });

  return {
    slug: input.slug,
    config: {
      set: async ({ roles, scope }, context) => {
        const configUri = getConfigUri(scope, context);
        // adapters created once per call with fresh configUri
        // ...
      }
    }
  };
};
```

### 15. **Separate Boot Content Generation from Config Writing**

Current `setBrainCliConfig` does:

2. Collect hooks
3. Write via adapters

**Fix:** Split:
```typescript
// Pure transformer
const genBrainCliConfigContent = (input: { roles }, context): { boots: string; hooks: HooksConfig };

// Side-effectful writer (orchestrator)  
const setBrainCliConfig = async (input: { roles, scope }, context: { adapters, ... }) => {
  const content = await genBrainCliConfigContent({ roles: orderedRoles }, context);
```
