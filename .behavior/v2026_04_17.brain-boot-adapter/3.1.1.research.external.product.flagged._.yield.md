# research: vision-flagged topics

## topic.1 = MCP servers cache impact

**question**: does different `.mcp.json` in different repos break cache prefix before CLAUDE.md?

### findings

**[FACT]** Tool definitions are part of the cached prefix, assembled BEFORE system prompt in cache hierarchy.
- Source [1]: "Cache prefixes are created in this hierarchy: 1. Tools → 2. System → 3. Messages"

**[FACT]** Claude Code uses deferred tool load (`defer_loading: true`) for MCP tools.
- Source [2]: "Only tool stubs (name + defer_loading flag) are cached at session start, not full schemas. Full tool schemas load only when the model selects them."

**[SUMP]** Deferred load keeps the prefix stable even with multiple MCP servers.
- Source [2]: "This keeps the cached prefix stable: the same stubs are always present in the same order."

**[FACT]** Project-level `.mcp.json` has documented bugs and may not load consistently.
- Source [6], [7]: GitHub issues #15215 and #3321 report project-level `.mcp.json` doesn't load tools reliably
- **Practical result**: Most users configure MCP in global `~/.claude.json`, so different repos don't trigger cache invalidation

**[KHUE]** If project-level `.mcp.json` did work reliably, would it break cache across repos?
- Likely yes, but current bugs prevent this from an issue

## topic.2 = project settings cache impact

**question**: do different `.claude/settings.json` files affect the prefix before CLAUDE.md?

### findings

**[FACT]** Settings.json uses scope hierarchy: Managed → User → Project → Local.
- Source [8]: Each level overrides previous ones

**[FACT]** Settings.json influences what gets assembled (tools, env vars, git status), not direct prompt content.
- Source [8], [9]: Settings control permissions, hooks, and environment—file itself isn't injected as text

**[SUMP]** Git status injection is a primary cache-buster.
- Source [9]: "Claude Code injects live git status output into the system prompt on every call"
- Set `"includeGitInstructions": false` helps stabilize the system prompt

**[KHUE]** Which settings.json changes bust the cache prefix?
- No complete list publicly documented
- Known: `includeGitInstructions` affects git status injection

### sources

8. [official] Claude Code Settings Docs (https://code.claude.com/docs/en/settings)
9. [practitioner] Claude Code Camp - How Prompt Caching Actually Works (https://www.claudecodecamp.com/p/how-prompt-caching-actually-works-in-claude-code)

## topic.3 = 1-hour TTL feature request

**question**: is there a feature request #19436 for 1-hour TTL on CLAUDE.md?

### findings

**[FACT]** Issue #19436 exists and proposes multi-tiered cache with 1-hour TTL.
- Source [10]: "Feature Request: Multi-Tiered Prompt Cache for Cost Optimization"
- Status: Closed as duplicate

**[FACT]** Proposal: three-tier cache structure
- Tier 1 (System prompt + tools): 1-hour TTL
- Tier 2 (CLAUDE.md): 1-hour TTL
- Tier 3 (Skills, hooks, environment): 5-minute TTL

**[FACT]** Quote from issue: "Claude Code currently uses only 5-minute prompt cache (ephemeral_5m) for all cached content, despite the Claude API support for 1-hour cache (ephemeral_1h) at the same price."

**[FACT]** TTL regression discovered: Anthropic silently reverted from 1-hour to 5-minute TTL in early March 2026.
- Source [11]: Issue #46829 documents 17.1% cost increase from regression
- Source [12]: Issue #45381 found telemetry bug that disables 1-hour TTL

**[OPIN]** 1-hour TTL would significantly reduce cache write costs for stable content like CLAUDE.md.
- Source [10]: "56% reduction on cache operations for 10 sessions/hour"

## conflicts detected

1. **TTL default ambiguity**: Sources [11], [12] conflict on whether 5-minute or 1-hour is the "intended" default. February 2026 logs show 1-hour; March 2026 shows 5-minute. Issue #46829 requests official clarification.

2. **Project-level MCP reliability**: Official docs (source [1]) say `.mcp.json` should work at project level, but GitHub issues (sources [6], [7]) report it doesn't load consistently.

---

## convergence signals

**3+ sources agree**:
- Tool definitions are part of cached prefix (sources [1], [2], [9])
- Deferred tool load reduces cache instability (sources [2], [4])
- 5-minute TTL is current default, despite API support for 1-hour (sources [10], [11], [13])

---

## implications for brain-boot-adapter

1. **MCP servers**: Not a blocker. Project-level `.mcp.json` bugs mean most users use global config, which doesn't vary per-repo.

2. **Settings.json**: Minor concern. The main cache-buster is git status injection, which happens regardless of CLAUDE.md placement.

3. **1-hour TTL**: Future opportunity. If Anthropic restores 1-hour TTL or implements multi-tiered cache, our CLAUDE.md approach would benefit even more.
