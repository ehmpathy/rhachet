# blocker.1: Missing real integration tests for external Claude CLI

**rule**: require.external-contract-integration-tests

**locations**:
- src/domain.operations/enroll/enrollBrainCli.test.ts

The rule requires that every external contract, such as SDKs, APIs, or services like the Claude CLI being spawned in enrollBrainCli, must have at least one integration test that calls the real service with real credentials and validates responses. Currently, all tests mock the external interactions (e.g., spawning the Claude CLI or invoking resources), providing false confidence without proving real integration works. Lack of credentials is not an excuse; real tests must exist. Add integration tests that actually spawn the 'claude' command with real args and verify real responses, ensuring no silent bypasses.
