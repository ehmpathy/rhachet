## key insights

1. **adapter pattern is established**: BrainHooksAdapter + DAO provides clean abstraction; BrainBootsAdapter follows same shape

2. **config output path evolution**:
   - current: `.claude/settings.enroll.$hash.local.json`
   - target: `.agent/.brain/claude/config/scope=$scope/` with CLAUDE.md + settings.json + .credentials.json symlink

3. **bootRoleResources already formats content**: no need to rewrite; capture stdout to file

4. **CLAUDE_CONFIG_DIR is the key**: env var tells Claude Code where to read CLAUDE.md from (no CLI flag exists)

5. **boot hook validation becomes obsolete**: `assertRegistryBootHooksDeclared` enforces that roles declare boot hooks; with BrainBootsAdapter, bootable content goes to CLAUDE.md directly
