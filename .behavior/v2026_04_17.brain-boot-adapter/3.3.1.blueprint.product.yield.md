# blueprint: brain-boot-adapter

## tenets

1. **zero brain supplier internals leak out of the factory** — orchestrators call `cli.config.set({ roles, scope })`, adapters handle paths, files, env vars. No knowledge of claude-code's directory structure, settings.json format, or CLAUDE.md file names escapes the adapter.

2. **BrainCli is discoverable like BrainAtom and BrainRepl** — brain packages export `getBrainClisByAnthropic()` with same pattern as `getBrainAtomsByAnthropic()` and `getBrainReplsByAnthropic()`. Discovery via `getAvailableBrains` returns `{ atoms, repls, clis }`. CLIs are programmatically accessible, not just CLI-invoked.

---

## summary

Move ~50k tokens of static bootable role content (briefs, skills) from SessionStart hooks (prompt position 23, after dynamic content) to CLAUDE.md (position 2, before dynamic content) to enable prompt cache prefix match across sessions.

**primary deliverables:**
1. `BrainCli` interface — `{ slug, config: { get(), set(input, ctx) }, spawn() }` — peer to BrainAtom, BrainRepl
2. `getBrainClisByAnthropic()` export — discoverable like atoms/repls
3. `getAvailableBrains` extended — returns `{ atoms, repls, clis }`
4. `genBrainCliForClaudeCode({ slug })` — supplier-defined factory (not in rhachet core)
5. `cli.config.set({ roles, scope })` — triggers adapter chain via setBrainCliConfig
6. Deprecate BrainHooksAdapter (replaced by BrainCli)
7. Config directory structure: `.agent/.brain/$brain/config/scope=$scope/`
8. `rhx init $brain --roles` generates default config with CLAUDE.md (e.g., `rhx init claude --roles mechanic,architect`)
9. `rhx enroll $brain` discovers cli via getAvailableBrains, calls `cli.config.set()` then `cli.spawn({ scope })`
10. `rhx upgrade` regenerates each brain's default config after role upgrades

**expected benefit:** ~$0.25 saved per compaction event, cache shared across sessions with same roles.

---

## vision-to-blueprint deviations

| vision said | blueprint does | rationale |
|-------------|----------------|-----------|
| `genBrainCli` in rhachet core | supplier-defined factories only | same pattern as BrainAtom/BrainRepl: suppliers define factories, rhachet defines interfaces |
| factory takes `{ roles, scope }` | factory takes `{ slug }` only | `config.set({ roles, scope })` triggers adapters — decouples factory from config contract |
| separate config vs spawn steps | `config.set()` then `spawn()` | explicit steps, caller controls when config is set vs when session spawns |
| `adapters: { hooks, boots, auth }` | `setBrainCliConfig` + adapter interfaces | rhachet publishes orchestrator + interfaces; suppliers implement adapters + call via config.set() |

**key insights:**
1. rhachet publishes `setBrainCliConfig` so suppliers don't reinvent the roles→config orchestration. Suppliers only implement brain-specific adapters (file formats, paths, env vars).
2. CLIs are discoverable via `getBrainClisBy*` exports, same pattern as atoms/repls. `getAvailableBrains` returns `{ atoms, repls, clis }` for programmatic access.
3. `genBrainCliForClaudeCode({ slug })` is supplier-defined (not in rhachet core) — same pattern as `genBrainAtom`, `genBrainRepl`.
4. `cli.config.set({ roles, scope })` triggers the adapter chain internally — orchestrators need not know BrainCliConfig contract.

## CLI contract changes

### rhx init (new $brain positional or --brain flag)

```
rhx init $brain --bind [--roles $roles]
rhx init --brain $brain --bind [--roles $roles]

positional:
  $brain              brain slug (claude, opencode) — NEW positional arg

flags:
  --brain $brain      brain slug — alternative to positional (NEW)
  --bind              bind roles to brain config (creates CLAUDE.md + settings.json) — REQUIRED for config
  --roles $roles      comma-separated role slugs (default: all linked roles) — extant flag

examples:
  rhx init claude --bind                           # all linked roles
  rhx init claude --bind --roles mechanic,architect
  rhx init --brain claude --bind --roles mechanic,architect  # flag alternative
```

**output:** creates `.agent/.brain/$brain/config/scope=default/` with:
- `CLAUDE.md` — boot content (briefs + skills) with header comment
- `settings.json` — hooks config
- `.gitignore` — ignores `.credentials.json` (created at enroll time, not committed)

**note:** `--bind` is required to trigger config creation. Without it, `rhx init` does legacy behavior (link roles only).

**note:** `.credentials.json` symlink is NOT created at init time — it's created at enroll time. This avoids cross-machine symlink breakage (absolute paths with different usernames).

**validation (at contract boundary):**
- if `--roles` empty or invalid → `ConstraintError: --roles required with --bind`
- validation in `invokeInit`, NOT in domain operations (item.8)

### rhx enroll (positional or --brain flag)

```
rhx enroll $brain [--roles $roles]
rhx enroll --brain $brain [--roles $roles]

positional:
  $brain              brain slug (claude, opencode) — extant positional

flags:
  --brain $brain      brain slug — alternative to positional (NEW)
  --roles $roles      custom role set (e.g., +architect,-driver) — creates scoped config

examples:
  rhx enroll claude                                 # use default config
  rhx enroll claude --roles +architect,-driver      # create/use scoped config
  rhx enroll --brain claude                         # flag alternative
```

**behavior change:** discovers cli, calls config.set then cli.spawn({ scope })

**credentials symlink:** created at enroll time (not init time):
- creates `.credentials.json` symlink to `~/.claude/.credentials.json`
- symlink is gitignored — not committed to repo
- if credentials absent → `ConstraintError: credentials not found. run claude to authenticate first.`

**spawn behavior:** cli.spawn() returns BrainCliSession (abstract). internally:
- `detached: false` — caller owns lifecycle (not orphaned)
- `stdio: 'inherit'` — user interacts directly with brain
- returns `{ wait(), terminate() }` — abstract, not Node ChildProcess

**validation (at contract boundary):**
- if default config absent → `ConstraintError: no config for scope=default. run 'rhx init $brain --bind' first`

### rhx upgrade (updated behavior)

```
rhx upgrade [--roles $roles]
```

**behavior change:** after role packages upgraded, regenerates default config CLAUDE.md via genBrainCli

**brain discovery:** reads initialized brains from registry (via getAvailableBrains), not filesystem glob

**architecture:**
- `BrainCli` interface — `{ slug, config: { get(), set(input, ctx) }, spawn() }` — peer to BrainAtom, BrainRepl
- `genBrainCliForClaudeCode({ slug })` — supplier-defined factory, returns BrainCli (no context at factory time)
- `cli.config.set({ roles, scope }, context)` — triggers adapter chain via setBrainCliConfig internally
- `cli.config.get({ scope }, context)` — read config from disk (uri, scope, roles) or null
- `cli.spawn({ scope?, config?: { uri } }, context)` — spawns interactive session (requires scope or config.uri)
- `setBrainCliConfig({ roles, scope }, context: { adapters })` — orchestrator published by rhachet
- adapter interfaces: `BrainCliConfigBootsAdapter`, `BrainCliConfigHooksAdapter`, `BrainCliConfigAuthAdapter`, `BrainCliConfigMetaAdapter`
- suppliers implement adapters + call `setBrainCliConfig` via config.set()
- `rhachet-brains-anthropic` is proof of concept for full integration
- pattern matches BrainAtom/BrainRepl exactly: factory returns capability, internals hidden
- genBrainCli is NOT in rhachet core — suppliers define their own factories
- deprecates separate `BrainHooksAdapter` and `BrainBootsAdapter`

```typescript
// src/domain.objects/BrainCli.ts
// peer to BrainAtom, BrainRepl — supplier-defined factory returns capability, internals hidden
// NOTE: factory takes only slug, config.set accepts context at call time
export interface BrainCli<TContext = Empty> {
  /**
   * .what = supplier repo
   * .why = groups by supplier in discovery (same pattern as BrainAtom/BrainRepl)
   * .example = "anthropic"
   */
  repo: string;

  /**
   * .what = brain slug this cli is for
   * .example = "claude-code"
   */
  slug: string;

  /**
   * .what = config operations (get/set/del)
   * .why = set triggers adapter chain (boots, hooks, auth, meta); get reads from disk
   * .note = stateless — no in-memory cache, disk is source of truth
   * .note = upgrade always overwrites (no version tracking for now)
   * .note = set returns config for immediate use (avoids roundtrip via get)
   */
  config: {
    get(input: { scope: string }, context: TContext & ContextCli): Promise<{ uri: string; scope: string; roles: RoleSlug[] } | null>;
    set(input: { roles: RoleSlug[]; scope: string }, context: TContext & ContextCli): Promise<{ uri: string; scope: string; roles: RoleSlug[] }>;
    del(input: { scope: string }, context: TContext & ContextCli): Promise<void>;
  };

  /**
   * .what = spawn interactive session
   * .why = encapsulates CLI invocation (env vars, args, stdio)
   * .note = context required (gitroot for cwd); must pass scope or config.uri (no in-memory state)
   * .note = returns BrainCliSession — abstract lifecycle, not Node ChildProcess
   */
  spawn(input: { scope?: string; config?: { uri: string } }, context: TContext & ContextCli): BrainCliSession;
```

```typescript
export const setBrainCliConfig = async (
```

### asScopeHash (deterministic hash for custom roles)

```typescript
// src/domain.operations/brain/cli/asScopeHash.ts
// NOTE: no validation here — invokeInit/invokeEnroll validate at contract boundary
// NOTE: 8 chars = 32 bits = 4B combinations — sufficient for now
export const asScopeHash = (input: { roles: RoleSlug[] }): string => {
  // dedupe (prevents ['a', 'a'] → different hash than ['a'])
  const deduped = [...new Set(input.roles)];
  // trim (prevents ['a ', ' a'] → different hash than ['a'])
  const trimmed = deduped.map((r) => r.trim());
  // sort for determinism across machines
  const sorted = [...trimmed].sort();
  // version prefix enables future format changes without orphaned configs
  const payload = `v1:${sorted.join(',')}`;
  // sha256 truncated to 8 chars
  return createHash('sha256').update(payload).digest('hex').slice(0, 8);
};

// collision detection: check if scope dir exists with different roles
export const checkScopeHashCollision = async (
  input: { roles: RoleSlug[]; scope: string },
  context: { configDir: string },
): Promise<void> => {
  const metaPath = join(context.configDir, `scope=${input.scope}`, 'meta.json');
  if (!existsSync(metaPath)) return;  // no collision — scope does not exist
  const meta = JSON.parse(readFileSync(metaPath, 'utf-8'));
  const rolesMatch = JSON.stringify([...input.roles].sort()) === JSON.stringify([...meta.roles].sort());
  if (!rolesMatch) {
    throw new ConstraintError('scope hash collision: directory exists with different roles', {
      scope: input.scope,
      rolesRequested: input.roles,
      rolesExtant: meta.roles,
    });
  }
};
```

### parseRolesSpec (transformer — parse +/- role spec)

```typescript
// src/domain.operations/roles/parseRolesSpec.ts
// parses role spec syntax: +role adds, -role removes
// base set = all linked roles from .agent/repo=*/role=*/
export const parseRolesSpec = (input: {
  spec: string;  // e.g., "+architect,-driver"
  linkedRoles: RoleSlug[];
}): RoleSlug[] => {
  const parts = input.spec.split(',').map((p) => p.trim()).filter(Boolean);

  // validate syntax: each part must start with + or -
  for (const part of parts) {
    if (!part.startsWith('+') && !part.startsWith('-')) {
      throw new ConstraintError('invalid role spec syntax. use +role to add, -role to remove.', { part });
    }
  }

  // collect adds and removes
  const adds = parts.filter((p) => p.startsWith('+')).map((p) => p.slice(1));
  const removes = parts.filter((p) => p.startsWith('-')).map((p) => p.slice(1));

  // validate adds/removes exist in linked roles (or are being added)
  for (const role of [...adds, ...removes]) {
    if (!input.linkedRoles.includes(role)) {
      throw new ConstraintError('role not found in .agent/', { role, linkedRoles: input.linkedRoles });
    }
  }

  // compute final set: base + adds - removes
  const base = new Set(input.linkedRoles);
  for (const role of adds) base.add(role);
  for (const role of removes) base.delete(role);

  return [...base];
};
```

### genBrainCliForClaudeCode (supplier factory — repo + slug, context at config.set time)

```typescript
// src/_topublish/rhachet-brains-anthropic/src/genBrainCliForClaudeCode.ts
import { spawn } from 'child_process';
// supplier factory: returns BrainCli capability
// encapsulates all claude-code specifics (CLAUDE.md, settings.json, credentials, CLAUDE_CONFIG_DIR)
// NOTE: repo + slug at factory time — context passed at config.set time (matches atom/repl pattern)
export const genBrainCliForClaudeCode = (
  input: { repo: string; slug: string },
): BrainCli => {
  // NOTE: no in-memory state — disk is source of truth
  // config.get() reads from disk, config.set() writes to disk

  // helper: get config uri for a scope (uses centralized path convention)
  const getConfigUri = (scope: string, context: ContextCli) =>
    getScopeConfigDir({ gitroot: context.gitroot, brain: 'claude-code', scope });

  // return BrainCli capability — config.set triggers adapter chain
  return {
    repo: input.repo,
    slug: input.slug,
    config: {
      get: async ({ scope }, context) => {
        const configUri = getConfigUri(scope, context);

        // check if config exists on disk
        const claudeMdPath = join(configUri, 'CLAUDE.md');
        if (!existsSync(claudeMdPath)) return null;

        // read roles from meta.json (roundtrip: set persists, get retrieves)
        const metaPath = join(configUri, 'meta.json');
        const meta = existsSync(metaPath) ? JSON.parse(readFileSync(metaPath, 'utf-8')) : { roles: [] };

        return { uri: configUri, scope, roles: meta.roles };
      },
      set: async ({ roles, scope }, context) => {
        const configUri = getConfigUri(scope, context);

        // check for hash collision before write
        await checkScopeHashCollision({ roles, scope }, { configDir: getBrainConfigDir({ gitroot: context.gitroot, brain: 'claude-code' }) });

        // create config directory
        await mkdir(configUri, { recursive: true });

        // create claude-specific adapters (cheap to recreate — no state)
        const adapters = {
          boots: genClaudeCodeBootsAdapter({ configUri }),      // writes CLAUDE.md
          hooks: genClaudeCodeHooksAdapter({ configUri }),      // writes settings.json
          auth: genClaudeCodeAuthAdapter({ configUri }),        // symlinks .credentials.json
          meta: genClaudeCodeMetaAdapter({ configUri }),        // writes meta.json (roles, scope)
        };

        // call rhachet's orchestrator (roles/scope → adapter calls)
        const config = await setBrainCliConfig({ roles, scope }, { ...context, adapters });

        // gitignore for scoped configs (not tracked)
        if (!isTrackedScope({ scope })) {
          await writeFile(join(configUri, '.gitignore'), '*\n');
        }

        // return config for immediate use (avoids roundtrip via get)
        return config;
      },
      del: async ({ scope }, context) => {
        const configUri = getConfigUri(scope, context);

        // remove entire scope directory
        if (existsSync(configUri)) {
          await rm(configUri, { recursive: true, force: true });
        }
      },
    },
    spawn: (options, context): BrainCliSession => {
      // config.uri or scope required — no in-memory state
      if (!options?.config?.uri && !options?.scope) {
        throw new ConstraintError('spawn requires config.uri or scope. pass { config: { uri } } or { scope }');
      }

      // compute config uri from scope if not explicit
      const configUri = options?.config?.uri ?? getConfigUri(options.scope!, context);

      // verify credentials exist before spawn (failfast)
      const credPath = join(configUri, '.credentials.json');
      if (!existsSync(credPath)) {
        throw new ConstraintError('credentials not found. run claude to authenticate first.', { credPath });
      }

      // spawn claude with scoped config — detached with stdio inherit
      // NOTE: stdio inherit lets user interact directly
      const child = spawn('claude', [], {
        cwd: context.gitroot,
        stdio: 'inherit',
        detached: false,  // not detached — caller owns lifecycle
        env: { ...process.env, CLAUDE_CONFIG_DIR: configUri },
      });

      // abstract to BrainCliSession (not Node-specific)
      return {
        wait: () => new Promise((done) => {
          child.on('exit', (code) => done({ exitCode: code ?? 0 }));
        }),
        terminate: () => child.kill(),
      };
    },
  };
};
```

### genClaudeCodeAuthAdapter (claude-specific adapter impl)

```typescript
// src/_topublish/rhachet-brains-anthropic/src/adapters/genClaudeCodeAuthAdapter.ts
// implements BrainCliConfigAuthAdapter — symlinks .credentials.json
export const genClaudeCodeAuthAdapter = (input: { configUri: string }): BrainCliConfigAuthAdapter => {
  // respect CLAUDE_HOME if set (user override), else default ~/.claude
  const claudeHome = process.env.CLAUDE_HOME ?? join(homedir(), '.claude');
  const defaultCredSrc = join(claudeHome, '.credentials.json');
  const credDst = join(input.configUri, '.credentials.json');

  return {
    get: async () => {
      if (!existsSync(credDst)) return null;
      return { uri: credDst };
    },
    set: {
      upsert: async ({ source }) => {
        const credSrc = source ?? defaultCredSrc;  // explicit source or default
        if (!existsSync(credSrc))
          throw new ConstraintError('credentials not found. run claude to authenticate first.', { credSrc });
        // use setSymlink utility — handles broken symlink removal
        await setSymlink({ from: credSrc, to: credDst, idem: 'upsert' });
      },
      findsert: async ({ source }) => {
        const credSrc = source ?? defaultCredSrc;  // explicit source or default
        // skip silently if source doesn't exist — credentials checked at spawn time
        if (!existsSync(credSrc)) return;
        // use setSymlink utility — skips if valid symlink extant
        await setSymlink({ from: credSrc, to: credDst, idem: 'findsert' });
      },
    },
  };
};
```

### setSymlink (utility: symlink upsert/findsert)

```typescript
// src/infra/fs/setSymlink.ts
// utility: symlink with broken-symlink support and idempotent modes
export const setSymlink = async (input: {
  from: string;
  to: string;
  idem: 'upsert' | 'findsert';
}): Promise<void> => {
  // check if symlink extant — note: existsSync returns false for broken symlinks
  const symlinkExtant = lstatSync(input.to, { throwIfNoEntry: false })?.isSymbolicLink() ?? false;

  if (input.idem === 'findsert' && symlinkExtant) {
    // findsert: skip only if valid symlink (target exists)
    // handles credential expiry / re-auth case where old symlink points to deleted target
    const targetExists = existsSync(input.to);  // follows symlink, returns false if broken
    if (targetExists) return;  // valid symlink — skip
    // else: broken symlink — fall through to recreate
  }

  if (symlinkExtant) {
    // remove extant symlink (broken or valid — we're replacing it)
    await unlink(input.to);
  }

  // create parent directories if needed
  await mkdir(dirname(input.to), { recursive: true });

  // create symlink
  await symlink(input.from, input.to);
};
```

## error cases

| scenario | when | error | resolution |
|----------|------|-------|------------|
| no roles linked | init | `no roles found in .agent/` | run `rhx roles link` first |
| invalid role | init | `role 'foobar' not found in .agent/` | use valid role slug |
| empty roles | setBrainCliConfig | `roles array cannot be empty` | pass at least one role |
| brain not found | getBrainCliBySlug | `brain cli not found: 'foobar'` | use supported brain (claude/code) |
| credentials absent | cli.spawn | `credentials not found. run claude to authenticate first.` | run `claude` to authenticate, then retry enroll |
| no scope or uri | cli.spawn | `spawn requires config.uri or scope. pass { config: { uri } } or { scope }` | pass scope or config.uri to spawn() |
| default config absent | invokeEnroll | `no config for scope=default. run 'rhx init claude --bind' first` | run init first |
| invalid roles spec | invokeEnroll | `invalid role spec syntax` | use +role or -role format |
| scope hash collision | cli.config.set | `scope hash collision. delete scope dir or use different roles.` | delete old scope or choose different roles |

**failfast point:** credentials absence fails at spawn time (not config.set time). This enables `rhx init` to succeed without credentials — credentials are validated at enroll time.
