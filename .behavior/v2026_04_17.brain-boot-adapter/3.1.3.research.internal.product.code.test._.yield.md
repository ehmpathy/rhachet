**what**: BDD-style acceptance test with given/when/then, uses fixtures, asserts on stdout/stderr/status

**relation to wish**: pattern for new init/enroll acceptance tests

**tag**: [REUSE]

### citations

1. `blackbox/cli/roles.boot.acceptance.test.ts:6-35`
```typescript
describe('rhachet roles boot', () => {
  given('[case1] repo with briefs', () => {
    const repo = useBeforeAll(async () =>
      genTestTempRepo({ fixture: 'with-briefs' }),
    );

    when('[t0] roles boot --repo this --role any', () => {
      const result = useBeforeAll(async () =>
        invokeRhachetCliBinary({
          args: ['roles', 'boot', '--repo', 'this', '--role', 'any'],
          cwd: repo.path,
        }),
      );

      then('exits with status 0', () => {
        expect(result.status).toEqual(0);
      });

      then('outputs stats', () => {
        expect(result.stdout).toContain('<stats>');
      });
```

## key insights

1. **fixture-based isolation**: each test gets fresh temp repo from fixture template; need fixtures with `.agent/.brain/claude/config/scope=default/` structure

2. **black-box via subprocess**: tests invoke compiled binary, assert on stdout/stderr/status; no internal imports

3. **BDD style established**: given/when/then pattern with useBeforeAll for setup; new tests follow this

4. **verification pattern**: read file from temp repo path, parse content, assert on structure; will apply to CLAUDE.md verification
