# blocker.2: operation names do not conform to get/set/gen verb pattern

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/evolvable.domain.operations/rule.require.get-set-gen-verbs.md.min

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:65 (doesAwsProfileExist)
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:86 (getAwsSsoProfileConfig)
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:159 (listAwsSsoStartUrls)
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:219 (isAwsCliInstalled)
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:228 (initiateAwsSsoAuth)

Several operations in setupAwsSsoProfile.ts use verbs that are not allowed: 'does' (doesAwsProfileExist), 'list' (listAwsSsoStartUrls), 'initiate' (initiateAwsSsoAuth). Also 'getAwsSsoProfileConfig' lacks the required One/All cardinality suffix (should be getOneAwsSsoProfileConfig). All domain operations must use exactly one of get, set, or gen with appropriate cardinality. The verbs 'does', 'list', 'initiate' are synonyms that hide the operation's intent and break the closed verb system.

**snippet**:
```ts
// operation without get/set/gen
export const doesAwsProfileExist = async (input: { profileName: string }): Promise<boolean> => { ... };

export const listAwsSsoStartUrls = async (): Promise<Array<...>> => { ... };

export const isAwsCliInstalled = (): boolean => { ... };

export const initiateAwsSsoAuth = async (input: {...}): Promise<void> => { ... };

// get without cardinality
export const getAwsSsoProfileConfig = async (input: {...}): Promise<...> => { ... };
```
