# nitpick.1: success confirmation depends on regex, could fail silently on output change

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.friction-hazards.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts (initiateAwsSsoAuth)

in `initiateAwsSsoAuth`, the success message is gated by a regex match on aws cli output. if aws changes the output format in a future version, the regex may not match, and the user will not see the '✓ authenticated' confirmation, even though the operation succeeded. consider using the exit code as the primary success indicator and logging confirmation independently.

**snippet**:
```ts
const successMatch = outputBuffer.match(/Successfully logged into/);
if (successMatch) {
  console.log('   │  └─ ✓ authenticated');
}
```
