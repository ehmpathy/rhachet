# blocker.1: sso login timeout not handled in setupAwsSsoProfile function

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.require.behavior-intent-coverage.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:~393

The `setupAwsSsoProfile` function in setupAwsSsoProfile.ts invokes aws sso login via execSync and catches any failure to throw an `UnexpectedCodePathError`. This does not fulfill the wish and vision which require that sso login timeouts produce a loud `ConstraintError` with a clear message that the human did not respond, and instructs the robot to halt. The current implementation treats all sso login failures as unexpected code path errors, masking the timeout scenario and preventing the intended fail-fast behavior with proper error semantics.

**snippet**:
```ts
// trigger sso login to validate (suppress output — guide already showed auth info)
try {
  execSync(`aws sso login --profile "${input.profileName}"`, {
    stdio: 'pipe',
  });
} catch {
  throw new UnexpectedCodePathError('aws sso login failed', {
    profileName: input.profileName,
  });
}
```

# nitpick.1: incomplete unlock command hint in error message

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.require.behavior-intent-coverage.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:~440
- src/domain.operations/keyrack/adapters/vaults/aws.config/vaultAdapterAwsConfig.ts:~198

Both `initiateAwsSsoAuth` and `triggerSsoLogin` print error messages that include the hint `run: rhx keyrack unlock` without specifying the `--owner` and `--env` flags. The vision explicitly shows `rhx keyrack unlock --owner ehmpath --env test` as the fix command. Including owner and env would make the command immediately actionable for the robot. This is a minor deviation that reduces clarity for automated handling.

**snippet**:
```ts
// from initiateAwsSsoAuth:
            reject(
              new ConstraintError(
                'aws sso login timed out: human did not respond to browser sso prompt',
                {
                  code,
                  output: outputBuffer,
                  hint: 'run: rhx keyrack unlock',
                },
              ),
            );

// from triggerSsoLogin:
          reject(
            new ConstraintError(
              'aws sso login timed out: human did not respond to browser sso prompt',
              {
                profileName,
                exitCode: code,
                output: outputBuffer,
                hint: 'run: rhx keyrack unlock',
              },
            ),
          );
```
