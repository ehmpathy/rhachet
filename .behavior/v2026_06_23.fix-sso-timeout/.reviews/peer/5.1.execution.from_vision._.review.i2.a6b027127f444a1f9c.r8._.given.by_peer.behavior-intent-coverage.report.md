# blocker.1: Non-timeout SSO login failures classified with wrong error type in unlock flow

**rule**: bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.require.behavior-intent-coverage.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts (non-timeout reject)
- src/domain.operations/keyrack/adapters/vaults/aws.config/vaultAdapterAwsConfig.ts (non-timeout reject)

The vision explicitly requires network/connection failures to throw a MalfunctionError (exit code 1), not a ConstraintError (exit code 2) or UnexpectedCodePathError (code defect). In vaultAdapterAwsConfig.ts, triggerSsoLogin throws ConstraintError for all non-timeout failures, and in setupAwsSsoProfile.ts, initiateAwsSsoAuth throws UnexpectedCodePathError. Both should be MalfunctionError to fulfill the vision's error classification.

**snippet**:
```ts
// In setupAwsSsoProfile.ts, initiateAwsSsoAuth:
          } else {
            reject(
              new UnexpectedCodePathError('aws sso login failed', {
                code,
                output: outputBuffer,
              }),
            );
          }

// In vaultAdapterAwsConfig.ts, triggerSsoLogin:
        } else {
          reject(
            new ConstraintError('aws sso login failed', {
              profileName,
              exitCode: code,
              output: outputBuffer,
              hint: 'complete browser auth to continue',
            }),
          );
        }
```
