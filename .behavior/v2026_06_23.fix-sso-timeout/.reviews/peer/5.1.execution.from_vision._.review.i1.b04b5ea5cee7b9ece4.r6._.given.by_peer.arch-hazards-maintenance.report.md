# blocker.1: Silent catch on config file read masks errors

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:45-48

The try/catch block when reading ~/.aws/config silently swallows all errors. The comment indicates intent to handle 'file not found', but any other error (e.g., permissions) is also ignored. This is a failhide maintenance hazard: unexpected errors will be hidden, leading to silent defects. Must allowlist expected errors (e.g., check for 'ENOENT' code) and rethrow others.

**snippet**:
```ts

    configCurrent = await fs.readFile(configPath, 'utf-8');
  } catch {
    // file doesn't exist, start fresh
  }
```
