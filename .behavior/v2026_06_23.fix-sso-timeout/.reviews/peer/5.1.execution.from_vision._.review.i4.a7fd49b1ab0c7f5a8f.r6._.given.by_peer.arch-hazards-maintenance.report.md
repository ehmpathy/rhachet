# blocker.1: failhide: silent catch on fs.readFile hides non-ENOENT errors

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:52
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:184
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:245

try/catch around fs.readFile silently swallows all errors, not just file-not-found. If the file has permission issues or other read failures, the error is hidden and execution continues with an empty string. This causes hard-to-debug defects. Catch should check error.code === 'ENOENT' and rethrow unexpected errors.

**snippet**:
```ts
  let configCurrent = '';
  try {
    configCurrent = await fs.readFile(configPath, 'utf-8');
  } catch {
    // file doesn't exist, start fresh
  }
```
