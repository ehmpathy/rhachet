# blocker.1: Race condition on ~/.aws/config read-modify-write in setupAwsSsoProfile

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.behavior-hazards.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:55-75

setupAwsSsoProfile reads the entire config file, modifies it in memory, and writes it back. If two calls execute concurrently (e.g., parallel invocations for different profiles), the second write can overwrite changes from the first, causing lost data or duplicate sections. The operation lacks file locking or atomic transaction.

This is a classic race condition that will cause intermittent failures and data corruption in concurrent scenarios.

**snippet**:
```ts
// read current config or start fresh
let configCurrent = '';
try {
  configCurrent = await fs.readFile(configPath, 'utf-8');
} catch {
  // file doesn't exist, start fresh
}

// ... modify configCurrent ...

// append to config
const configNew = configCurrent.trimEnd() + '\n' + profileConfig;
await fs.writeFile(configPath, configNew, 'utf-8');
```

# blocker.2: Race condition on AWS CLI cache directory read-write in vaultAdapterAwsConfig.unlock

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.behavior-hazards.md

**locations**:
- src/domain.operations/keyrack/adapters/vaults/aws.config/vaultAdapterAwsConfig.ts:330-340

In unlock(), after a fresh SSO login, the code reads the CLI cache directory with readdirSync and deletes all files. If two unlock calls execute concurrently, they may read the same file list and attempt to delete the same files, or one may fail because the other already deleted them. Additionally, the deletion is non-atomic and could leave the cache in an inconsistent state if interrupted.

This race can cause failures in credential resolution and breaks the guarantee of safe concurrent usage.

**snippet**:
```ts
const cliCacheDir = join(homedir(), '.aws', 'cli', 'cache');
if (existsSync(cliCacheDir)) {
  const cacheFiles = readdirSync(cliCacheDir);
  for (const file of cacheFiles) {
    unlinkSync(join(cliCacheDir, file));
  }
}
```
