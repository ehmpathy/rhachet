# blocker.1: Mixed grain orchestration with inline compute and I/O

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.prefer.decomposable-architecture.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts:1-106

`setupAwsSsoProfile` acts as an orchestrator but contains raw I/O (execSync, fs ops) and inline compute (string manipulation, helper function defined inside). This violates the decomposable architecture rule which requires orchestrators to compose leaf operations, transformers for pure compute, and communicators for raw I/O. Extract the compute logic (e.g., `removeSection`) into a transformer, and the I/O operations (AWS CLI calls, config file writes) into communicators. This makes the orchestrator read as narrative.

**snippet**:
```ts
// From setupAwsSsoProfile
const profileHeader = `[profile ${input.profileName}]`;
const sessionHeader = `[sso-session ${input.profileName}]`;
const profileExists = configCurrent.includes(profileHeader);

if (profileExists && !input.overwrite) {
  throw new BadRequestError(...);
}

const removeSection = (config: string, header: string): string => {
  // inline helper with decode-friction
  const headerIndex = config.indexOf(header);
  if (headerIndex === -1) return config;
  const sectionStart = headerIndex;
  const nextSection = config.indexOf('\n[', headerIndex + 1);
  const sectionEnd = nextSection === -1 ? config.length : nextSection;
  return config.slice(0, sectionStart) + config.slice(sectionEnd);
};
```
