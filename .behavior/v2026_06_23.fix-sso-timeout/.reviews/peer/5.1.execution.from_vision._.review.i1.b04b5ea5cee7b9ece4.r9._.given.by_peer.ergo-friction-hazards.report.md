# blocker.1: unclear error when aws sso login fails (uses UnexpectedCodePathError with vague message)

**rule**: agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.friction-hazards.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts (approx lines 63-67)

when aws sso login fails inside setupAwsSsoProfile, the error is thrown as UnexpectedCodePathError with a generic message 'aws sso login failed'. this is a blocker because:
- the error type is for code defects, not user-facing failures
- the message does not tell the user what went wrong or how to fix it
- users will see a confusing error and won't know to check network, credentials, or browser flow
- the error is not snapped/tested, so it would ship as-is without review

fix: change to a user-friendly error (e.g., ConstraintError) with a hint like 'failed to complete AWS SSO login. ensure the browser flow was completed and the session is still valid.'

**snippet**:
```ts
    try {
      execSync(`aws sso login --profile "${input.profileName}"`, {
        stdio: 'pipe',
      });
    } catch {
      throw new UnexpectedCodePathError('aws sso login failed', {
        profileName: input.profileName,
      });
    }
```

# blocker.2: unclear error when sts get-caller-identity fails after sso login (uses UnexpectedCodePathError with vague message)

**rule**: agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.friction-hazards.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/aws.sso/setupAwsSsoProfile.ts (approx lines 72-78)

when sts validation fails after a successful sso login, the error is thrown as UnexpectedCodePathError with a generic message. this is a blocker because:
- the error type is for code defects, not user-facing failures
- the message does not indicate the cause or resolution
- users will be stuck with no guidance on what to do next

fix: change to a user-friendly error with a hint like 'SSO profile validated but AWS STS call failed. try again or check that the profile has necessary permissions.'

**snippet**:
```ts
    try {
      execSync(`aws sts get-caller-identity --profile "${input.profileName}"`, {
        stdio: 'pipe',
      });
    } catch {
      throw new UnexpectedCodePathError(
        'aws sts get-caller-identity failed after sso login',
        { profileName: input.profileName },
      );
    }
```

- the message 'aws sso login failed' lacks actionability
- the output buffer is included but not parsed to extract user-relevant info

```ts
          } else {
```
