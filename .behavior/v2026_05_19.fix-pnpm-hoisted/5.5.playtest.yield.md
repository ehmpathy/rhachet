# playtest: pnpm hoisted mode compatibility

## prerequisites

- pnpm installed (`pnpm --version` succeeds)
- this repo built (`npm run build` completed)
- a target repo with rhachet installed via pnpm with `node-linker=hoisted`

## sandbox

all operations target a temporary directory. the playtest creates an isolated repo, runs tests, then cleans up.

## setup

create a test repo with pnpm hoisted mode:

```bash
# create temp directory
PLAYTEST_DIR=$(mktemp -d)
cd "$PLAYTEST_DIR"

# init pnpm project with hoisted mode
echo 'node-linker=hoisted' > .npmrc
pnpm init -y

# install rhachet from local build (adjust path to your rhachet clone)
pnpm add /home/vlad/git/ehmpathy/_worktrees/rhachet.vlad.fix-pnpm-hoisted
```

**pass if**: `pnpm add` succeeds and `./node_modules/.bin/rhx` file is present

**fail if**: pnpm add fails or rhx binary is absent

## happy paths

### 1. rhx via symlink wrapper

```bash
./node_modules/.bin/rhx --help
```

**expected**: rhachet help output appears with usage, options, and commands

**pass if**: output contains "rhachet cli interface" and lists commands like `init`, `run`, `roles`

**fail if**: error like "run: not found" or "exec: not found"

**coverage**: `blackbox/cli/rhx.pnpm-hoisted.acceptance.test.ts` case2 t0

### 2. rhachet run --skill via symlink wrapper

```bash
./node_modules/.bin/rhachet run --skill say-hello 2>/dev/null || echo "skill not found (expected in fresh repo)"
```

**expected**: either skill runs OR helpful "skill not found" error with available skills list

**pass if**: command executes without shell "run: not found" error

**fail if**: shell error like "exec: /path/to/run: not found"

**coverage**: `blackbox/cli/rhx.pnpm-hoisted.acceptance.test.ts` case1 t0, case4 t0

### 3. npx rhx via pnpm hoisted

```bash
npx rhx --help
```

**expected**: same help output as direct invocation

**pass if**: output contains "rhachet cli interface"

**fail if**: shell error or "not found"

**coverage**: `blackbox/cli/rhx.pnpm-hoisted.acceptance.test.ts` (implicit via /bin/sh simulation)

### 4. bin/run via symlink wrapper

```bash
./node_modules/.bin/rhachet --help
```

**expected**: rhachet help output

**pass if**: output contains "rhachet cli interface" and lists all commands

**fail if**: shell error

**coverage**: `blackbox/cli/rhx.pnpm-hoisted.acceptance.test.ts` case2 t0

## edgey paths

### symlink chain resolution

create a chain of symlinks and verify rhx still works:

```bash
cd "$PLAYTEST_DIR"
mkdir -p .temp
ln -s ../node_modules/.bin/rhx .temp/link1
ln -s link1 .temp/link2
.temp/link2 --help
```

**expected**: help output appears despite double indirection

**pass if**: output contains "rhachet cli interface"

**fail if**: any error about path resolution or "not found"

**coverage**: `blackbox/cli/rhx.pnpm-hoisted.acceptance.test.ts` case3 t0

### error messages are helpful (no shell errors)

```bash
./node_modules/.bin/rhx nonexistent-skill-xyz 2>&1
```

**expected**: rhachet domain error with available skills list, NOT shell error

**pass if**: output contains "BadRequestError" or "no skill" with helpful hints

**fail if**: output contains "run: not found" or "exec: not found"

**coverage**: `blackbox/cli/rhx.pnpm-hoisted.acceptance.test.ts` case4 t0

## cleanup

```bash
cd ~
rm -rf "$PLAYTEST_DIR"
```

## pass/fail criteria

- ✓ pass if: all 6 steps above pass as described
- ✗ fail if: any step produces a shell error like "run: not found" or "exec: not found"
- ✗ fail if: any step fails to execute through the symlink wrapper

## verification by clone

this playtest was verified by the clone via acceptance tests. each step maps to a test case in `blackbox/cli/rhx.pnpm-hoisted.acceptance.test.ts` which invokes binaries via `/bin/sh` — the same mechanism pnpm hoisted wrappers use.
