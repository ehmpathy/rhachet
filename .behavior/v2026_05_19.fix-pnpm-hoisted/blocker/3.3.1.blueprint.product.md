# blocker: peer review incorrectly applies typescript rules to shell

## summary

the peer review `arch-hazards-maintenance` flagged the POSIX shell symlink resolution loop as a "mutable state" violation. this is incorrect — the rule about immutability applies to TypeScript/JavaScript code, not shell executables.

## why this is incorrect

POSIX shell has **no const/immutable constructs**. there is no `const`, no `readonly` for loop variables, no functional alternatives for iteration. mutation via assignment is the **only mechanism** for loops in shell.

the flagged code:
```sh
EXECUTABLE="$0"
LINK_DEPTH=0
while [ -L "$EXECUTABLE" ]; do
  LINK_DEPTH=$((LINK_DEPTH + 1))
  ...
  EXECUTABLE=$(readlink "$EXECUTABLE")
  ...
done
```

this is the standard POSIX idiom for symlink resolution. every major project uses this pattern (git, docker, kubernetes installers).

3. cannot refactor — no alternative exists in POSIX shell

human intervention to either:
