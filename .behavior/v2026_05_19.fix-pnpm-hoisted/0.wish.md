 # pnpm node-linker=hoisted breaks rhx

## date
2026-05-19

## symptom

```
./node_modules/.bin/rhx: 28: exec: /path/to/node_modules/.bin/run: not found
```

All `rhx` commands fail. Claude Code hooks also fail (catch-22: hooks use rhx to check permissions).

## root cause

`.npmrc` had `node-linker=hoisted` which changes how pnpm links binaries.

The `rhx` shell script expects a sibling `run` binary at `node_modules/.bin/run`. With `node-linker=hoisted`, pnpm doesn't create this symlink correctly.

## fix

Remove `node-linker=hoisted` from `.npmrc`, then reinstall:

```bash
# edit .npmrc to remove node-linker=hoisted
rm -rf node_modules && pnpm install
```

## context

- pnpm's default isolated mode (`node-linker=isolated`) works correctly with rhachet
- `node-linker=hoisted` flattens the dependency tree like npm but breaks certain bin symlinks
- the `rhx` shell wrapper delegates to `run` binary from rhachet package

## never do

Do not add `node-linker=hoisted` to `.npmrc` in repos that use rhachet/rhx.
