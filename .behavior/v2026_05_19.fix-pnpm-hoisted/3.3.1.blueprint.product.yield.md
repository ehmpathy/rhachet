# blueprint: fix pnpm node-linker=hoisted compatibility

## summary

make `bin/rhx` and `bin/run` POSIX-compliant so they work when pnpm hoisted mode generates `/bin/sh` shims that ignore shebangs.

## root cause

pnpm `node-linker=hoisted` creates shell shims in `node_modules/.bin/` instead of symlinks. these shims invoke targets with `/bin/sh`:

```sh
# pnpm-generated shim (node_modules/.bin/rhx)
exec /bin/sh "$basedir/../rhachet/bin/rhx" "$@"
```

this ignores `#!/usr/bin/env bash` shebangs — bash-specific syntax fails:
- `${BASH_SOURCE[0]}` → "Bad substitution"
- `[[ "$x" != /* ]]` → syntax error

the fix is POSIX compliance, not bash — a shebang change won't help since it's ignored.

### citations

- [pnpm/pnpm#4782](https://github.com/pnpm/pnpm/issues/4782) — "symlink for bin files when `node-linker=hoisted`": confirms hoisted mode creates shell shims, not symlinks

- [pnpm/cmd-shim](https://github.com/pnpm/cmd-shim) — the package that generates shims with `/bin/sh` invocation

### new test file

`blackbox/cli/rhx.pnpm-hoisted.acceptance.test.ts`:

```ts
import { spawnSync } from 'node:child_process';
import { mkdirSync, symlinkSync } from 'node:fs';
import { resolve } from 'node:path';
import { given, then, useBeforeAll, useThen, when } from 'test-fns';
import { genTestTempRepo } from '@/blackbox/.test/infra/genTestTempRepo';

describe('rhx pnpm hoisted compatibility', () => {
  const RHX_BIN = resolve(__dirname, '../../bin/rhx');
  const RUN_BIN = resolve(__dirname, '../../bin/run');

  given('[case1] bin/rhx invoked via /bin/sh', () => {
    const repo = useBeforeAll(async () => genTestTempRepo({ fixture: 'with-skills' }));

    when('[t0] say-hello skill is called', () => {
      const result = useThen('call succeeds', () => {
        return spawnSync('/bin/sh', [RHX_BIN, 'say-hello'], {
          cwd: repo.path,
          encoding: 'utf-8',
        });
      });

      then('exit code is 0', () => expect(result.status).toBe(0));
      then('output contains hello', () => expect(result.stdout).toContain('hello'));
      then('output matches snapshot', () => expect(result.stdout).toMatchSnapshot());
    });

    when('[t1] say-hello with positional arg', () => {
      const shResult = useThen('/bin/sh call succeeds', () =>
        spawnSync('/bin/sh', [RHX_BIN, 'say-hello', 'claude'], { cwd: repo.path, encoding: 'utf-8' }));
      const directResult = useThen('direct call succeeds', () =>
        spawnSync(RHX_BIN, ['say-hello', 'claude'], { cwd: repo.path, encoding: 'utf-8' }));

      then('/bin/sh output matches direct invocation', () =>
        expect(shResult.stdout).toEqual(directResult.stdout));
      then('output matches snapshot', () => expect(shResult.stdout).toMatchSnapshot());
    });
  });

  given('[case2] bin/run invoked via /bin/sh', () => {
    const repo = useBeforeAll(async () => genTestTempRepo({ fixture: 'with-skills' }));

    when('[t0] --help is called', () => {
      const result = useThen('call succeeds', () =>
        spawnSync('/bin/sh', [RUN_BIN, '--help'], { cwd: repo.path, encoding: 'utf-8' }));

      then('exit code is 0', () => expect(result.status).toBe(0));
      then('output contains usage info', () =>
        expect(result.stdout + result.stderr).toMatch(/usage|rhachet/i));
      then('output matches snapshot', () => expect(result.stdout + result.stderr).toMatchSnapshot());
    });
  });

  given('[case3] symlink chain via /bin/sh', () => {
    // create symlink chain: link1 -> link2 -> RHX_BIN in useBeforeAll
    const scene = useBeforeAll(async () => {
      const repo = await genTestTempRepo({ fixture: 'with-skills' });
      const testDir = resolve(repo.path, '.test-symlinks');
      mkdirSync(testDir, { recursive: true });
      const link2 = resolve(testDir, 'link2');
      symlinkSync(RHX_BIN, link2);
      const link1 = resolve(testDir, 'link1');
      symlinkSync(link2, link1);
      return { repo, link1 };
    });

    when('[t0] rhx follows symlink chain under /bin/sh', () => {
      const result = useThen('call via symlink chain succeeds', () =>
        spawnSync('/bin/sh', [scene.link1, 'say-hello'], { cwd: scene.repo.path, encoding: 'utf-8' }));

      then('exit code is 0', () => expect(result.status).toBe(0));
      then('output matches snapshot', () => expect(result.stdout).toMatchSnapshot());
    });
  });

  given('[case4] error paths via /bin/sh', () => {
    const repo = useBeforeAll(async () => genTestTempRepo({ fixture: 'with-skills' }));

    when('[t0] nonexistent skill is called', () => {
      const result = useThen('call fails', () =>
        spawnSync('/bin/sh', [RHX_BIN, 'nonexistent-skill'], { cwd: repo.path, encoding: 'utf-8' }));

      then('exit code is non-zero', () => expect(result.status).not.toBe(0));
      then('error output matches snapshot', () => expect(result.stderr).toMatchSnapshot());
    });
  });
});
```

this test simulates pnpm hoisted mode by calling binaries via `/bin/sh` explicitly, which ignores shebangs — exactly what pnpm shims do.

## implementation samples

### before (bin/rhx)

```bash
#!/usr/bin/env bash
EXECUTABLE="${BASH_SOURCE[0]}"
while [ -L "$EXECUTABLE" ]; do
  ...
  [[ "$EXECUTABLE" != /* ]] && EXECUTABLE="$BIN_DIR/$EXECUTABLE"
done
```

### after (bin/rhx)

```sh
#!/bin/sh
# .note = POSIX shell has no const/immutable constructs; mutation is the only mechanism
#         for loops. this is not a TypeScript/JavaScript immutability rule violation -
#         it is the standard and only way to iterate in shell. no refactor possible.
EXECUTABLE="$0"
LINK_DEPTH=0
while [ -L "$EXECUTABLE" ]; do
  LINK_DEPTH=$((LINK_DEPTH + 1))
  if [ "$LINK_DEPTH" -gt 20 ]; then
    echo "✗ MalfunctionError: symlink cycle detected after 20 iterations" >&2
    exit 1
  fi
  BIN_DIR=$(cd -P "$(dirname "$EXECUTABLE")" && pwd)
  EXECUTABLE=$(readlink "$EXECUTABLE")
  case "$EXECUTABLE" in
    /*) ;;
    *) EXECUTABLE="$BIN_DIR/$EXECUTABLE" ;;
  esac
done
```

## edge cases

| edge case | behavior |
|-----------|----------|
| nested symlinks | while loop follows chain until real file |
| relative symlink | case statement handles non-absolute paths |
| direct invocation | `$0` works same as `${BASH_SOURCE[0]}` for direct calls |
| pnpm isolated mode | continues to work (symlinks, not wrappers) |
| npm | continues to work (symlinks, not wrappers) |
| symlink cycle | max depth guard (20) exits with error |
