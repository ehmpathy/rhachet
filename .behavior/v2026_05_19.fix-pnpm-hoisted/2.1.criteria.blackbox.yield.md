# blackbox criteria: fix pnpm node-linker=hoisted compatibility

## usecase.1 = invoke rhx via symlink (pnpm hoisted mode)

```
given rhx binary is accessed via symlink
  # simulates: node_modules/.bin/rhx → ../../actual/path/bin/rhx

  when user invokes rhx with --help
    then help output is shown
      sothat user knows rhx is functional

  when user invokes rhx with a valid skill
    then skill executes successfully
      sothat user can run skills via pnpm hoisted mode

  when user invokes rhx with an invalid skill
    then error message is shown
      sothat user gets clear feedback (not "run: not found")
```


## usecase.5 = error clarity

```

    then error message does NOT say "run: not found"
      sothat users get actionable errors, not symlink-related confusion
```
