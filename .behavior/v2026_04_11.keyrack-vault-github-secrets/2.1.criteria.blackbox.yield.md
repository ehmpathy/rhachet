# blackbox criteria: keyrack vault github.secrets

## usecase.1 = set key to github.secrets

```
given(gh cli is authenticated)
given(user has write access to repo secrets)
  when(user runs `keyrack set --key X --vault github.secrets --mech EPHEMERAL_VIA_GITHUB_APP`)
    then(guided setup prompts for org selection)
    then(guided setup prompts for app selection)
    then(guided setup prompts for private key path)
    then(secret is pushed to github via gh api)
    then(stdout shows "pushed to github.secrets (no roundtrip — write-only vault)")
    then(summary shows slug, mech, and vault)
    then(host manifest records the key with vault=github.secrets)
      sothat(status can report the key exists)

given(gh cli is authenticated)
given(user has write access to repo secrets)
  when(user runs `keyrack set --key X --vault github.secrets` without --mech)
    then(prompts for secret value via PERMANENT_VIA_REPLICA)
    then(secret is pushed to github via gh api)
    then(host manifest records the key)
```

## usecase.2 = delete key from github.secrets

```
given(key exists in github.secrets)
given(key exists in host manifest with vault=github.secrets)
  when(user runs `keyrack del --key X`)
    then(secret is removed from github via gh api DELETE)
    then(key is removed from host manifest)
      sothat(status no longer reports the key)

given(key does not exist in github.secrets)
  when(user runs `keyrack del --key X --vault github.secrets`)
    then(gh api returns success (idempotent delete))
    then(key is removed from host manifest if present)
```

## usecase.3 = get key from github.secrets (failfast)

```
given(key exists in host manifest with vault=github.secrets)
  when(user runs `keyrack get --key X`)
    then(failfast with "github secrets cannot be retrieved via api")
    then(status shows "locked")
    then(fix shows null)
      sothat(user knows unlock is not possible)
```

## usecase.4 = unlock key from github.secrets

```
given(key exists in host manifest with vault=github.secrets)
  when(user runs `keyrack unlock --key X` specifically)
    then(failfast with "github.secrets cannot be unlocked")
      sothat(user knows this vault is write-only)

given(key exists in host manifest with vault=github.secrets)
given(other keys exist in other vaults)
  when(user runs `keyrack unlock --for repo`)
    then(github.secrets keys are skipped silently)
    then(other vault keys are unlocked normally)
      sothat(bulk unlock workflows are not broken)
```

## usecase.5 = status of github.secrets key

```
given(key exists in host manifest with vault=github.secrets)
  when(user runs `keyrack status --key X`)
    then(status shows "locked")
    then(vault shows "github.secrets")
      sothat(user knows the key was set but cannot be retrieved)

given(key does not exist in host manifest)
  when(user runs `keyrack status --key X`)
    then(status shows "absent")
```

## usecase.6 = upsert semantics

```
given(key already exists in github.secrets)
  when(user runs `keyrack set --key X --vault github.secrets`)
    then(gh api PUT overwrites the secret)
    then(host manifest is updated)
      sothat(set is idempotent)
```

## usecase.7 = error cases

```
given(gh cli is not authenticated)
  when(user runs `keyrack set --key X --vault github.secrets`)
    then(failfast with "gh auth required")

given(repo does not exist)
  when(user runs `keyrack set --key X --vault github.secrets`)
    then(failfast with "repo not found")

given(user lacks write access to repo secrets)
  when(user runs `keyrack set --key X --vault github.secrets`)
    then(failfast with "permission denied")
```

## usecase.8 = vault adapter declares get: null

```
given(vault adapter for github.secrets)
  then(get method is null)
  then(set method is defined)
  then(del method is defined)
    sothat(write-only is explicit at interface level)
    sothat(failfast happens at dispatch, not deep in call stack)
```
