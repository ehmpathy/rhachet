### step 4.2: acceptance test file

- [ ] create `blackbox/cli/keyrack.vault.githubSecrets.acceptance.test.ts`

**test cases:**
- `[case1]` set key via EPHEMERAL_VIA_GITHUB_APP → snapshot stdout
- `[case2]` set key via PERMANENT_VIA_REPLICA → snapshot stdout
- `[case3]` get key → failfast snapshot
- `[case4]` unlock --key X → failfast snapshot
- `[case5]` unlock --for repo with github.secrets keys → skip silently
- `[case6]` status shows locked → snapshot
- `[case7]` del key → snapshot stdout
- `[case8]` error: gh auth required → snapshot
- `[case9]` error: repo not found → snapshot
- `[case10]` error: permission denied → snapshot

**acceptance criteria (per 2.1.criteria.blackbox.yield.md):**
```
given(gh cli is authenticated)
given(user has write access to repo secrets)
  when(user runs `keyrack set --key X --vault github.secrets --mech EPHEMERAL_VIA_GITHUB_APP`)
    then(guided setup prompts for org selection)
    then(guided setup prompts for app selection)
    then(guided setup prompts for private key path)
    then(secret is pushed to github via gh api)
    then(stdout shows "pushed to github.secrets (no roundtrip — write-only vault)")
    then(summary shows slug, mech, and vault)
```

**verify:**
```bash
npm run test:acceptance -- keyrack.vault.githubSecrets
```
