# blueprint: keyrack vault github.secrets

## summary

add `github.secrets` vault adapter that pushes keys to github actions secrets via `gh secret set`. this is a write-only vault: `get` is null (github secrets api does not support retrieval). the vault supports both `EPHEMERAL_VIA_GITHUB_APP` (guided setup) and `PERMANENT_VIA_REPLICA` (plain text) mechanisms.

key behaviors:
- `set` → push secret to github via `gh secret set`
- `get` → null (write-only vault, failfast at dispatch)
- `del` → remove secret via `gh secret delete`
- `unlock --key X` → failfast with "github.secrets cannot be unlocked"
- `unlock --for repo` → skip github.secrets keys silently
- `status` → shows `locked` with `fix: null`
