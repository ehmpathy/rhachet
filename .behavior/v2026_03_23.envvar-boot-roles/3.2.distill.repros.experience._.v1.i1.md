## summary

this document defines how user experiences will be reproduced in tests for envvar-boot-roles.

## experience reproductions

| experience | entry point | user actions | expected outcome | test type |
|------------|-------------|--------------|------------------|-----------|
| focused mechanic clone | CLI | `rhx enroll claude --roles mechanic` | boots with mechanic only | integration |
| subtract driver role | CLI | `rhx enroll claude --roles -driver` | defaults minus driver | integration |
| append architect role | CLI | `rhx enroll claude --roles +architect` | defaults plus architect | integration |
| typo error with hint | CLI | `rhx enroll claude --roles mechnic` | error with "did you mean mechanic?" | integration |
| passthrough resume | CLI | `rhx enroll claude --roles mechanic --resume` | brain receives --resume | integration |

## journey test sketches

### journey 1: replace default roles

```
given('[case1] repo with default roles [mechanic, driver, ergonomist]')
  when('[t0] before enroll')
    then('.claude/settings.json has hooks for all default roles')
    then('input/output matches snapshot')
  when('[t1] enroll with --roles mechanic')
    then('.claude/settings.local.json has hooks for mechanic only')
    then('spawn args include no --roles (consumed by wrapper)')
    then('input/output matches snapshot')
```

#### step table

| step | action | user sees |
|------|--------|-----------|
| t0 | before any changes | `.claude/settings.json` with all role hooks |
| t1 | `rhx enroll claude --roles mechanic --mode plan` | json output with mechanic-only config |

#### t1 success case (snapshot target)

```
$ rhx enroll claude --roles mechanic --mode plan

{
  "config": {
    "hooks": {
      "SessionStart": [
        { "matcher": "", "hooks": ["npx rhachet roles boot --repo ehmpathy --role mechanic"] }
      ]
    }
  },
  "configPath": ".claude/settings.local.json",
  "roles": ["mechanic"],
  "spawnArgs": []
}
```

## reproduction feasibility

all experiences can be reproduced with extant infra:

| experience | utilities available | setup required |
|------------|---------------------|----------------|
| replace roles | invokeRhachetCli, genTestTempDir | .agent/ with mock roles |
| subtract role | invokeRhachetCli, genTestTempDir | .agent/ with mock roles |
| typo error | invokeRhachetCli, getError | .agent/ with mock roles |
| passthrough | invokeRhachetCli | .agent/ with mock roles |

### concrete test sketch

```typescript
given('[case1] repo with default roles [mechanic, driver]', () => {
  const testDir = genTestTempDir({ base: __dirname, name: 'enroll-replace' });
  beforeAll(() => testDir.setup());
  afterAll(() => testDir.teardown());

  beforeAll(async () => {
    // setup .agent/ with mock roles
    await setupMockAgentDir(testDir.path, ['mechanic', 'driver']);
  });

  when('[t1] enroll with --roles mechanic --mode plan', () => {
    const result = useThen('it outputs plan', () =>
      invokeRhachetCli({
        args: ['enroll', 'claude', '--roles', 'mechanic', '--mode', 'plan'],
        cwd: testDir.path,
      }),
    );

    then('roles array contains only mechanic', () => {
      const output = JSON.parse(result.stdout);
      expect(output.roles).toEqual(['mechanic']);
    });

    then('output matches snapshot', () => {
      expect(result.stdout).toMatchSnapshot();
    });
  });
});
```
