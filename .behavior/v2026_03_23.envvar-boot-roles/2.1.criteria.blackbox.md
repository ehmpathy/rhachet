# blackbox criteria: envvar-boot-roles

## usecase.1 = replace default roles

```
given a repo with default roles [mechanic, driver, ergonomist]
  when human runs `rhx enroll claude --roles mechanic`
    then clone boots with ONLY mechanic briefs and skills
      sothat human can focus on code work without driver noise
    then default roles are not loaded
      sothat context window is not cluttered
```

## usecase.2 = append to default roles

```
given a repo with default roles [mechanic, driver]
  when human runs `rhx enroll claude --roles +architect`
    then clone boots with [mechanic, driver, architect]
      sothat human gets architect perspective in addition to defaults
    then architect briefs and skills are available
```

## usecase.3 = subtract from default roles

```
given a repo with default roles [mechanic, driver, ergonomist]
  when human runs `rhx enroll claude --roles -driver`
    then clone boots with [mechanic, ergonomist]
      sothat human avoids driver briefs when not needed
    then driver briefs and skills are not loaded
```

## usecase.4 = mixed append and subtract

```
given a repo with default roles [mechanic, driver, ergonomist]
  when human runs `rhx enroll claude --roles -driver,+architect`
    then clone boots with [mechanic, ergonomist, architect]
      sothat human can customize the role mix precisely
```

## usecase.5 = explicit multi-role

```
given a repo with default roles [mechanic, driver, ergonomist]
  when human runs `rhx enroll claude --roles mechanic,architect`
    then clone boots with ONLY [mechanic, architect]
      sothat human can specify exact role combination
    then no default roles outside the spec are loaded
```

## usecase.6 = resume with roles

```
given a prior session exists
  when human runs `rhx enroll claude --roles mechanic --resume`
    then clone resumes prior session
    then clone boots with mechanic role only
      sothat human can continue work with different role focus
```

## usecase.7 = error: no roles flag

```
given a repo with default roles [mechanic, driver, ergonomist]
  when human runs `rhx enroll claude` (no --roles flag)
    then error is surfaced: "--roles is required"
      sothat human must explicitly declare which roles to enroll
    then clone does not boot
```

## usecase.8 = error: typo in role name

```
given a repo with roles [mechanic, driver, ergonomist]
  when human runs `rhx enroll claude --roles mechnic`
    then error is surfaced: "role 'mechnic' not found, did you mean 'mechanic'?"
      sothat human catches typos immediately
    then clone does not boot
```

## usecase.9 = error: empty roles flag

```
given a repo with default roles
  when human runs `rhx enroll claude --roles ""`
    then error is surfaced: "--roles is empty, specify roles to enroll"
      sothat human is guided to correct usage
    then clone does not boot
```

## usecase.10 = error: conflict in ops

```
given any repo
  when human runs `rhx enroll claude --roles +mechanic,-mechanic`
    then error is surfaced: "cannot both add and remove 'mechanic'"
      sothat human is prevented from contradictory specs
    then clone does not boot
```

## usecase.11 = error: no .agent/ directory

```
given a repo without .agent/ directory
  when human runs `rhx enroll claude --roles mechanic`
    then error is surfaced: "no .agent/ found, run rhx init first"
      sothat human knows to initialize rhachet
    then clone does not boot
```

## usecase.12 = idempotent subtract of absent role

```
given a repo with default roles [mechanic, driver]
  when human runs `rhx enroll claude --roles -ergonomist`
    then clone boots with [mechanic, driver]
      sothat subtract of absent role is a no-op
    then no error is raised
```

## usecase.13 = idempotent append of present role

```
given a repo with default roles [mechanic, driver]
  when human runs `rhx enroll claude --roles +mechanic`
    then clone boots with [mechanic, driver]
      sothat append of already-present role is a no-op
    then no error is raised
```

## usecase.14 = passthrough of other args

```
given any repo with roles
  when human runs `rhx enroll claude --roles mechanic --dangerously-skip-permissions`
    then --dangerously-skip-permissions is passed to claude
      sothat human can use any claude flag alongside --roles
    then clone boots with mechanic role
```

## usecase.15 = rejects default configs

```
given a repo with .claude/settings.json with hooks
  and a global ~/.claude/settings.json with hooks
  when human runs `rhx enroll claude --roles mechanic`
    then clone boots with ONLY mechanic hooks from the generated config
    then global ~/.claude/settings.json hooks are NOT loaded
    then repo .claude/settings.json hooks are NOT loaded (except those filtered to mechanic)
      sothat the enrolled brain has exactly the roles specified, no inheritance
```

## usecase.16 = generates unique config file

```
given a repo with roles
  when human runs `rhx enroll claude --roles mechanic`
    then a unique config file is generated (e.g., .claude/settings.enroll.$hash.json)
    then claude is spawned with --bare --settings <unique-config-path>
      sothat each enrollment session has isolated config
```
