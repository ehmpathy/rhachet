# nitpick.1: BadRequestError uses exit code 1 instead of 2

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.errors/rule.require.exit-code-semantics.md

**locations**:
- src/contract/cli/invoke.ts:71

BadRequestError represents a constraint error where user action is required (e.g., invalid arguments or missing roles), which should exit with code 2 according to semantic exit codes. However, in the catch block, it manually exits with 1, overriding the intended constraint code.
Change process.exit(1) to process.exit(2) to properly signal a constraint error that requires user intervention.
