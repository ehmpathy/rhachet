# playtest: envvar-boot-roles

## prerequisites

- rhachet installed (`npx rhachet` or `rhx` alias available)
- a repo with `.agent/` directory configured via `rhx init`
- at least 3 linked roles (mechanic, driver, ergonomist)
- terminal access

## sandbox

- enroll commands write `.claude/settings.local.json`
- happy path commands will fail at spawn (nested claude blocked) â€” this is expected
- verify behavior via the generated config file before spawn fails
- cleanup: `rm .claude/settings.local.json` after playtest

## verification method

for happy paths:
1. run the enroll command
2. command writes `.claude/settings.local.json`
3. command attempts to spawn claude (will error: "cannot be launched inside another Claude Code session")
4. inspect `.claude/settings.local.json` to verify hooks are correct

for error paths:
1. run the enroll command
2. command errors before config is written
3. verify error message is helpful

## happy paths

### 1. replace mode â€” single role

**action:**
```bash
npx rhachet enroll claude --roles mechanic
```

**then inspect:**
```bash
cat .claude/settings.local.json | jq '.hooks.SessionStart'
```

**expected outcome:**
- `.claude/settings.local.json` created before spawn fails
- hooks.SessionStart contains entries with `role=mechanic` pattern only
- no `role=driver` or `role=ergonomist` patterns in matchers

**pass:** exactly mechanic role hooks present
**fail:** hooks contain other roles
