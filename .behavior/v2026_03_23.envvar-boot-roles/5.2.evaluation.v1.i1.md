## summary (as implemented)

implemented `rhx enroll <brain> --roles <spec>` command that:
1. parses roles spec (`mechanic`, `-driver`, `+architect`) — **`--roles` is required**
2. computes final roles from spec and linked roles
3. generates **unique** brain config file (e.g., `.claude/settings.enroll.$hash.json`)
4. spawns brain with `--bare --settings <custom-path>` — **rejects all defaults**

matches blueprint hard requirements exactly.

## divergence analysis

### blueprint vs actual

| aspect | blueprint | actual | divergence |
|--------|-----------|--------|------------|
| domain objects | BrainSlug, RoleSlug, BrainCliEnrollmentSpec, BrainCliEnrollmentOperation, BrainCliEnrollmentManifest | same | none |
| domain objects | BrainCliConfigArtifact (Artifact<typeof GitFile>) | exists but unused | intentional — configPath string sufficient |
| operations | parseBrainCliEnrollmentSpec | same | none |
| operations | computeBrainCliEnrollment | same | none |
| operations | genBrainCliConfigArtifact | same (signature refined) | minor |
| operations | enrollBrainCli | same | none |
| contract | invokeEnroll | same | none |
| contract | invoke.ts modification | same | none |
| test files | invokeEnroll.play.integration.test.ts | not created | intentional — consolidated |

### divergences addressed

1. **BrainCliConfigArtifact unused** — the blueprint proposed `type BrainCliConfigArtifact = Artifact<typeof GitFile>` and the file was created, but `genBrainCliConfigArtifact` returns `{ configPath: string }` directly. the type alias exists but is not used in the implementation — could be deleted, but left for potential future use.

2. **journey tests consolidated** — the blueprint proposed separate `invokeEnroll.play.integration.test.ts` for journey tests. these were consolidated into `invokeEnroll.integration.test.ts` since the tests are comprehensive and the separation would be artificial.

3. **genBrainCliConfigArtifact signature** — refined from `{ brain, enrollment, agentDir }` to `{ enrollment, repoPath }`. brain is already in `enrollment.brain`. `agentDir` renamed to `repoPath` for clarity (settings file lives at repo root).

### key decisions retained from blueprint

1. **`--bare` rejects defaults** — `claude --bare` skips default config load
2. **`--settings <path>` loads custom config only** — enrolled brain uses ONLY the generated config
3. **retain repo permissions** — permissions from `.claude/settings.json` copied to generated config
4. **unique config file** — `.claude/settings.enroll.$hash.json` ensures no collision
5. **`--roles` is required** — no fallback to defaults; explicit roles required
6. **passthrough args** — all unknown args pass to brain after `--settings <path>`
7. **spawn with inherited stdio** — brain inherits stdin/stdout/stderr, forward exit code
8. **rolesDefault = all linked roles** — when compute delta ops (+/-), defaults are all roles in .agent/
