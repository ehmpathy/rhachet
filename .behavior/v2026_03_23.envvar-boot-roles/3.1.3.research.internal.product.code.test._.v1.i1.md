# research: internal test codepaths

## summary

this document analyzes the test codepath patterns relevant to envvar-boot-roles. patterns are marked with `[REUSE]`, `[EXTEND]`, or `[REPLACE]` to indicate the implementation strategy.

---

## relevant test patterns

### 1. given/when/then from test-fns [REUSE]

**file**: all integration tests

**what**: bdd-style test structure via `test-fns` library

```typescript
import { given, when, then, getError } from 'test-fns';

given('[case1] role linked with briefs', () => {
  when('[t0] getRoleBriefs with role name', () => {
    then('returns artifacts for briefs that match', async () => {
      const result = await getRoleBriefs({ by: { role, briefs } });
      expect(result).toHaveLength(1);
    });
  });
});
```

**relation to wish**: use this pattern for all `rhx enroll` tests

**strategy**: [REUSE] — same pattern for roles spec parser, role computer, config generator, and full command tests

### 2. temp directory pattern [REUSE/EXTEND]

**file**: `src/.test/infra/genTestTempDir.ts`

**what**: creates managed temp test directories with cwd switch

```typescript
export const genTestTempDir = (input: {
  base: string;
  name: string;
}): {
  path: string;
  setup: () => void;
  teardown: () => void;
  rm: (relativePath: string) => void;
};
```

key capabilities:
- creates `.temp/{name}` under test file location
- cleans up prior runs
- switches cwd for test isolation
- restores original cwd in teardown

**relation to wish**: we need temp directories to test:
- `.agent/` discovery
- dynamic config file generation
- brain spawn behavior

**strategy**:
- [REUSE] for unit and integration tests
- [EXTEND] may need a variant that also creates `.claude/` directory
