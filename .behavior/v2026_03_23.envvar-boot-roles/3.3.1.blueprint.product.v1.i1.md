# blueprint: product

## summary

implement `rhx enroll <brain> --roles <spec>` command that:
1. parses roles spec (`mechanic`, `-driver`, `+architect`) — **`--roles` is required**
2. computes final roles from spec and defaults
3. generates **unique** brain config file (e.g., `.claude/settings.enroll.$hash.json`)
4. spawns brain with `--setting-sources local --settings <custom-path>` — **skips all default settings, auth loads from credentials file**

---

## hard requirement: custom config, skip all default settings

the enroll command MUST:
1. generate a CUSTOM config file with unique name (`.claude/settings.enroll.$hash.json`)
2. SKIP user settings (`~/.claude/settings.json`) via `--setting-sources local`
3. SKIP project settings (`.claude/settings.json`) via `--setting-sources local`
4. use ONLY the custom config via `--settings <path>`
5. auth credentials load independently from `~/.claude/.credentials.json` (not affected by --setting-sources)

this is non-negotiable. the enrolled brain boots with ONLY the specified roles' hooks — no merge, no inheritance, no defaults.

### spawn mechanism

```bash
claude --setting-sources local --settings .claude/settings.enroll.$hash.json [passthrough args]
```

- `--setting-sources local` — skips user (~/.claude/settings.json) and project (.claude/settings.json) settings
- `--settings <path>` — loads settings from the specified file only
- auth credentials load independently from `~/.claude/.credentials.json`

## key decisions

1. **`--setting-sources local` skips user and project settings** — auth loads independently from `~/.claude/.credentials.json`, not affected by --setting-sources
2. **`--settings <path>` loads custom config only** — the enrolled brain uses ONLY the generated config file for hooks
3. **retain repo permissions** — permissions from `.claude/settings.json` are copied to the generated config (hooks are filtered, permissions are retained)
4. **unique config file** — `.claude/settings.enroll.$hash.json` ensures no collision with extant configs
5. **`--roles` is required** — no fallback to defaults; explicit roles required for enrollment
6. **passthrough args** — all unknown args pass to brain after `--settings <path>`
7. **spawn with inherited stdio** — brain inherits stdin/stdout/stderr, forward exit code
8. **rolesDefault = all linked roles** — when computing delta ops (+/-), defaults are all roles in .agent/
