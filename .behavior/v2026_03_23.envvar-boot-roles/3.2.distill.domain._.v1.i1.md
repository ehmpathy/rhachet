# distill: domain objects and operations

## summary

this document distills the domain objects and operations needed to fulfill envvar-boot-roles.

---

## usecases to support

from 2.1.criteria.blackbox.md:
- replace default roles with explicit list
- append to default roles via `+role`
- subtract from default roles via `-role`
- mixed append/subtract operations
- passthrough args to brain cli
- error on typo with suggestion
- error on empty or conflict

---

## domain objects

### type aliases

```typescript
/**
 * slug for a brain (e.g., "claude")
 */
type BrainSlug = string;

/**
 * slug for a role (e.g., "mechanic", "driver")
 */
type RoleSlug = string;
```

### BrainCliEnrollmentSpec (literal)

the parsed representation of the `--roles` flag value for brain CLI enrollment.

```typescript
interface BrainCliEnrollmentSpec {
  /**
   * mode of application:
   * - "replace" = replace defaults entirely
   * - "delta" = modify defaults via ops
   */
  mode: 'replace' | 'delta';

  /**
   * ordered list of operations to apply
   */
  ops: BrainCliEnrollmentOperation[];
}

class BrainCliEnrollmentSpec extends DomainLiteral<BrainCliEnrollmentSpec> implements BrainCliEnrollmentSpec {}
```

### BrainCliEnrollmentOperation (literal)

a single role operation within an enrollment spec.

```typescript
interface BrainCliEnrollmentOperation {
  /**
   * action to perform:
   * - "add" = include this role
   * - "remove" = exclude this role
   */
  action: 'add' | 'remove';

  /**
   * the role slug (e.g., "mechanic", "driver")
   */
  role: RoleSlug;
}

class BrainCliEnrollmentOperation extends DomainLiteral<BrainCliEnrollmentOperation> implements BrainCliEnrollmentOperation {}
```

### BrainCliEnrollmentManifest (literal)

the final manifest of what to enroll.

```typescript
interface BrainCliEnrollmentManifest {
  /**
   * the brain to enroll (e.g., "claude")
   */
  brain: BrainSlug;

  /**
   * ordered list of role slugs to enroll
   */
  roles: RoleSlug[];
}

class BrainCliEnrollmentManifest extends DomainLiteral<BrainCliEnrollmentManifest> implements BrainCliEnrollmentManifest {}
```

### BrainCliConfigArtifact (artifact)

the dynamically generated brain config file for an enrollment session.

```typescript
import { Artifact, GitFile } from 'rhachet-artifact-git';

/**
 * the generated config file (e.g., .claude/settings.local.json)
 *
 * wraps a GitFile artifact with brain-specific metadata
 */
type BrainCliConfigArtifact = Artifact<typeof GitFile>;
```

**notes**:
- leverages `rhachet-artifact-git` for file operations
- GitFile provides: path, read, write, diff, commit state
- artifact pattern enables: undo, replay, composition

## domain operations

### parseBrainCliEnrollmentSpec

parses the `--roles` flag value into a structured BrainCliEnrollmentSpec.

```typescript
/**
 * .what = parses roles spec string into structured form
 * .why = enables validation and manipulation of enrollment ops
 */
export const parseBrainCliEnrollmentSpec = (input: {
  spec: string;
}): BrainCliEnrollmentSpec => { ... };
```

**examples**:
| input | output |
|-------|--------|
| `"mechanic"` | `{ mode: "replace", ops: [{ action: "add", role: "mechanic" }] }` |
| `"+architect"` | `{ mode: "delta", ops: [{ action: "add", role: "architect" }] }` |
| `"-driver"` | `{ mode: "delta", ops: [{ action: "remove", role: "driver" }] }` |
| `"-driver,+architect"` | `{ mode: "delta", ops: [{ action: "remove", role: "driver" }, { action: "add", role: "architect" }] }` |

**errors**:
- empty spec → BadRequestError
- conflict (+foo,-foo) → BadRequestError

## composition for usecases

### usecase.1-5, 7, 12-13 (happy path)

```
--roles flag → parseBrainCliEnrollmentSpec → computeBrainCliEnrollment → genBrainCliConfigArtifact → enrollBrainCli
```

### usecase.8 (typo error)

```
--roles flag → parseBrainCliEnrollmentSpec → computeBrainCliEnrollment → [BadRequestError with suggestion]
```

### usecase.9 (empty error)

```
--roles "" → parseBrainCliEnrollmentSpec → [BadRequestError: empty]
```

### usecase.10 (conflict error)

```
--roles +foo,-foo → parseBrainCliEnrollmentSpec → [BadRequestError: conflict]
```

### usecase.11 (no .agent/ error)

```
--roles x → [check .agent/] → [BadRequestError: no .agent/]
```

### usecase.14 (passthrough)

```
--roles x --resume → enrollBrainCli({ args: ["--resume"] })
```
