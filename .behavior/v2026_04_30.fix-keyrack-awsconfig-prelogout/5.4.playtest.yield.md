# playtest: keyrack aws.config pre-logout

## .what

manual verification of the three behaviors from the wish:

1. **pre-logout before auth** — clears SSO cache for selected domain before browser auth
2. **org label display** — shows org name next to each SSO start URL
3. **identity check on unlock** — detects wrong SSO user and auto-corrects

## .prerequisites

- AWS SSO configured with at least one profile in `~/.aws/config`
- active SSO session (or willingness to auth via browser)

---

## playtest 1: org label display

**goal**: verify org name appears in domain picker

**steps**:

```bash
# start the guided setup
rhx keyrack set --vault aws.config
```

**expected output**:

```
├─ which sso domain?
│  ├─ options
│  │  └─ 1. https://acme.awsapps.com/start (us-east-1) — acme
│  └─ choice
```

**observe**: the `— acme` suffix shows the org extracted from the URL subdomain

**result**: [ ] pass / [ ] fail

---

## playtest 2: pre-logout before auth

**goal**: verify cache is cleared for selected domain before browser auth

**steps**:

```bash
# 1. ensure you have an active SSO session
aws sso login --profile some-profile

# 2. verify cache files exist
ls ~/.aws/sso/cache/

# 3. start guided setup
rhx keyrack set --vault aws.config

# 4. select a domain, observe pre-logout step
```

**expected output**:

```
├─ which sso domain?
│  └─ choice
│     └─ 1 ✓
│     └─ as https://acme.awsapps.com/start
│
├─ which sso prior?
│  └─ ✓ clear
│
├─ which sso login?
```

**observe**:
- `├─ which sso prior? │ └─ ✓ clear` appears before browser auth
- cache files for that domain should be deleted

**result**: [ ] pass / [ ] fail

---

## playtest 3: targeted deletion (multi-org)

**goal**: verify only selected domain's cache is cleared, not other domains

**steps**:

```bash
# 1. login to two different SSO domains
aws sso login --profile profile-from-org-a
aws sso login --profile profile-from-org-b

# 2. verify both have cache files
ls ~/.aws/sso/cache/

# 3. start guided setup, select org-a
rhx keyrack set --vault aws.config
# select org-a domain

# 4. after pre-logout, check cache
ls ~/.aws/sso/cache/
```

**expected**: org-b cache files should still exist, only org-a cleared

**result**: [ ] pass / [ ] fail

---

## playtest 4: identity check on unlock

**goal**: verify mismatch detection when authed as wrong SSO user

**steps**:

```bash
# 1. setup a key that expects user alice@acme
rhx keyrack set --key AWS_ACCESS_KEY_ID --vault aws.config
# complete the guided setup as alice

# 2. logout and login as different user bob@acme
aws sso logout
aws sso login --profile bob-profile

# 3. try to unlock
rhx keyrack unlock --env test
```

**expected**:
- system detects bob is logged in but alice was expected
- auto-clears cache and prompts for re-auth as alice

**result**: [ ] pass / [ ] fail
