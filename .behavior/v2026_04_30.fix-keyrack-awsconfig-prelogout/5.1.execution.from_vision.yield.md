## implementation notes

### org label extraction

extract subdomain from `https://acme.awsapps.com/start` → `acme`

### keyrack set flow

after sso domain selection, before browser auth:
```
├─ with sso prior?
│  ├─ ✗ https://acme.awsapps.com/start, expired 2026-04-30T15:44:07Z
│  └─ ✓ cleared, to prevent collisions
```

or if no prior session:
```
├─ with sso prior?
│  └─ ✓ no prior session
```

### keyrack unlock flow

no prior session:
```
├─ with sso prior?
│  └─ ✓ clear, no prior session
```

prior session works:
```
├─ with sso prior?
│  ├─ ✓ alice@acme, access confirmed
│  └─ ✓ clear, will reuse
```

prior session conflict:
```
├─ with sso prior?
│  ├─ ✗ https://acme.awsapps.com/start, access denied
│  └─ ✓ cleared, will re-auth
```

note: collision case shows URL instead of username because `get-caller-identity` failed.

note: on `set` we don't know which user will auth, so we always clear prior.
todo: track username per key as host metadata so unlock can compare directly.
