# blackbox criteria: keyrack output modes

## keyrack get --value

### usecase.1 = get raw secret value for pipe use

```
given(key is granted)
  when(user runs `rhx keyrack get --key API_KEY --env test --value`)
    then(stdout contains raw secret value)
      sothat(secret can be piped to other commands)
    then(stdout has no final newline)
      sothat(piped value has no extra whitespace)
    then(exit code is 0)

given(key is granted)
  when(user runs `rhx keyrack get --key API_KEY --env test --output value`)
    then(behavior is identical to --value flag)
      sothat(both syntaxes are equivalent)
```

### usecase.2 = handle not granted states

```
given(key is locked)
  when(user runs `rhx keyrack get --key API_KEY --env test --value`)
    then(exit code is 2)
    then(stderr shows locked status)
      sothat(user knows to unlock first)

given(key is absent)
  when(user runs `rhx keyrack get --key API_KEY --env test --value`)
    then(exit code is 2)
    then(stderr shows absent status)
      sothat(user knows key does not exist)

given(key is blocked)
  when(user runs `rhx keyrack get --key API_KEY --env test --value`)
    then(exit code is 2)
    then(stderr shows blocked status)
```

### usecase.3 = --value requires --key

```
given(no --key specified)
  when(user runs `rhx keyrack get --env test --value`)
    then(exit code is non-zero)
    then(stderr shows error: --value requires --key)
      sothat(user knows --value only works with single key)

given(--for repo specified instead of --key)
  when(user runs `rhx keyrack get --for repo --env test --value`)
    then(exit code is non-zero)
    then(stderr shows error: --value requires --key)
```

## keyrack get --output modes

### usecase.4 = output mode selection

```
given(key is granted)
  when(user runs `rhx keyrack get --key API_KEY --env test --output json`)
    then(stdout contains JSON structure)
    then(behavior is identical to --json flag)

given(key is granted)
  when(user runs `rhx keyrack get --key API_KEY --env test --output vibes`)
    then(stdout contains turtle treestruct format)
    then(this is the default when no --output specified)

given(key is granted)
  when(user runs `rhx keyrack get --key API_KEY --env test`)
    then(stdout contains turtle treestruct format)
      sothat(default output is vibes)
```

## keyrack source

### usecase.5 = source all repo keys

```
given(all repo keys are granted for env)
  when(user runs `rhx keyrack source --env test --owner ehmpath`)
    then(stdout contains export statements for each key)
      sothat(output can be eval'd in shell)
    then(export format is `export KEY='secret'`)
    then(secrets with single quotes are escaped as `'sec'\''ret'`)
    then(exit code is 0)

given(user evals the output)
  when(user runs `eval "$(rhx keyrack source --env test --owner ehmpath)"`)
    then(environment variables are set in current shell)
      sothat(subsequent commands can access secrets)
```

### usecase.6 = source single key

```
given(key is granted)
  when(user runs `rhx keyrack source --key API_KEY --env test --owner ehmpath`)
    then(stdout contains single export statement)
    then(only the specified key is exported)
    then(exit code is 0)
```

### usecase.7 = strict mode (default)

```
given(some repo keys are not granted)
  when(user runs `rhx keyrack source --env test --owner ehmpath`)
    then(exit code is 2)
    then(stderr shows which keys are not granted)
    then(stderr hints: "use --lenient if partial results are acceptable")
      sothat(user knows how to proceed if partials are okay)
    then(no export statements are emitted to stdout)
      sothat(eval does not partially set env)

given(some repo keys are not granted)
  when(user runs `rhx keyrack source --env test --owner ehmpath --strict`)
    then(behavior is identical to default)
      sothat(--strict is explicit form of default)
```

### usecase.8 = lenient mode

```
given(some repo keys are not granted)
  when(user runs `rhx keyrack source --env test --owner ehmpath --lenient`)
    then(stdout contains export statements for granted keys only)
    then(absent keys are silently skipped)
      sothat(SDK parity is maintained)
    then(exit code is 0)
      sothat(partial results are acceptable)

given(no repo keys are granted)
  when(user runs `rhx keyrack source --env test --owner ehmpath --lenient`)
    then(stdout is empty)
    then(exit code is 0)
      sothat(empty result is acceptable in lenient mode)
```

### usecase.9 = required inputs

```
given(--env not specified)
  when(user runs `rhx keyrack source --owner ehmpath`)
    then(exit code is non-zero)
    then(stderr shows error: --env is required)

given(--owner not specified)
  when(user runs `rhx keyrack source --env test`)
    then(exit code is non-zero)
    then(stderr shows error: --owner is required)
```

## edge cases

### usecase.10 = secrets with special characters

```
given(secret contains single quote)
  when(user runs `rhx keyrack source --key QUOTED_SECRET --env test --owner ehmpath`)
    then(export statement escapes quote as `'sec'\''ret'`)
      sothat(shell eval works correctly)

given(secret contains newlines)
  when(user runs `rhx keyrack get --key MULTILINE --env test --value`)
    then(newlines are preserved in output)
      sothat(multiline secrets work correctly)

given(secret contains newlines)
  when(user runs `rhx keyrack source --key MULTILINE --env test --owner ehmpath`)
    then(export statement uses $'...' syntax or equivalent)
      sothat(shell eval preserves newlines)
```
