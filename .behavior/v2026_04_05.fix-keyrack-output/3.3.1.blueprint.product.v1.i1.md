# blueprint: keyrack output modes

## summary

implement two features for keyrack CLI:
1. `keyrack get --value` / `--output value` — output raw secret value for pipe use
2. `keyrack source` — output shell export statements for eval

both features match SDK semantics (strict/lenient modes).

### keyrack source

```
invokeKeyrack.ts
├─ [+] keyrack.command('source')
│  ├─ [+] .description('output export statements for shell eval')
│  ├─ [+] .option('--key <keyname>', 'single key to source')
│  ├─ [+] .option('--env <env>', 'target env (required)')
│  ├─ [+] .option('--owner <owner>', 'owner identity (required)')
│  ├─ [+] .option('--strict', 'fail if any key not granted (default)')
│  ├─ [+] .option('--lenient', 'skip absent keys silently')
│  └─ [+] .action(async (opts) => { ... })
│     ├─ [+] validate: --env required
│     ├─ [+] validate: --owner required
│     ├─ [+] validate: --strict and --lenient mutually exclusive
│     ├─ [+] if opts.key → get single key
│     │  └─ [←] reuse keyrack get --key --for single key
│     ├─ [+] else → get all repo keys
│     │  └─ [←] reuse keyrack get --for repo
│     ├─ [+] filter to granted keys
│     ├─ [+] strict mode (default):
│     │  ├─ if any not granted → stderr + hint + exit 2
│     │  └─ no stdout (prevent partial eval)
│     ├─ [+] lenient mode:
│     │  └─ silently skip not granted (SDK parity)
│     └─ [+] emit export statements (inline)
│        ├─ for each granted attempt:
│        │  ├─ keyName = slug.split('.').slice(2).join('.')  # handles dots in keyname
│        │  └─ console.log(`export ${keyName}=${asShellEscapedSecret({ secret })}`)
│        └─ [→] asShellEscapedSecret({ secret })
```

### snapshot strategy

snapshots verify output format stability for:
- `--value` raw output (no final newline)
- `--output json` structure
- `--output vibes` treestruct format
- `source` export statement format
- error messages with hints

snapshots enable visual diff in PRs to catch format regressions.

## contracts

### keyrack get --output / --value

| input | type | required | default |
|-------|------|----------|---------|
| `--key <keyname>` | string | yes (for --value) | — |
| `--env <env>` | string | yes | — |
| `--output <mode>` | 'value' \| 'json' \| 'vibes' | no | 'vibes' |
| `--value` | boolean | no | false |
| `--json` | boolean | no | false |

| output | condition |
|--------|-----------|
| stdout: raw secret | --output value, granted |
| stdout: JSON | --output json |
| stdout: treestruct | --output vibes (default) |
| stderr: status message | not granted with --value |
| exit 0 | granted |
| exit 2 | not granted |

### keyrack source

| input | type | required | default |
|-------|------|----------|---------|
| `--key <keyname>` | string | no | all repo keys |
| `--env <env>` | string | yes | — |
| `--owner <owner>` | string | yes | — |
| `--strict` | boolean | no | true (default) |
| `--lenient` | boolean | no | false |

| output | condition |
|--------|-----------|
| stdout: export statements | success |
| stdout: empty | strict mode failure (no partial) |
| stderr: which keys not granted | strict mode failure |
| stderr: hint for --lenient | strict mode failure |
| exit 0 | all granted OR lenient mode |
| exit 2 | strict mode + some not granted |

## shell escape rules

| secret content | escaped output | reason |
|----------------|----------------|--------|
| `secret` | `'secret'` | single quotes for safety |
| `sec'ret` | `'sec'\''ret'` | escape single quote |
| `line1\nline2` | `$'line1\nline2'` | ANSI-C quoted for newlines |
| `back\slash` | `'back\\slash'` | preserve backslash |

---

## SDK parity

| SDK behavior | CLI behavior |
|--------------|--------------|
| `keyrack.source({ mode: 'strict' })` | `keyrack source --strict` |
| `keyrack.source({ mode: 'lenient' })` | `keyrack source --lenient` |
| strict = default | strict = default |
| lenient skips silently | lenient skips silently |
| exit 2 on strict failure | exit 2 on strict failure |

CLI extends SDK with `--key` filter (SDK has no key filter).
