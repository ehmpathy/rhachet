# blackbox criteria matrix: keyrack output modes

## matrix 1: keyrack get --value / --output value

### dimensions

- **ind: key status** — granted, locked, absent, blocked
- **ind: output flag** — `--value`, `--output value`, none
- **ind: key specifier** — `--key X`, `--for repo`, none
- **dep: stdout** — raw value, vibes, error
- **dep: exit code** — 0, 2, non-zero

### coverage

| ind: key status | ind: output flag | ind: key specifier | dep: stdout | dep: exit code |
|-----------------|------------------|-------------------|-------------|----------------|
| granted | `--value` | `--key X` | raw value (no newline) | 0 |
| granted | `--output value` | `--key X` | raw value (no newline) | 0 |
| granted | none | `--key X` | vibes (treestruct) | 0 |
| locked | `--value` | `--key X` | stderr: locked status | 2 |
| absent | `--value` | `--key X` | stderr: absent status | 2 |
| blocked | `--value` | `--key X` | stderr: blocked status | 2 |
| any | `--value` | `--for repo` | stderr: --value requires --key | non-zero |
| any | `--value` | none | stderr: --value requires --key | non-zero |

### gaps: none

all meaningful combinations covered.
