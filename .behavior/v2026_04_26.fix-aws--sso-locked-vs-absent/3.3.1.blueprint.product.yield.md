# blueprint.product: aws sso locked vs absent

## summary

add a generic inventory system that tracks which keys have been set on the host. this enables accurate "locked vs absent" status for all vaults (not just os.secure and os.direct).

### research references

per `3.1.1.research.external.product.flagged._.yield.md`: no external dependencies — behavior operates on internal keyrack systems only.

per `3.1.3.research.internal.product.code.prod._.yield.md`:
- patterns [EXTEND]: inferKeyrackKeyStatusWhenNotGranted, setKeyrackKeyHost, delKeyrackKeyHost
- patterns [REUSE]: slug-hash (`asHashSha256Sync(slug).slice(0, 16)`), owner directory (`owner=${owner ?? 'default'}`), vault path structure

### key changes

1. create `daoKeyrackInventory` for inventory entry crud
2. update `setKeyrackKeyHost` to set inventory entry after set
3. update `delKeyrackKeyHost` to remove inventory entry (last)
4. update `inferKeyrackKeyStatusWhenNotGranted` to check inventory first

## implementation samples

### getHomeDir.ts (extract to src/infra/)

```ts
import { UnexpectedCodePathError } from 'helpful-errors';

/**
 * .what = resolves the home directory
 * .why = uses HOME env var to support test isolation
 *
 * .note = os.homedir() caches at module load; we read process.env.HOME directly
 * .note = extracted from getKeyrackHostManifestPath.ts:11-15 for reuse
 */
export const getHomeDir = (): string => {
  const home = process.env.HOME;
  if (!home) throw new UnexpectedCodePathError('HOME not set', {});
  return home;
};
```

### asKeyrackOwnerDir.ts (shared transformer)

```ts
/**
 * .what = computes the owner directory segment for keyrack paths
 * .why = DRY — 3+ usages: os.secure vault, os.direct vault, inventory

export const asKeyrackOwnerDir = (input: { owner: string | null }): string =>
```

### asKeyrackSlugHash.ts (shared transformer)

```ts
import { asHashSha256Sync } from 'hash-fns';

/**
 * .what = computes the 16-char slug hash for keyrack paths
 * .why = DRY — 3+ usages: os.secure vault, os.direct vault, inventory

export const asKeyrackSlugHash = (input: { slug: string }): string =>
```

### daoKeyrackInventory/index.ts

```ts
import { existsSync, mkdirSync, rmSync, writeFileSync } from 'node:fs';
import { join } from 'node:path';

import { getHomeDir } from '../../../infra/getHomeDir';
import { asKeyrackOwnerDir } from '../../../domain.operations/keyrack/asKeyrackOwnerDir';
import { asKeyrackSlugHash } from '../../../domain.operations/keyrack/asKeyrackSlugHash';

const getInventoryPath = (input: { slug: string; owner: string | null }): string => {
  const home = getHomeDir();
  const ownerDir = asKeyrackOwnerDir({ owner: input.owner });
  const hash = asKeyrackSlugHash({ slug: input.slug });
  return join(home, '.rhachet', 'keyrack', 'inventory', ownerDir, `${hash}.stocked`);
};

export const daoKeyrackInventory = {
  exists: (input: { slug: string; owner: string | null }): boolean => {
    return existsSync(getInventoryPath(input));
  },

  set: (input: { slug: string; owner: string | null }): void => {
    const path = getInventoryPath(input);
    const dir = join(path, '..');
    mkdirSync(dir, { recursive: true, mode: 0o700 });
    writeFileSync(path, '', { mode: 0o600 });
  },

  del: (input: { slug: string; owner: string | null }): void => {
    const path = getInventoryPath(input);
    if (existsSync(path)) rmSync(path);
  },
};
```

## error handle

| scenario | error | resolution |
|----------|-------|------------|
| HOME env var unset | getHomeDir throws | UnexpectedCodePathError('HOME not set') |
| inventory dir not writable | mkdir fails | throw with chmod hint |
| inventory file not writable | writeFile fails | throw with chmod hint |
| orphan inventory entry | del fails | silent no-op (idempotent) |

## edge cases

| edge case | behavior |
|-----------|----------|
| inventory exists, vault gone | status=locked → unlock fails → user runs set |
| vault exists, inventory gone | status=locked via vault check (backward compat) |
| set idempotent | re-set is no-op (file already exists) |
| del idempotent | re-del is no-op (file already gone) |
| env=all fallback | checks both exact slug and env=all slug |
| owner=null | uses `owner=default` directory |

## scope note

**vault type coverage:** the wish is specifically about aws.config vault (AWS_PROFILE). acceptance tests focus on aws.config vault as the primary validation. the inventory mechanism is vault-agnostic and works for all vault types (os.secure, os.direct, aws.config, 1password), but acceptance tests target the specific usecase from the wish.

**unit/integration tests** for daoKeyrackInventory and inferKeyrackKeyStatusWhenNotGranted provide generic coverage that validates the mechanism works regardless of vault type.
