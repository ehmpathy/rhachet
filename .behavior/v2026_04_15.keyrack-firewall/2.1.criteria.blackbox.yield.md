# 2.1.criteria.blackbox.yield — keyrack firewall

blackbox criteria for the keyrack firewall GitHub Action.

---

## usecase.1 = credential translation

the action translates self-descriptive credential blobs to usable tokens.

```
given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(secrets input contains toJSON(secrets))
  given(github secret GITHUB_TOKEN contains JSON blob with mech: EPHEMERAL_VIA_GITHUB_APP)
  given(keyrack.yml declares GITHUB_TOKEN)
    when(action executes)
      then(subsequent steps have GITHUB_TOKEN env var with ghs_* token)
        sothat(test code can use the translated token directly)
      then(original JSON blob is not visible in env)
        sothat(source credentials are not leaked)
      then(token value is masked in logs)
        sothat(translated tokens cannot be extracted from CI output)
```

---

## usecase.2 = credential filtering

the action only processes keys declared in keyrack.yml.

```
given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(secrets input contains toJSON(secrets) with 10 secrets)
  given(keyrack.yml declares only 2 keys: GITHUB_TOKEN and AWS_CREDS)
    when(action executes)
      then(only GITHUB_TOKEN and AWS_CREDS are processed)
        sothat(no wasted translation work)
      then(only GITHUB_TOKEN and AWS_CREDS are exported to $GITHUB_ENV)
        sothat(keyrack.yml is single source of truth for which keys matter)
      then(other 8 secrets are ignored)
        sothat(workflow author does not need to enumerate secrets manually)
```

## usecase.3 = credential blocking

the action blocks dangerous long-lived credentials.

```
given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(github secret GITHUB_TOKEN contains raw ghp_* token without mech field)
  given(keyrack.yml declares GITHUB_TOKEN)
    when(action executes)
      then(action fails)
        sothat(dangerous credentials never reach workflow steps)
      then(output shows blocked status with reason)
        sothat(developer knows why it failed)
      then(output shows fix suggestion)
        sothat(developer knows how to fix)

given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(github secret AWS_KEY contains raw AKIA* token without mech field)
  given(keyrack.yml declares AWS_KEY)
    when(action executes)
      then(action fails)
        sothat(long-lived AWS keys are blocked)
      then(output shows blocked status: detected aws access key)
```

## usecase.4 = credential passthrough

the action passes through secrets without mech field (after firewall validation).

```
given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(github secret API_KEY contains plain string value without mech field)
  given(API_KEY does not match blocked patterns (ghp_*, AKIA*))
  given(keyrack.yml declares API_KEY)
    when(action executes)
      then(subsequent steps have API_KEY env var with original value)
        sothat(non-translatable secrets still work)
      then(value is masked in logs)
        sothat(passthrough secrets are protected)
```

## usecase.5 = debug experience on success

the action provides clear output on successful translation.

```
given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(github secret GITHUB_TOKEN contains valid JSON blob with mech field)
  given(keyrack.yml declares GITHUB_TOKEN)
    when(action executes successfully)
      then(output shows treestruct with granted status)
      then(output shows mechanism used for translation)
      then(output shows token prefix with mask (ghs_***))
      then(output shows expiry time if applicable)
        sothat(developer can verify translation occurred)
```

## usecase.6 = debug experience on failure

the action provides clear output on translation failure.

```
given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(github secret GITHUB_TOKEN contains malformed JSON)
  given(keyrack.yml declares GITHUB_TOKEN)
    when(action executes)
      then(action fails fast)
      then(output shows parse error with key name)
      then(output shows hint for fix)
        sothat(developer can fix the secret format)

given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(github secret GITHUB_TOKEN contains JSON with unknown mech field)
  given(keyrack.yml declares GITHUB_TOKEN)
    when(action executes)
      then(action fails fast)
      then(output shows unknown mechanism error)
      then(output lists supported mechanisms)
        sothat(developer knows valid options)

given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(github secret GITHUB_TOKEN contains valid JSON with mech: EPHEMERAL_VIA_GITHUB_APP)
  given(github app credentials are invalid or rate limited)
  given(keyrack.yml declares GITHUB_TOKEN)
    when(action executes)
      then(action fails fast)
      then(output shows github API error)
      then(output shows which key failed)
        sothat(developer can diagnose the credential issue)
```

## usecase.7 = multiple secrets with partial failure

the action fails fast when any secret fails.

```
given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(keyrack.yml declares GITHUB_TOKEN and AWS_CREDS)
  given(GITHUB_TOKEN contains valid JSON blob)
  given(AWS_CREDS contains malformed JSON)
    when(action executes)
      then(action fails fast on first error)
        sothat(partial success is not possible)
      then(output identifies which secret failed)
        sothat(developer knows what to fix)
      then(GITHUB_TOKEN is NOT exported to env)
        sothat(no credentials are exposed on partial failure)
```

---

## usecase.8 = keyrack.yml not found

the action handles absence of keyrack.yml.

```
given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(secrets input contains toJSON(secrets))
  given(repo does not have keyrack.yml)
    when(action executes)
      then(action fails fast)
      then(output shows keyrack.yml not found error)
      then(output shows hint: create keyrack.yml with key declarations)
        sothat(developer knows to declare keys)
```

---

## usecase.9 = secret not in github secrets

the action handles keys declared in keyrack.yml but absent from github secrets.

```
given(workflow step uses ehmpathy/rhachet/keyrack/firewall@v1)
  given(keyrack.yml declares GITHUB_TOKEN and MISSING_KEY)
  given(secrets input has GITHUB_TOKEN but not MISSING_KEY)
    when(action executes)
      then(GITHUB_TOKEN is processed and exported)
      then(MISSING_KEY is reported as absent)
      then(action does NOT fail)
        sothat(workflow can proceed with available keys)
      then(output shows which keys were absent)
        sothat(developer can add the secret if needed)
```

---

## usecase.10 = per-job isolation

credentials cannot flow between jobs.

```
given(workflow has job-A with firewall step)
  given(workflow has job-B without firewall step)
  given(job-B depends on job-A)
    when(job-A exports GITHUB_TOKEN to $GITHUB_ENV)
      then(job-B does NOT have GITHUB_TOKEN in env)
        sothat(credentials stay within job boundary)
        sothat(each job must add its own firewall step)
```
