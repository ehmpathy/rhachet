## what blocks me

### root cause: review conflates test types

| test type | convention | purpose | real calls? |
|-----------|------------|---------|-------------|
| unit | `.test.ts` | internal logic | mocks OK |
| integration | `.integration.test.ts` | external contracts | real calls |
| acceptance | `blackbox/` | user-faced contracts | subprocess |

the review treats ALL test files the same and flags unit tests for:
1. no snapshots (rule targets user-faced contracts, not internal adapters)
2. no real API calls (that role belongs to integration tests)
