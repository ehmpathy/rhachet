# 5.5.playtest.yield — keyrack firewall

byhand verification of the keyrack firewall CLI.

---

## prerequisites

1. rhachet installed: `npm ci` in repo root
2. test repo with keyrack.yml:
   ```yaml
   keys:
     - slug: ehmpath.test.GITHUB_TOKEN
     - slug: ehmpath.test.AWS_CREDS
     - slug: ehmpath.test.SAFE_KEY
   ```
3. temp file for GITHUB_ENV mock: `export GITHUB_ENV=$(mktemp)`

---

## playtest 1: credential translation (github app → ghs_*)

**goal**: verify github app JSON blob translates to ghs_* token

**setup**:
```bash
export SECRETS='{"GITHUB_TOKEN":"{\"appId\":\"123\",\"privateKey\":\"-----BEGIN RSA PRIVATE KEY-----\\ntest\\n-----END RSA PRIVATE KEY-----\",\"installationId\":\"456\",\"mech\":\"EPHEMERAL_VIA_GITHUB_APP\"}"}'
```

**invoke**:
```bash
npx rhachet keyrack firewall --env test --owner ehmpath --from 'json(env://SECRETS)' --into json
```

**pass criteria**:
- [ ] output contains `"status": "granted"`
- [ ] output shows `"mech": "EPHEMERAL_VIA_GITHUB_APP"`
- [ ] secret value starts with `ghs_` (in CI with real app creds)
- [ ] exit code 0

**note**: translation requires valid GitHub App credentials. in local test without real creds, the mech detection and grant structure should still be visible.
