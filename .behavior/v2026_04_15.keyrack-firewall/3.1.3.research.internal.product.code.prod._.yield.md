## key patterns found

### 1. mechanism adapters

**pattern**: adapter interface for credential translation with `acquireForSet` and `deliverForGet` methods.

**location**: `src/domain.operations/keyrack/adapters/mechanisms/`

**relevant adapters**:

| adapter | purpose | for firewall |
|---------|---------|--------------|
| `mechAdapterGithubApp` | translates JSON blob → `ghs_*` token via `@octokit/auth-app` | [REUSE] |
| `mechAdapterReplica` | passthrough with firewall validation (`LONG_LIVED_PATTERNS`) | [REUSE] |
| `mechAdapterAwsSso` | translates AWS SSO profile → session credentials | [REUSE] |

**`mechAdapterGithubApp.deliverForGet()` signature**:
```typescript
deliverForGet: async (input: { source: string }): Promise<{ secret: string; expiresAt?: IsoTimeStamp }> => {
  const creds = JSON.parse(input.source);
  const auth = createAppAuth({
    appId: creds.appId,
    privateKey: creds.privateKey,
    installationId: Number(creds.installationId),
  });
  const { token } = await auth({ type: 'installation' });
  const expiresAt = addDuration(asIsoTimeStamp(new Date()), { minutes: 55 });
  return { secret: token, expiresAt };
}
```

**decision**: [REUSE] — the firewall action will import and call the same mechanism adapters.

### 2. firewall validation patterns

**pattern**: regex array for dangerous long-lived token detection.

**location**: `src/domain.operations/keyrack/adapters/mechanisms/mechAdapterReplica.ts:19-29`

```typescript
const LONG_LIVED_PATTERNS = [
  /^ghp_[a-zA-Z0-9]{36}$/,  // github classic pat
  /^gho_[a-zA-Z0-9]{36}$/,  // github oauth
  /^ghu_[a-zA-Z0-9]{36}$/,  // github user-to-server
  /^ghs_[a-zA-Z0-9]{36}$/,  // BUG: installation tokens are SHORT-lived
  /^ghr_[a-zA-Z0-9]{36}$/,  // github refresh
  /^AKIA[A-Z0-9]{16}$/,     // aws access key
];
```

**bug identified**: `ghs_*` pattern is incorrectly included. GitHub App installation tokens (`ghs_*`) are short-lived (1 hour max, enforced by GitHub). they are the OUTPUT of `EPHEMERAL_VIA_GITHUB_APP` translation and should NOT be blocked.

**decision**: [EXTEND] — reuse `LONG_LIVED_PATTERNS` but remove `ghs_*` entry. this fix must also be applied to the local `mechAdapterReplica.ts`.

### 7. sourceAllKeysIntoEnv pattern

**pattern**: inject keyrack grants into `process.env` for test setup.

**location**: `src/domain.operations/keyrack/sourceAllKeysIntoEnv.ts`

**key flow**:
1. call `rhx keyrack get --for repo --json`
2. parse JSON response (array of `KeyrackGrantAttempt`)
3. filter to keys for the requested env
4. fail-fast if any key not granted (strict mode)
5. inject into `process.env`

```typescript
for (const key of keysForEnv) {
  if (key.status !== 'granted') continue;
  const envVarName = key.grant.slug.split('.').pop();
  if (envVarName && !process.env[envVarName]) {
    process.env[envVarName] = key.grant.key.secret;
  }
}
```

**decision**: [EXTEND] — the firewall action will follow similar pattern but:
- use `core.setSecret()` to mask values in logs
- use `core.exportVariable()` to write to `$GITHUB_ENV`
- work directly with github secrets input (not subprocess)

## codepath for firewall action

the firewall action will compose these patterns:

```
toJSON(secrets) input
    ↓
daoKeyrackRepoManifest.get() → filter to declared keys
    ↓
for each declared key:
    ↓
    parse value → detect mech field?
    ↓
    ├── yes: mechAdapter[mech].deliverForGet() → translated token
    └── no: mechAdapterReplica.validate() → passthrough or block
    ↓
    core.setSecret(translated/passthrough)
    core.exportVariable(keyName, translated/passthrough)
    ↓
emit structured output (KeyrackGrantAttempt[] equivalent)
```
