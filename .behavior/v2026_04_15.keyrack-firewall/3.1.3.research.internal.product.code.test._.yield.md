### 2. invokeRhachetCliBinary

**pattern**: subprocess invocation with output capture.

**location**: `blackbox/.test/infra/invokeRhachetCliBinary.ts`

**signature**:
```typescript
invokeRhachetCliBinary(input: {
  args: string[];
  cwd: string;
  env?: Record<string, string>;
  stdin?: string;
}): {

  stdout: string;
  stderr: string;
```

- captures stdout, stderr, exit status
- supports custom env vars for test scenarios

### 4. firewall acceptance test structure

**pattern**: BDD tests with `given/when/then` for firewall behavior.

**location**: `blackbox/cli/keyrack.firewall.acceptance.test.ts`

**test structure**:
```typescript
describe('keyrack firewall', () => {
  given('[case1] safe api key', () => {
    const repo = useBeforeAll(() => genTestTempRepo({ fixture: 'with-firewall-test' }));

    when('[t0] firewall processes key', () => {
      const result = useThen('it succeeds', async () =>
        invokeRhachetCliBinary({
          args: ['keyrack', 'firewall', '--json'],
          cwd: repo.path,
          env: { SAFE_API_KEY: 'test-value-safe' },
        }),
      );

      then('key is granted', () => {
        expect(result.status).toBe(0);
        const output = JSON.parse(result.stdout);
        expect(output.grants.SAFE_API_KEY.status).toBe('granted');
      });
    });
  });

  given('[case2] dangerous ghp token', () => {
    when('[t0] firewall processes key', () => {
      then('key is blocked', async () => {
        const result = await invokeRhachetCliBinary({
          args: ['keyrack', 'firewall', '--json'],
          cwd: repo.path,
          env: { GHP_TOKEN: 'ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx' },
        });
        expect(result.status).toBe(2); // constraint error
        const output = JSON.parse(result.stdout);
        expect(output.grants.GHP_TOKEN.status).toBe('blocked');
      });
    });
  });
});
```

**decision**: [EXTEND] — firewall action tests will follow same BDD structure but test action execution rather than CLI.

## test codepath for firewall action

the firewall action tests will compose these patterns:

```
1. test setup
   ├─ genTestTempRepo({ fixture: 'with-firewall-action-test' })
   └─ create mock secrets input

2. action invocation
   ├─ simulate action execution (node action entry point)
   └─ pass secrets as JSON input

3. output verification
   ├─ check $GITHUB_ENV writes
   ├─ check core.setSecret() calls
   ├─ check exit status
   └─ snapshot action output

4. cleanup
   └─ killKeyrackDaemonForTests() if needed
```
