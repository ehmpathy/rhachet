> **github action limits** [research]: any limits on env var size or count we need to consider?

## findings

### 1. individual secret/variable size limit

**[FACT]** secrets are limited to 48 KB in size.

> "Secrets are limited to 48 KB in size."
> — [1] GitHub Docs - Secrets reference [official] (2026)

**conflict detected**: sources [1] and [6] disagree:
- [1] official GitHub docs states 48 KB
- [6] practitioner blog states 64 KB

**conclusion**: the official GitHub documentation (48 KB) is authoritative. the 64 KB figure may be outdated or refer to a different limit.

---

### 2. combined variable size limit per workflow

**[FACT]** the total combined size limit for organization and repository variables is 256 KB per workflow run.

> "The total combined size limit for organization and repository variables is 256 KB per workflow run."
> — [2] GitHub Docs - Variables reference [official] (2026)

**note**: environment-level variables do NOT count toward this 256 KB limit.

---

### 3. variable count limits

**[FACT]** you can store:
- up to 1,000 organization secrets
- up to 100 repository secrets
- up to 100 environment secrets

> "You can store up to 1,000 organization secrets, 100 repository secrets, and 100 environment secrets."
> — [1] GitHub Docs - Secrets reference [official] (2026)

**convergence**: sources [1], [3], [4] all confirm the 100 repository secrets limit.

---

### 4. GITHUB_ENV file limits

**[SUMP]** the GITHUB_ENV file does not have a separate documented size limit beyond the per-variable 48 KB limit.

no official documentation found that specifies a separate limit for the total size of the GITHUB_ENV file. the assumption is that the 48 KB per-variable limit applies to each variable written via `core.exportVariable()`.

---

### 5. workarounds for large secrets

**[FACT]** for secrets larger than 48 KB, GitHub recommends base64 encode or encrypted files.

> "You can use Base64 to encode and store small binary blobs as secrets."
> — [7] GitHub Docs - Secrets in GitHub Actions [official] (2026)

> "For larger files... store the BASE64 encoded value of a file or use a file in the repository"
> — [6] Medium - Large Multi-line Secrets [practitioner] (2023)

---

### 6. job execution limits (context)

**[FACT]** each job has a maximum runtime of 6 hours.

> "Each job has a maximum runtime of 6 hours. If a job exceeds this limit, it will be terminated."
> — [8] GitHub Docs - Actions limits [official] (2026)

**relevance**: the 1-hour token expiry for GitHub App tokens is well within the 6-hour job limit.

---

## impact assessment for keyrack firewall

| limit | value | impact on firewall |
|-------|-------|-------------------|
| individual secret size | 48 KB | ✅ safe — github app JSON blobs are ~2-5 KB (private key is largest component) |
| combined variables per workflow | 256 KB | ✅ safe — unlikely to hit with typical keyrack.yml declarations |
| repository secrets count | 100 | ✅ safe — keyrack.yml unlikely to declare 100+ keys |
| GITHUB_ENV per variable | 48 KB | ✅ safe — translated tokens are small (< 1 KB) |

**conclusion**: no limits pose risk to the keyrack firewall action. typical credential blobs and translated tokens are well within all documented limits.

## open questions

**[KHUE]** what happens if you try to write more than 48 KB to a single env var via `core.exportVariable()`? does it truncate, error, or silently fail?

**mitigation**: add defensive check in firewall action — if translated token exceeds 40 KB, fail fast with clear error.
