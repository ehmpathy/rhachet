# blueprint: keyrack --prikey for remaining commands

## .what

add `--prikey` flag to all keyrack commands that decrypt the host manifest but currently lack it.

## .why

the `--prikey` flag was added to `init`, `set`, and `unlock` in v1.37.3. however, several other commands also decrypt the host manifest and need `--prikey` for custom owners with non-standard ssh key locations.

## .scope

### commands needing --prikey

| command | calls | needs --prikey |
|---------|-------|----------------|
| `list` | `genKeyrackHostContext({ owner })` | YES |
| `del` | `genKeyrackHostContext({ owner })` | YES |
| `recipient get` | `daoKeyrackHostManifest.get({ owner })` | YES |
| `recipient set` | `daoKeyrackHostManifest.get({ owner })` | YES |
| `recipient del` | `daoKeyrackHostManifest.get({ owner })` | YES |

### commands NOT needing --prikey

| command | why |
|---------|-----|
| `get` | reads from daemon via `genContextKeyrackGrantGet` |
| `status` | reads from daemon |
| `relock` | sends message to daemon |
