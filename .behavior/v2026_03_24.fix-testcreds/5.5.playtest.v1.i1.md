# playtest: keyrack for test credentials

## prerequisites

- rhachet installed (`./node_modules/.bin/rhx` available)
- ssh key at `~/.ssh/ehmpath` (for keyrack unlock)
- keyrack secrets populated for owner=ehmpath

## sandbox

no file operations required — this playtest verifies test runner behavior

---

## happy paths

### 1. verify legacy files deleted

```bash
ls -la .agent/repo=.this/role=any/skills/use.apikeys.*
```

**expected:** `ls: cannot access '.agent/repo=.this/role=any/skills/use.apikeys.*': No such file or directory`

**pass if:** files do not exist
**fail if:** either `use.apikeys.sh` or `use.apikeys.json` exists

---

### 2. verify keyrack locked behavior

first, ensure keyrack is locked:

```bash
rhx keyrack relock --owner ehmpath
```

then run integration tests:

```bash
npm run test:integration
```

**expected:** tests fail immediately with ConstraintError that displays:
- message: `keyrack not unlocked. unlock to run integration tests:`
- fix: `rhx keyrack unlock --env test --owner ehmpath`

**pass if:** error message shows exact unlock command
**fail if:** tests hang, show different error, or attempt to run without keys

---

### 3. verify keyrack unlocked behavior

unlock the keyrack:

```bash
rhx keyrack unlock --env test --owner ehmpath
```

then run integration tests:

```bash
npm run test:integration
```

**expected:** tests run with keys injected into process.env

**pass if:** tests execute (may pass or fail based on other factors, but keys are available)
**fail if:** ConstraintError about keyrack, or absent API key errors

---

### 4. verify acceptance tests same behavior

with keyrack still unlocked:

```bash
npm run test:acceptance
```

**expected:** tests run with keys injected

**pass if:** no keyrack-related errors
**fail if:** ConstraintError about keyrack

---

### 5. verify CI passthrough (env var override)

with keyrack relocked:

```bash
rhx keyrack relock --owner ehmpath
```

then run with explicit env var:

```bash
OPENAI_API_KEY=test-value npm run test:unit -- --testPathPattern="keyrack"
```

**expected:** unit tests run (keyrack check doesn't block because env vars are checked first by keyrack get)

**pass if:** tests run without keyrack unlock prompt
**fail if:** ConstraintError despite env var present

---

## edgey paths

### absent keys

if a key is configured in keyrack.yml but not populated in the vault:

**expected:** ConstraintError with:
- message: `keyrack keys absent:`
- list of absent keys with `rhx keyrack set` commands

---

## pass/fail criteria

- ✓ pass if:
  - legacy `use.apikeys.*` files are deleted
  - locked keyrack → ConstraintError with unlock command
  - unlocked keyrack → tests run with keys injected
  - CI env vars work as passthrough

- ✗ fail if:
  - legacy files still exist
  - error messages don't show fix commands
  - tests run without keys (silent failure)
  - env var passthrough broken
