1. extend prikey auto-discovery in daoKeyrackHostManifest

## phase 2: replace jest env apikeys pattern

### prereqs

read before start:
- `.behavior/v2026_03_24.fix-testcreds/3.1.3.research.internal.product.code.test._.v1.stone` — current jest env pattern
- `.behavior/v2026_03_24.fix-testcreds/3.3.1.blueprint.product.v1.i1.md` — keyrack get contract and ConstraintError format

### tasks

- [ ] 2.1 update jest.integration.env.ts
  - [ ] 2.1.1 remove use.apikeys.json read
  - [ ] 2.1.2 spawn `rhx keyrack get --for repo --env test --json --owner ehmpath`
  - [ ] 2.1.3 parse JSON response
  - [ ] 2.1.4 inject secrets into process.env when unlocked
  - [ ] 2.1.5 throw ConstraintError with unlock command when locked
  - [ ] 2.1.6 throw ConstraintError with set commands when keys absent
  - [ ] 2.1.7 add `.note` comment for hardcoded owner
- [ ] 2.2 update jest.acceptance.env.ts (same pattern as integration)
- [ ] 2.3 verify: `npm run test:integration` works with keyrack unlocked
- [ ] 2.4 verify: `npm run test:integration` fails with ConstraintError when keyrack locked

### acceptance criteria

```
given developer runs integration tests
  when keyrack is unlocked with required keys
    then tests run with keys injected into process.env
  when keyrack is locked
    then failfast with ConstraintError
    then error message shows: rhx keyrack unlock --env test --owner ehmpath
  when keys are absent
    then failfast with ConstraintError
    then error message shows rhx keyrack set command for each absent key
```
