# blackbox criteria: keyrack for test credentials

## usecase.1 = run integration tests

```
given('developer runs integration tests')
  when('keyrack is unlocked with required keys')
    then('tests run with keys injected into process.env')
      sothat('developer can run tests without manual source commands')
    then('no error message about keys')

  when('keyrack is locked')
    then('failfast with ConstraintError')
    then('error message shows exact unlock command')
      sothat('developer knows exactly what to run')
    then('error message includes: rhx keyrack unlock --env test --owner ehmpath')

  when('keyrack is unlocked but keys are absent')
    then('failfast with ConstraintError')
    then('error message shows which keys are absent')
    then('error message shows rhx keyrack set command for each absent key')
      sothat('developer knows how to populate the vault')
```

## usecase.2 = run acceptance tests

```
given('developer runs acceptance tests')
  when('keyrack is unlocked with required keys')
    then('tests run with keys injected into process.env')

  when('keyrack is locked')
    then('failfast with ConstraintError')
    then('error message shows exact unlock command')

  when('keyrack is unlocked but keys are absent')
    then('failfast with ConstraintError')
    then('error message shows rhx keyrack set command for each absent key')
```

## usecase.3 = CI environment

```
given('tests run in CI environment')
  when('keys are present via os.envvar passthrough')
    then('tests run with keys from environment')
    then('no keyrack unlock required')
      sothat('CI pipelines work without interactive unlock')
```

## usecase.4 = keyrack unlock with prikey auto-discovery

```
given('developer runs keyrack unlock')
  when('--prikey is not specified')
    then('keyrack auto-discovers prikey from ~/.ssh/$owner')
      sothat('developer does not need to specify --prikey ~/.ssh/ehmpath')

  when('--prikey is specified')
    then('keyrack uses the specified prikey')
```
