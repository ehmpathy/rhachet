# nitpick.1: Prefer HelpfulError.wrap over manual try/catch for error wrapping

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.errors/rule.prefer.helpful-error-wrap.md

**locations**:
- src/access/daos/daoKeyrackHostManifest/index.ts

The code uses manual try/catch blocks to catch and rethrow errors with additional context using BadRequestError. While this is not hiding errors (it's rethrowing), the rule prefers using HelpfulError.wrap for better observability and consistency. This makes errors more observable by automatically handling error wrapping.
Change to use HelpfulError.wrap around the async operations like decryptWithIdentity and JSON.parse to align with the preferred pattern.

**snippet**:
```ts
    // decrypt with explicit identity if available
    let plaintext: string | undefined;
    if (explicitIdentity) {
      try {
        plaintext = await decryptWithIdentity({
          ciphertext,
          identity: explicitIdentity,
        });
        // save explicit identity for subsequent operations (e.g., vault decryption)
        sessionIdentity = explicitIdentity;
      } catch (error) {
        throw new BadRequestError('failed to decrypt host manifest', {
          path,
          owner,
          error: error instanceof Error ? error.message : String(error),
        });
      }
    }
```
