## question 2: slug field addition

**question:** vision adds `slug` to execution; wish only specifies `exid`. is `slug` desired?

**can this be reasoned?** yes

**reason:**
- `slug` enables defensive validation — suppliers can verify execution matches expected tool
- without `slug`, only exid correlates — but exid is opaque, doesn't tell us which tool
- vision's rationale: "suppliers can validate that the execution matches the expected tool type"
- this is a pit-of-success design — prevents mismatched executions

**answer:** ✅ `slug` should be included. it enables validation without contract breaks.

**escalate?** no — this is a design improvement, not a deviation from wish

## question 13: repl.act() vs repl.ask() tool use

**question:** any difference in how they handle tool results?

**can this be researched?** yes — via codebase

**research conducted:**
read `src/domain.objects/BrainRepl.ts` (lines 48-92):

```ts
// ask: readonly analysis operation
ask: <TOutput, TPlugs extends BrainPlugs = BrainPlugs>(
  input: {
    on?: PickOne<{ episode: BrainEpisode; series: BrainSeries }>;
    plugs?: TPlugs;
    role: { briefs?: Artifact<typeof GitFile>[] };
    prompt: string;
    schema: { output: z.Schema<TOutput> };
  },
  context?: Empty,
) => Promise<BrainOutput<TOutput, 'repl', TPlugs>>;

// act: read+write action operation
act: <TOutput, TPlugs extends BrainPlugs = BrainPlugs>(
  input: {
    // identical signature
  },
  context?: Empty,
) => Promise<BrainOutput<TOutput, 'repl', TPlugs>>;
```

key finding from comments:
- `ask`: `disallowedTools=["Edit","Write","Bash","NotebookEdit"]`
- `act`: `allowedTools=["Read","Edit","Write","Bash","Glob","Grep"]`

**difference:** permission scope (which tools the repl can execute), NOT tool handling mechanics

**answer:** ✅ signatures are identical. both accept same `plugs?.tools` and return same output shape. the only difference is which built-in tools the repl can use (read-only vs read-write). user-provided tools via `plugs.tools` are handled identically.

**evidence quality:** high — verified from source code

**escalate?** no
