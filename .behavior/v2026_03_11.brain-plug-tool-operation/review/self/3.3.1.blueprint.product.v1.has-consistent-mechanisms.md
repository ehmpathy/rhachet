## conclusion

### what i found

i searched for extant mechanisms before evaluating each new component in the blueprint. i found:

**no duplication:**
- BrainPlugToolExecution — fills gap between invocation and continuation
- genMockedBrainPlugToolExecution — follows extant genMocked* pattern
- genMockedBrainPlugToolInvocation — fills gap for extant domain object
- AsBrainPromptFor — follows extant As*For pattern
- signal enum — distinct from error types (data vs throw)
- metrics.cost.time — subset of extant pattern

**patterns reused:**
- DomainLiteral base class (from domain-objects)
- genMocked* factory pattern (6 extant factories)
- As*For conditional type pattern (AsBrainOutputCallsFor)
- IsoDuration for time metrics (from iso-time)

### why no mechanisms were flagged

each new mechanism either:
1. fills an identified gap (BrainPlugToolExecution, genMockedBrainPlugToolInvocation)
2. follows extant pattern (genMocked*, As*For)
3. serves distinct use case from similar extant (signal vs error types)

the blueprint maximizes reuse of extant patterns while it introduces only what is needed for the new feature.

## second pass: what about the modifications?

reviewed the blueprint for modifications to extant mechanisms, not just new ones.

**genMockedBrainAtom modification**

blueprint adds `calls?: { tools: BrainPlugToolInvocation[] } | null` parameter.

question: is there a separate factory for brain outputs with calls? could we avoid modification of genMockedBrainAtom?

answer: no. genMockedBrainAtom returns a mock atom, not output. the calls parameter enables tests that mock atoms to return tool invocations. this is the right place for it.

**AsBrainOutputCallsFor modification**

blueprint adds TGrain parameter to extant conditional type.

question: does this break extant usages of AsBrainOutputCallsFor?

answer: no. the type is internal - callers use BrainAtom and BrainRepl, which supply the grain. direct usage of AsBrainOutputCallsFor is not expected.

**no issues found with modifications**

modifications extend extant mechanisms without breakage.
