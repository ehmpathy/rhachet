
atom tool:
```ts
const stripeCustomerLookup: BrainPlugToolDefinition<{ email: string }, Customer, 'atom'> = {
  slug: 'stripe-customer-lookup',
  name: 'Stripe Customer Lookup',
  description: 'lookup a customer by email in stripe',
  schema: { input: z.object({ email: z.string().email() }) },
  // no execute - atoms don't use it
};
```

### actual implementation (BrainPlugToolDefinition.ts:59-71)

```ts
export type BrainPlugToolDefinition<
  TInput = unknown,
  TOutput = unknown,
  TGrain extends BrainGrain = BrainGrain,
> = TGrain extends 'repl'
  ? BrainPlugToolDefinitionBase<TInput> & {
      execute: BrainPlugToolExecute<TInput, TOutput>;
    }
  : BrainPlugToolDefinitionBase<TInput>;
```

### trace

1. when `TGrain = 'repl'`: conditional evaluates `'repl' extends 'repl'` = true → returns `Base & { execute: required }`
2. when `TGrain = 'atom'`: conditional evaluates `'atom' extends 'repl'` = false → returns `BrainPlugToolDefinitionBase<TInput>` (no execute)

**why it holds**: the conditional type produces exactly what vision describes - repl has required execute, atom omits it entirely.

### type test verification (BrainAtom.types.test.ts:500-543)

```ts

// @ts-expect-error - 'execute' does not exist on atom tool definition

// repl tool: execute is present
```

the type tests verify the conditional behavior at compile time.

```ts
export type AsBrainOutputCallsFor<
  TPlugs extends BrainPlugs,
  TGrain extends BrainGrain = BrainGrain,
> = TGrain extends 'repl'
  ? null // repls execute tools internally, never return calls
  : TGrain extends 'atom'
    ? TPlugs extends { tools: BrainPlugToolDefinition[] }
      ? { tools: BrainPlugToolInvocation[] } | null
      : null
    : // fallback for unknown grain
      TPlugs extends { tools: BrainPlugToolDefinition[] }
      ? { tools: BrainPlugToolInvocation[] } | null
      : null;
```

### comparison

| grain | vision spec | actual | match? |
|-------|-------------|--------|--------|
| repl | null | null (line 33) | yes |

| atom no tools | null | null (line 37) | yes |
| unknown grain | not specified | fallback to `{ tools } \| null` | extension |

**why it holds**: vision spec is implemented exactly. the fallback branch handles edge cases vision did not specify, but does not contradict the spec.
