# self-review: has-pruned-backcompat

## findings

### pruned: dead third branch in BrainPlugToolDefinition

**what was there**:

```ts
TGrain extends 'repl'
  ? Base & { execute: required }     // branch 1: repl
  : TGrain extends 'atom'
    ? Base                           // branch 2: atom (no execute)
    : Base & { execute?: optional }  // branch 3: fallback ← DEAD CODE
```

**why it was backwards compat**:
- branch 3 was added "to be safe" for cases where TGrain might be neither 'atom' nor 'repl'
- but BrainGrain = 'atom' | 'repl' — exactly two values
- with TypeScript conditional distribution over unions, branch 3 is NEVER reachable
- no code path exists where TGrain extends BrainGrain but is neither 'atom' nor 'repl'

**action taken**:
- removed branch 3 entirely
- simplified to two-branch conditional: repl → execute required, else → no execute
- verified: `npm run test:types` passes

**what remains**:

```ts
TGrain extends 'repl'
  ? Base & { execute: required }     // repl: execute required
  : Base;                            // atom: no execute
```

### verified: no other backwards compat

reviewed each component:
