## reference 1: anthropic tool use documentation

**url:** https://platform.claude.com/docs/en/agents-and-tools/tool-use/implement-tool-use

**terms established:**
- `tool_use` — content block type for model tool requests

**key concept:** tool results are content blocks appended to messages, correlated via id

## concepts we leverage

| concept | from | usage in design |
|---------|------|-----------------|
| exid correlation | anthropic/openai | `execution.exid` matches `invocation.exid` |
| error signal | anthropic `is_error` | `signal: 'success' \| 'error:constraint' \| 'error:malfunction'` |
| role-based messages | openai | prompt union mirrors message role distinction |
| error type semantics | helpful-errors | ConstraintError (4xx) vs MalfunctionError (5xx) |
| parallel execution | agentic-patterns | array of executions in prompt continuation |
| termination condition | google cloud | output.complete or schema fulfillment |
| immutable domain objects | domain-objects | typed contract with generics |
| duration format | ISO 8601 | `metrics.cost.time: IsoDuration` |

---

## how references inform design

1. **correlation via id** — both anthropic and openai use unique ids for tool call correlation. we adopt `exid` as our universal correlation key.

2. **error as signal** — anthropic's `is_error` boolean is limited. we expand to three-way signal for richer semantics.

3. **typed contracts** — domain-objects patterns inform our generic `BrainPlugToolExecution<TInput, TOutput>` design.

4. **standardized metrics** — ISO 8601 duration format ensures metrics are portable and comparable.

5. **parallel support** — agentic patterns validate our array-based execution return design.
