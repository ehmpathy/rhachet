# playtest: brain plug tool operation

## prerequisites

- node.js installed
- repo cloned and dependencies installed (`pnpm install --frozen-lockfile`)
- `npm run build` passes

---

## sandbox

- all test files already exist in `src/`
- no file operations required for playtest
- this is a type-level and domain object behavior - verification is via code inspection and test execution

## edgey paths

### edge 1: type tests use function expressions (not block scopes)

**action:**
```bash
grep -n "^() => {" src/domain.objects/BrainAtom.types.test.ts | head -10
```

**expected outcome:**
- type test blocks are wrapped in function expressions `() => { ... }`
- NOT block scopes `{ ... }` which would crash at runtime

---

### edge 2: BrainPlugToolDefinition distinguishes atom vs repl

**action:**
```bash
grep -A10 "BrainPlugToolDefinition" src/domain.objects/BrainPlugToolDefinition.ts | head -20
```

## pass/fail criteria

### pass if:
- [x] all happy path steps produce expected outcomes
- [x] all edge path steps produce expected outcomes
- [x] `npm run test` exits with code 0
- [x] no type errors in `npm run test:types`

### fail if:
- [ ] any test suite fails
- [ ] type errors exist
- [ ] BrainPlugToolExecution is not exported from sdk
- [ ] type tests crash at runtime (block scope instead of function expression)
