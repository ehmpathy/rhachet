## the constraint

**the bottleneck: type system iteration**

for this feature, the main constraint is correct conditional types:
- `AsBrainOutputCallsFor<TPlugs, TGrain>` — must consider both plugs and grain
- `BrainPlugToolDefinition<TInput, TOutput, TBrainGrain>` — execute field conditional on grain
- prompt union `string | BrainPlugToolExecution[]` — type inference for continuation

compile-time type tests catch errors fast (~2s), but iteration may require multiple attempts to achieve correct generic constraints.

**fix:** iterate in small batches with type tests. each type change runs against compile-time tests immediately.
