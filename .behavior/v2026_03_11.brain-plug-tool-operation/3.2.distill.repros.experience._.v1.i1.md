## reproduction feasibility

### usecase.1: single tool call (atom)

**utilities available:**
- `genMockedBrainAtom` — extend to return tool invocations
- `genMockedBrainPlugToolInvocation` — new factory for invocations
- `genMockedBrainPlugToolExecution` — new factory for executions

**setup required:**
- mocked brain atom that returns invocations on first ask, output on second ask
- sample tool definition without execute

**test sketch:**
```ts
given('[case1] a brain atom with tools plugged', () => {
  const tool: BrainPlugToolDefinition<{ email: string }, Customer, 'atom'> = {
    slug: 'lookup-customer',
    description: 'lookup customer by email',
    schema: { input: z.object({ email: z.string() }) },
  };

  const invocation = genMockedBrainPlugToolInvocation({
    slug: 'lookup-customer',
    input: { email: 'test@example.com' }
  });

  const atom = genMockedBrainAtom({
    calls: { tools: [invocation] },  // first ask returns invocation
  });

  when('[t0] .ask() returns tool invocations', () => {
    then('calls.tools contains invocations', async () => {
      const result = await atom.ask({ prompt: 'lookup customer', plugs: { tools: [tool] } });
      expect(result.calls?.tools).toHaveLength(1);
      expect(result.calls?.tools[0].exid).toEqual(invocation.exid);
    });
  });
});
```

**feasible:** yes

**test sketch:**
```ts
/**
 * type test: generics flow from invocation to execution
 */
async () => {
  const invocation: BrainPlugToolInvocation<{ email: string }> = {
    exid: 'test',
    slug: 'test',
    input: { email: 'test@example.com' },
  };

  const execution: BrainPlugToolExecution<{ email: string }, { id: string }> = {
    exid: invocation.exid,
    slug: invocation.slug,
    input: invocation.input,
    output: { id: 'cus_123' },
    signal: 'success',
    metrics: { cost: { time: 'PT0.1S' } },
  };

  // positive: input type flows through
  const _email: string = execution.input.email;

  // negative: wrong type fails
  // @ts-expect-error - output.id is string, not number
  const _id: number = execution.output.id;
};
```

**feasible:** yes

### usecase.10: prompt union semantics

**utilities available:**
- compile-time type tests pattern

**setup required:**
- type-level test for prompt union

**test sketch:**
```ts
/**
 * type test: prompt accepts string or execution array
 */
async () => {
  declare const atom: BrainAtom;
  const tools: BrainPlugToolDefinition[] = [];

  // positive: string prompt
  await atom.ask({ prompt: 'hello', plugs: { tools } });

  // positive: execution array prompt
  const execution = genMockedBrainPlugToolExecution();
  await atom.ask({ prompt: [execution], plugs: { tools } });

  // negative: single execution (not array) fails when tools plugged
  // @ts-expect-error - prompt must be string or BrainPlugToolExecution[]
  await atom.ask({ prompt: execution, plugs: { tools } });
};
```

**feasible:** yes
