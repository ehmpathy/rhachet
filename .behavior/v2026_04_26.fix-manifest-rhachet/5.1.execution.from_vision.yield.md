## implementation details

### regex pattern

```regexp
/\b(npx|bunx)\b.*?\b(rhachet|rhx)\b/
```

catches:
- `npx rhachet ...`
- `npx rhx ...`
- `npx --yes rhachet ...` (flags between npx and command)
- `bunx rhachet ...`
- `bunx rhx ...`

does NOT catch (allowed):
- `pnpm exec rhachet ...`
- `yarn exec rhachet ...`
- `./node_modules/.bin/rhachet ...`
- `rhachet ...` (global)

### error format

```
✋ hooks with forbidden npx patterns
   │
   ├─ these hooks use npx which adds 500ms-2s latency per invocation:
   │
   ├─ mechanic
   │  ├─ hook: onBrain.onBoot[0]
   │  ├─ command: npx rhachet roles boot --role mechanic
   │  └─ hint: use './node_modules/.bin/rhachet roles boot --role mechanic'
   │
   └─ fix: replace 'npx rhachet' with './node_modules/.bin/rhachet' or 'rhachet' (global)
```
