# blocker.1: failhide: catch swallows unexpected errors in JSON parsing

**rule**: .agent/repo=bhuild/role=behaver/role=architect/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/mechAdapterGithubApp.ts:34

The try/catch block in parseGithubAppCredentials catches all errors from JSON.parse without allowlisting expected errors or rethrowing unexpected ones. JSON.parse may throw SyntaxError for invalid JSON, but other unexpected errors (e.g., TypeError if value is not a string) would be silently hidden, violating failfast principles. To fix, catch only SyntaxError or rethrow other errors.

**snippet**:
```ts

    return { valid: false, reasons: ['value is not valid json'] };
  }
```
