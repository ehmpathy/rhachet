# blocker.1: Flow friction in acquireForSet - invalid user input not clearly handled

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.friction-hazards.md

**locations**:
- src/domain.operations/keyrack/adapters/mechanisms/mechAdapterGithubApp.ts:129-140

The acquireForSet method in mechAdapterGithubApp prompts the user to select an organization or app, but if the user enters an invalid choice (e.g., non-numeric or out-of-range), it silently falls back to selecting the first option. This creates flow friction as the user is not informed of the invalid input or given a chance to correct it, violating the requirement for clear communication of blocked states. Additionally, this error path is not acceptance tested or snapped, compounding the friction hazard.
