# blueprint.product: keyrack ephemeral and 1password vaults

## summary

enable two vault types:
1. **os.daemon** — ephemeral storage in daemon memory, no disk persistence
2. **1password** — remote vault via `op` cli, stores exid pointer in host manifest

both vaults already exist in the codebase but are incomplete:
- os.daemon: exists but not exported, needs EPHEMERAL_VIA_SESSION mech
- 1password: exists but set() throws, needs exid prompt and roundtrip validation

## codepath tree

### os.daemon vault

```
keyrack set --key K --vault os.daemon --env E
├─ [○] parseCliArgs()
├─ [○] genContextKeyrack()
├─ [~] setKeyrackKey()
│  ├─ [○] promptHiddenInput()                   # prompt for secret
│  ├─ [~] vaultAdapterOsDaemon.set()
│  │  ├─ [←] daemonAccessUnlock()               # reuse daemon sdk
│  │  └─ [~] return mech=EPHEMERAL_VIA_SESSION
│  ├─ [○] write host manifest                   # retain: vault=os.daemon so unlock knows where to look
│  └─ [○] write repo manifest                   # retain: repo still requires this key
└─ [○] outputResult()

keyrack unlock --key K --env E (where K is os.daemon)
├─ [○] unlockKeyrackKeys()
├─ [~] vaultAdapterOsDaemon.get()
│  ├─ [○] check daemon for key
│  ├─ [+] if found → return secret ("already unlocked")
│  └─ [+] if absent → return null (daemon restarted, key gone)
├─ [+] if absent → report "absent" with re-set hint
└─ [○] outputResult()
```

### 1password vault

```
keyrack set --key K --vault 1password --env E
├─ [○] parseCliArgs()
├─ [○] genContextKeyrack()
├─ [~] setKeyrackKey()
│  ├─ [~] vaultAdapter1Password.set()
│  │  ├─ [+] isOpCliInstalled()                 # encapsulated: check op availability
│  │  │  └─ [+] throw constraint error if absent # exit 2 with install instructions
│  │  ├─ [+] promptVisibleInput() for exid      # prompt for op://uri (visible, not secret)
│  │  ├─ [+] validateRoundtrip()                # op read $exid
│  │  └─ [+] return { exid }
│  ├─ [○] write host manifest                   # stores vault=1password, exid
│  └─ [○] write repo manifest                   # non-sudo
└─ [○] outputResult()

keyrack unlock --key K --env E (where K is 1password)
├─ [○] unlockKeyrackKeys()
├─ [~] vaultAdapter1Password.get()
│  ├─ [○] execOp(['read', exid])                # retain: fetches secret
│  ├─ [+] if op read succeeds → return secret
│  └─ [+] if op read fails (item gone) → return null
├─ [+] if absent → report "absent" with hint (item may have been deleted)
├─ [○] daemonAccessUnlock()                     # send to daemon (if found)
└─ [○] outputResult()
```

### op cli check (encapsulated in 1password adapter)

```
isOpCliInstalled()                              # called by vaultAdapter1Password.set()
├─ [+] execa('which', ['op'])
├─ [+] return true if found
└─ [+] return false if not

vaultAdapter1Password.set() entry
├─ [+] if !isOpCliInstalled()
│  ├─ [+] display ubuntu install instructions
│  └─ [+] exit 2 (constraint error)
└─ [+] proceed with exid prompt and validation
```

```typescript
type KeyrackGrantMechanism =
  | 'PERMANENT_VIA_REPLICA'      // extant: os.secure, os.direct
  | 'EPHEMERAL_VIA_SESSION'      // [~] os.daemon (session-scoped, in daemon memory)
  | 'EPHEMERAL_VIA_GITHUB_APP'   // extant

  | 'EPHEMERAL_VIA_GITHUB_OIDC'  // extant
```

```typescript
interface KeyrackHostVaultAdapter {

  get: (input) => Promise<string | null>;
  set: (input) => Promise<void | { exid: string }>;  // note: can return exid
```

## error messages

### op cli not installed

```
🔐 keyrack set
   └─ ✗ op cli not found

   to install on ubuntu:

   1. add 1password gpg key and apt repository
      (see 1password docs: https://developer.1password.com/docs/cli/get-started/)

   2. install op cli:
      sudo apt update && sudo apt install 1password-cli

   3. verify installation:
      op --version

   4. authenticate:
      - preferred: enable "integrate with 1password cli" in 1password app settings
      - alternative: op signin
```

### invalid exid

```
🔐 keyrack set
   └─ ✗ invalid 1password reference: op read failed

   the exid you entered could not be read from 1password.

   verify:
   - exid format is op://vault/item/field
   - item exists in 1password
   - op cli is authenticated (run: op whoami)
```
