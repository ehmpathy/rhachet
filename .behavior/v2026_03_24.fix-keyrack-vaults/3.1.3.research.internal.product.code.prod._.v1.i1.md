# research.internal.product.code.prod: keyrack vault adapters

## summary

this research identifies production code patterns relevant to keyrack os.daemon and 1password vault implementation. patterns are marked as [REUSE], [EXTEND], or [REPLACE].

---

## pattern.1: vault adapter interface [REUSE]

**file:** `src/domain.objects/keyrack/KeyrackHostVaultAdapter.ts`

the vault adapter interface defines 5 core operations:

```typescript
export interface KeyrackHostVaultAdapter {
  unlock: (input: { identity: string | null; silent?: boolean; exid?: string | null }) => Promise<void>;
  isUnlocked: (input?: { exid?: string | null }) => Promise<boolean>;
  get: (input: { slug: string; exid?: string | null; vaultRecipient?: string | null; owner?: string | null }) => Promise<string | null>;
  set: (input: { slug: string; env: string; org: string; exid?: string | null; expiresAt?: string | null; vaultRecipient?: string | null; recipients?: KeyrackKeyRecipient[]; owner?: string | null }) => Promise<void | { exid: string }>;
  del: (input: { slug: string; exid?: string | null; owner?: string | null }) => Promise<void>;
  relock?: (input: { slug: string; exid?: string | null }) => Promise<void>;
}
```

**keyrack leverage:** both os.daemon and 1password adapters implement this interface. no changes needed.

## pattern.2: os.daemon vault adapter [EXTEND]

**file:** `src/domain.operations/keyrack/adapters/vaults/vaultAdapterOsDaemon.ts`

the os.daemon adapter already exists but is **not exported** from the index.

```typescript
export const vaultAdapterOsDaemon: KeyrackHostVaultAdapter = {
  unlock: async () => {
    // noop: daemon validates caller on each request
  },
  isUnlocked: async () => {
    return isDaemonReachable({});
  },
  get: async (input) => {
    const result = await daemonAccessGet({ slugs: [input.slug] });
    if (!result) return null;
    const keyEntry = result.keys.find((k) => k.slug === input.slug);
    return keyEntry?.key.secret ?? null;
  },
  set: async (input) => {
    const secret = await promptHiddenInput({
      prompt: `enter secret for ${input.slug}: `,
    });
    const grade = inferKeyGrade({ vault: 'os.daemon', mech: 'PERMANENT_VIA_REPLICA' });
    const defaultTtlMs = 9 * 60 * 60 * 1000;
    const expiresAt = asIsoTimeStamp(new Date(Date.now() + defaultTtlMs));
    await daemonAccessUnlock({
      keys: [{ slug: input.slug, key: { secret, grade }, source: { vault: 'os.daemon', mech: 'PERMANENT_VIA_REPLICA' }, env: input.env, org: input.org, expiresAt }],
    });
  },
  del: async (input) => {
    await daemonAccessRelock({ slugs: [input.slug] });
  },
};
```

**needed changes:**
1. export from `adapters/vaults/index.ts`
2. register in context generators
3. update mech to `EPHEMERAL_SESSION` (new mechanism type)

## pattern.3: 1password vault adapter [EXTEND]

**file:** `src/domain.operations/keyrack/adapters/vaults/vaultAdapter1Password.ts`

the 1password adapter already exists but is **not exported** from the index.

```typescript
export const vaultAdapter1Password: KeyrackHostVaultAdapter = {
  unlock: async () => {
    // noop: 1password cli handles auth via biometric or service account token
  },
  isUnlocked: async () => {
    try {
      await execOp(['whoami']);
      return true;
    } catch {
      return false;
    }
  },
  get: async (input) => {
    if (!input.exid) throw new UnexpectedCodePathError('1password vault requires exid', { input });
    const { stdout } = await execOp(['read', input.exid]);
    return stdout.trim();
  },
  set: async (input) => {
    throw new UnexpectedCodePathError('1password vault does not support set via keyrack', { input });
  },
  del: async (input) => {
    throw new UnexpectedCodePathError('1password vault does not support del via keyrack', { input });
  },
};
```

**issue:** `set` throws error. per vision, set should:
1. prompt for exid (secret reference URI)
2. validate roundtrip via `op read $exid`
3. store pointer in host manifest

**needed changes:**
1. export from `adapters/vaults/index.ts`
2. register in context generators
3. implement `set` to prompt for exid and validate roundtrip
4. add op CLI installation check with ubuntu install instructions
