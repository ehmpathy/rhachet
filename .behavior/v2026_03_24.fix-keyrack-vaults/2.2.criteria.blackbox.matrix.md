# criteria.blackbox.matrix: keyrack ephemeral and 1password vaults

## matrix.1 = `keyrack set` behavior

| ind: vault | ind: op cli state | ind: exid valid | dep: prompt | dep: storage | dep: exit code |
|------------|-------------------|-----------------|-------------|--------------|----------------|
| os.daemon | n/a | n/a | secret | daemon memory | 0 |
| 1password | not installed | n/a | none | none | 2 |
| 1password | installed, not authed | n/a | none | none | 2 |
| 1password | installed, authed | invalid | exid | none | 2 |
| 1password | installed, authed | valid | exid | host manifest | 0 |

**notes:**
- os.daemon has no op cli dependency
- 1password validates roundtrip before storage
- exit code 2 = constraint error (user must fix)

---

## matrix.2 = `keyrack unlock` behavior

| ind: vault | ind: key state | ind: auth method | dep: action | dep: result | dep: exit code |
|------------|----------------|------------------|-------------|-------------|----------------|
| os.daemon | in daemon | n/a | none | "already unlocked" | 0 |
| os.daemon | absent | n/a | none | hint to set | 0 |
| 1password | in manifest | biometric | op read | secret in daemon | 0 |
| 1password | in manifest | service account | op read | secret in daemon | 0 |
| 1password | in manifest, exid invalid | any | op read | error from op | 1 |
| 1password | absent | n/a | none | hint to set | 0 |

**notes:**
- os.daemon keys are already in daemon; unlock is a no-op
- 1password keys are fetched from remote on unlock
- exit code 1 = malfunction (transient or config issue)

## matrix.3 = `keyrack get` behavior

| ind: key state | dep: result | dep: hint |
|----------------|-------------|-----------|
| granted in daemon | secret value | none |
| locked (not unlocked) | absent | "unlock" |
| expired | absent | "re-set or re-unlock" |
| never set | absent | "set" |

**notes:**
- get only reads from daemon
- get does not care about vault type; daemon is uniform

## matrix.4 = 1password auth modes

| ind: auth context | ind: OP_SERVICE_ACCOUNT_TOKEN | dep: auth prompt | dep: human interaction |
|-------------------|-------------------------------|------------------|------------------------|
| interactive (tty) | absent | biometric/password | required |
| interactive (tty) | present | none | not required |
| ci (no tty) | absent | none | fails |

**notes:**
- service account token enables headless operation
- requires op cli >= 2.18.0
