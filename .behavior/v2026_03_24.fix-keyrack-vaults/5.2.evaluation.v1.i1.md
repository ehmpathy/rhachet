## divergences

### D1: deprecated aliases retained

| blueprint | actual | resolution |
|-----------|--------|------------|
| "remove deprecated aliases" | retained REPLICA, REFERENCE, GITHUB_APP, AWS_SSO | rationale: backwards compat with extant host manifests |

**rationale:** extant host manifests may use deprecated mech names. removing them would break unlock for those keys. aliases are marked @deprecated in docstring; new code uses canonical names.

### D2: 1password test fixture uses REFERENCE alias

| blueprint | actual | resolution |
|-----------|--------|------------|
| mech=PERMANENT_VIA_REFERENCE | mech=REFERENCE in fixture | rationale: demonstrates backwards compat works |

**rationale:** test fixture intentionally uses deprecated alias to validate backwards compatibility. acceptance tests confirm REFERENCE alias works correctly with 1password vault.

### D3: os.daemon writes to host manifest (clarification)

| blueprint | actual | resolution |
|-----------|--------|------------|
| "write host manifest" | writes vault=os.daemon pointer | matches blueprint |

**clarification:** this is NOT a divergence from blueprint. blueprint explicitly says "[✓] write host manifest" with comment "retain: vault=os.daemon so unlock knows where to look". the vision doc explored "no manifest entry" but the blueprint resolved this: manifest tracks vault location (where to look), secret lives in daemon only (no disk). blueprint and implementation match.
