# research.external.product.access: keyrack 1password integration

## summary

this research examines remote access requirements for keyrack's 1password vault integration. the primary external dependency is the 1password CLI (`op`).

---

## remote repositories

### 1password CLI (`op`)

**what:** command-line tool for access to 1password vaults
**interface:** cli binary with json output
**contract:** secret references in `op://vault/item/field` format

---

## lesson.1: secret reference uri format

the 1password CLI uses a structured URI format for secret references.

> "A secret reference is a URI that looks like this: `op://<vault-name>/<item-name>/[section-name/]<field-name>`"
> — [1] 1Password Developer Docs, Secret Reference Syntax [official]

> "To replace secret references with the secrets they refer to at runtime, use `op read`."
> — [2] 1Password Developer Docs, Secret References [official]

**convergence:** sources [1] and [2] agree on the URI format and `op read` as the access method.

---

## lesson.2: op read command

the `op read` command retrieves secrets from 1password.

> "To retrieve the contents of a specific field, use `op read`."
> — [3] 1Password Developer Docs, read command reference [official]

> "You can use `op read` with secret references directly in your executable or with environment variables. With this method, secrets are only passed to the single command that includes the secret reference."
> — [4] 1Password Developer Docs, Load secrets into executables [official]

**best practice:** use `op read` for single-value retrieval; this aligns with keyrack's unlock flow.

---

## lesson.3: service account authentication

service accounts enable non-interactive authentication for CI/CD.

> "Service Accounts require 1Password CLI version 2.18.0 or later."
> — [5] 1Password Developer Docs, Use service accounts with CLI [official]

> "Each service account has a service account token that you can provide as an environment variable for authentication."
> — [6] 1Password Developer Docs, Service Accounts overview [official]

> "You can run supported 1Password CLI commands authenticated as the service account."
> — [5] 1Password Developer Docs, Use service accounts with CLI [official]

**convergence:** sources [5] and [6] confirm `OP_SERVICE_ACCOUNT_TOKEN` as the auth mechanism.

---

## lesson.4: biometric app integration

the 1password desktop app enables biometric authentication for CLI.

> "You can use the 1Password desktop app integration to quickly and securely sign in to 1Password CLI. You can authenticate 1Password CLI the same way you unlock your device, like with your fingerprint, face, Apple Watch, Windows Hello PIN, or device user password."
> — [7] 1Password Developer Docs, App Integration [official]

> "1Password accounts can only be accessed through 1Password CLI after the user provides explicit biometric authorization and authorization is limited to a single account at a time."
> — [8] 1Password Developer Docs, App Integration Security [official]

**security note:** biometric auth provides both security and ux benefits.

---

## lesson.5: op cli exit codes

the CLI uses numeric exit codes for error categorization.

> "1Password CLI now correctly exits with code 1 instead of 0 when encountering server error codes that are not recognized by the CLI."
> — [9] 1Password CLI Release Notes [official]

> "Currently, it seems like only two exit codes are used (0 and 1), where 0 indicates success and 1 indicates a failure."
> — [10] 1Password Community, exit codes discussion [practitioner]

**conflict:** limited exit code granularity. keyrack must interpret error messages for categorization between constraint (exit 2) vs malfunction (exit 1).

---

## lesson.6: linux installation via apt

ubuntu installation uses the apt repository.

> "To install 1Password CLI using APT on Debian- and Ubuntu-based distributions, you need to add the key for the 1Password apt repository."
> — [11] 1Password Developer Docs, CLI Get Started [official]

> "Add the GPG key: `curl -sS https://downloads.1password.com/linux/keys/1password.asc | sudo gpg --dearmor --output /usr/share/keyrings/1password-archive-keyring.gpg`"
> — [12] 1Password Support, Install Linux [official]

**best practice:** use apt for automatic updates rather than manual binary install.

---

## lesson.7: subprocess security with op run

`op run` provides secure subprocess execution with secrets as env vars.

> "1Password CLI loads the specified secrets, then runs the provided command in a subprocess with the secrets made available as environment variables only for the duration of the process."
> — [13] 1Password Developer Docs, run command [official]

> "If a subprocess used with `op run` prints a secret to stdout, the secret will be concealed by default."
> — [14] 1Password Developer Docs, Secrets Environment Variables [official]

**anti-pattern:** do NOT export secrets to shell env globally. use `op run` or fetch-and-use pattern.

---

## lesson.8: connect server vs cli tradeoffs

1password offers both CLI and Connect Server for different use cases.

> "Connect Server is recommended for automated pipelines and servers, while the CLI is more suited for individual developers."
> — [15] 1Password Community, Connect Server discussion [practitioner]

> "A 1Password Connect Server allows you to securely access items in your company's apps and cloud infrastructure using a private REST API."
> — [16] 1Password Developer Docs, Connect Get Started [official]

**decision:** keyrack uses CLI, not Connect Server. CLI is simpler for local dev and CI, which are the target use cases.

---

## lesson.9: secrets management anti-patterns

**what NOT to do:**

> "Storing secrets in plain text makes it much easier for an attacker to get into the target system."
> — [17] OWASP Secrets Management Cheat Sheet [academic]

> "Storing secrets in Git repositories is the most dangerous anti-pattern—a developer might commit a Kubernetes Secret to a repository because 'it is just the prep password'."
> — [18] Cycode, Secrets Management Best Practices [practitioner]

> "In 2023, GitHub reported scanning over 1.4 billion commits and catching over 15 million secrets."
> — [18] Cycode, Secrets Management Best Practices [practitioner]

**convergence:** sources [17] and [18] agree: never persist secrets in version control.

---

## lesson.10: ephemeral credentials best practices

ephemeral credentials reduce attack surface.

> "Systems should automatically request credentials with a defined time for expiration rather than relying on static keys."
> — [19] Doppler, Secrets Management for Ephemeral Environments [practitioner]

> "Just-in-time dynamic secrets create temporary, automatically expiring secrets for ephemeral access, ensuring Zero Standing Privileges."
> — [20] Akeyless, Top Secrets Management Tools [practitioner]

> "Dynamic secrets eliminate vulnerability windows by generating credentials on-demand with automatic expiration."
> — [21] Infisical, Secrets Management Best Practices [practitioner]

**convergence:** sources [19], [20], and [21] agree: ephemeral > persistent. this validates keyrack's daemon-based ephemeral storage model.

---

## lesson.11: principle of least privilege

scope access to only required secrets.

> "Follow the principle of least privilege with 1Password Service Accounts, which support restricting CLI access to only the items required."
> — [22] 1Password Developer Docs, CLI Best Practices [official]

> "Authenticate with a 1Password Service Account to follow the principle of least privilege. You can scope service account access to specific vaults."
> — [14] 1Password Developer Docs, Secrets Environment Variables [official]

**best practice:** use scoped service accounts, not full-access tokens.

---

## lesson.12: field types and item structure

1password items have structured field types.

> "Use the fieldType with assignment statements and the type with an item JSON template."
> — [23] 1Password Developer Docs, Item Fields [official]

> "To create a new item in your 1Password account and assign information to it, use the `op item create` command."
> — [24] 1Password Developer Docs, Create Items [official]

**note:** keyrack only reads from 1password (refed vault), does not create items.
