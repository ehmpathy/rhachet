# criteria.blackbox: keyrack ephemeral and 1password vaults

## usecase.1 = os.daemon: ephemeral key storage

### episode: set → get → expire

```
given(user wants ephemeral key storage)
  when(user runs `keyrack set --key API_KEY --vault os.daemon --env prod`)
    then(prompts for secret via stdin)
      sothat(secret never appears in bash history)
    then(stores key in daemon memory only)
      sothat(no disk persistence occurs)
    then(displays confirmation with vault=os.daemon and mech=EPHEMERAL_VIA_SESSION)
    then(displays expiration time)

  when(user runs `keyrack get --key API_KEY --env prod`)
    then(returns the secret value from daemon)

  when(session ends or 9h passes)
    then(key is no longer accessible)
      sothat(ephemeral keys die with session)
```

### exchange: set with os.daemon vault

```
given(daemon is active)
  when(set --key K --vault os.daemon --env E)
    then(prompts "enter secret for K:")
    then(returns mech=EPHEMERAL_VIA_SESSION)
    then(returns vault=os.daemon)
    then(returns expiration time)

given(daemon is not active)
  when(set --key K --vault os.daemon --env E)
    then(auto-starts daemon)
    then(proceeds with set)
```

### exchange: unlock with os.daemon keys

```
given(keys were set directly to os.daemon)
  when(user runs `keyrack unlock --env prod`)
    then(reports os.daemon keys as "already unlocked")
      sothat(no re-prompt occurs for keys already in daemon)

given(host manifest has os.daemon entry)
  given(daemon was restarted and key is gone)
    when(user runs `keyrack unlock --key K --env E`)
      then(reports key as "absent")
      then(hint suggests re-set via `keyrack set --key K --vault os.daemon --env E`)
        sothat(user knows how to restore ephemeral key)
```

## usecase.2 = 1password: remote source of truth

### episode: set → unlock → get

```
given(user wants 1password as source of truth)
  given(op cli is installed and authenticated)
    when(user runs `keyrack set --key API_KEY --vault 1password --env prod`)
      then(prompts for 1password uri)
      then(validates roundtrip via `op read $exid`)
        sothat(broken exids fail fast at set time, not unlock time)
      then(stores pointer in host manifest)
        sothat(secret is never written to keyrack storage)
      then(displays confirmation with vault=1password and mech=PERMANENT_VIA_REFERENCE)

    when(user runs `keyrack unlock --env prod`)
      then(1password prompts for auth)
        sothat(biometric or master password secures access)
      then(fetches secret from 1password)
      then(caches secret in daemon)
      then(displays expiration time)

    when(user runs `keyrack get --key API_KEY --env prod`)
      then(returns secret from daemon)

    when(1password rotates the key)
      when(next session starts and user runs `keyrack unlock --env prod`)
        then(fetches fresh value from 1password)
          sothat(rotations are picked up automatically)
```

### exchange: set with 1password vault

```
given(op cli is installed)
  given(op cli is authenticated)
    when(set --key K --vault 1password --env E)
      then(prompts "enter 1password uri (e.g., op://vault/item/field):")
      then(validates via `op read $exid`)
      then(stores exid in host manifest)
      then(returns mech=PERMANENT_VIA_REFERENCE)
      then(returns vault=1password)
      then(returns exid)
```

### exchange: unlock with 1password vault

```
given(host manifest has 1password entry with exid)
  when(unlock --env E)
    then(invokes op cli to read secret)
    then(caches secret in daemon)
    then(returns expiration time)

given(host manifest has 1password entry with exid)
  given(item was deleted from 1password)
    when(unlock --key K --env E)
      then(op cli returns not found)
      then(reports key as "absent")
      then(hint suggests item may have been deleted from 1password)
        sothat(user knows to check 1password or re-set)
```

## usecase.3 = 1password: ci with service accounts

### episode: ci job pulls secrets

```
given(ci job has OP_SERVICE_ACCOUNT_TOKEN env var)
  given(host manifest has 1password exids)
    when(ci runs `keyrack unlock --env test`)
      then(1password authenticates via service account token)
        sothat(no human interaction required)
      then(secrets flow to daemon)
      then(tests can run with secrets)
```

### exchange: unlock with service account

```
given(OP_SERVICE_ACCOUNT_TOKEN is set in environment)
  given(op cli version >= 2.18.0)
    when(unlock --env E)
      then(uses service account token for auth)
      then(no biometric or password prompt)
      then(returns secrets to daemon)
```

## boundary conditions

### os.daemon

```
given(key already exists in daemon for same slug)
  when(set --key K --vault os.daemon --env E)
    then(overwrites the previous value)
      sothat(upsert semantics apply)

given(no --env specified)
  when(set --key K --vault os.daemon)
    then(fails fast)
    then(hint suggests --env)
```

### 1password

```
given(key already exists in host manifest for same slug)
  when(set --key K --vault 1password --env E)
    then(overwrites the previous exid)
      sothat(upsert semantics apply)

given(exid points to non-existent 1password item)
  when(unlock --env E)
    then(reports key as "absent")
    then(hint suggests item may have been deleted from 1password)
      sothat(user knows to check 1password or re-set)

given(1password requires reauthentication)
  when(unlock --env E)
    then(1password prompts for auth)
    then(proceeds after auth succeeds)
```

### daemon lifecycle

```
given(daemon has expired keys)
  when(get --key K --env E)
    then(returns absent)
    then(hint suggests re-set or re-unlock)

given(daemon is not active)
  when(get --key K --env E)
    then(returns absent)
    then(hint suggests unlock)
```
