# playtest: keyrack ephemeral and 1password vaults

## prerequisites

- rhachet cli installed (`npx rhachet` or `rhx`)
- 1password cli installed (`op --version`)
- 1password cli authenticated (`op whoami`)
- a test 1password item with a credential field (or create one in this playtest)

---

## sandbox

all file operations target the daemon (ephemeral) or host manifest. no cleanup required — daemon keys expire, manifest entries can be deleted via `keyrack del`.

---

## part 1: os.daemon vault (ephemeral keys)

### happy path 1.1: set a key to daemon

```sh
./bin/rhx keyrack set --key PLAYTEST_DAEMON_KEY --vault os.daemon --env test
```

1. enter a test secret when prompted (e.g., `my-test-secret-123`)
2. verify output shows:
   - `vault: os.daemon`
   - `mech: EPHEMERAL_VIA_SESSION`
   - expiration time (e.g., `expires in: 9h`)

**pass if:** output contains `mech: EPHEMERAL_VIA_SESSION` and `vault: os.daemon`

### happy path 1.2: get the key from daemon

```sh
./bin/rhx keyrack get --key PLAYTEST_DAEMON_KEY --env test
```

**pass if:** output shows `status: granted` and your secret value

### happy path 1.3: list shows the daemon key

```sh
./bin/rhx keyrack list --env test
```

**pass if:** output includes `PLAYTEST_DAEMON_KEY` with `vault: os.daemon`

## part 2: 1password vault (refed keys)

### setup: create a test item in 1password

if you don't have a test item, create one via 1password app:
- vault: any vault you have access to
- item name: `playtest-credential`
- field: `credential` with value `op-test-secret-456`

note the uri: `op://{vault-name}/playtest-credential/credential`

### happy path 2.1: set a 1password key

```sh
./bin/rhx keyrack set --key PLAYTEST_1PASSWORD_KEY --vault 1password --env test
```

1. when prompted, enter your 1password uri (e.g., `op://Private/playtest-credential/credential`)
2. verify roundtrip validation runs (may prompt for biometric/auth)
3. verify output shows:
   - `vault: 1password`
   - `mech: PERMANENT_VIA_REFERENCE`
   - `exid: op://...`

**pass if:** output contains `mech: PERMANENT_VIA_REFERENCE` and `vault: 1password`

### happy path 2.2: unlock the 1password key

```sh
./bin/rhx keyrack unlock --key PLAYTEST_1PASSWORD_KEY --env test
```

1. 1password may prompt for authentication
2. verify output shows the key was unlocked with expiration time

**pass if:** output shows key unlocked with `expires in: ...`

### happy path 2.3: get the 1password key

```sh
./bin/rhx keyrack get --key PLAYTEST_1PASSWORD_KEY --env test
```

**pass if:** output shows `status: granted` and the secret from 1password

## part 3: edge cases

### edge 3.1: op cli not installed (simulate)

if you can temporarily rename `op` to test:

```sh
# backup op
sudo mv /usr/local/bin/op /usr/local/bin/op.bak

# attempt set
./bin/rhx keyrack set --key TEST_KEY --vault 1password --env test

# restore op
sudo mv /usr/local/bin/op.bak /usr/local/bin/op
```

**pass if:** error message includes install instructions and exits with code 2

### edge 3.2: invalid 1password exid

```sh
./bin/rhx keyrack set --key BAD_EXID_KEY --vault 1password --env test
```

enter an invalid exid: `op://nonexistent/fake/field`

**pass if:** roundtrip validation fails with clear error message

### edge 3.3: daemon key after relock

```sh
# set a daemon key
./bin/rhx keyrack set --key RELOCK_TEST_KEY --vault os.daemon --env test

# relock the keyrack
./bin/rhx keyrack relock

# attempt to get the key
./bin/rhx keyrack get --key RELOCK_TEST_KEY --env test
```

**pass if:** get returns `status: absent` with hint to re-set

## cleanup

```sh
# remove 1password pointer from host manifest
./bin/rhx keyrack del --key PLAYTEST_1PASSWORD_KEY --env test

# daemon keys expire automatically, no cleanup needed
```
