# research.internal.product.code.test: keyrack vault adapters

## summary

this research identifies test code patterns relevant to keyrack os.daemon and 1password vault implementation. patterns are marked as [REUSE], [EXTEND], or [REPLACE].

---

## pattern.1: bdd given/when/then structure [REUSE]

**files:** `src/**/*.test.ts`, `src/**/*.integration.test.ts`

tests use given/when/then from test-fns for behavior-driven structure:

```typescript
import { given, when, then, useBeforeAll } from 'test-fns';

describe('featureName', () => {
  given('[case1] scenario description', () => {
    const scene = useBeforeAll(async () => {
      return { entity: await createEntity() };
    });

    when('[t0] action is performed', () => {
      then('expected outcome', async () => {
        expect(await performAction(scene.entity)).toEqual(expectedValue);
      });
    });
  });
});
```

**keyrack leverage:** all new vault adapter tests follow this pattern. no changes needed.

## pattern.5: daemon integration test setup [REUSE]

**files:** `src/domain.operations/keyrack/daemon/**/*.integration.test.ts`

starts and stops daemon for integration tests:

```typescript
import { daemonAccessUnlock } from '../sdk/src/domain.operations/daemonAccessUnlock';
import { daemonAccessGet } from '../sdk/src/domain.operations/daemonAccessGet';
import { daemonAccessRelock } from '../sdk/src/domain.operations/daemonAccessRelock';

describe('daemon integration', () => {
  const scene = useBeforeAll(async () => {
    // daemon auto-starts on first access
    await daemonAccessUnlock({
      keys: [{ slug: 'test.key', key: { secret: 'value', grade: 'encrypted' }, ... }],
    });
    return { slug: 'test.key' };
  });

  afterAll(async () => {
    await daemonAccessRelock({ slugs: ['test.key'] });
  });

  when('[t0] get is called', () => {
    then('returns unlocked key', async () => {
      const result = await daemonAccessGet({ slugs: [scene.slug] });
      expect(result?.keys[0]?.key.secret).toEqual('value');
    });
  });
});
```

**keyrack leverage:** os.daemon adapter tests use this pattern directly.

## pattern.7: execa mock for cli tools [EXTEND]

**file:** `src/.test/mocks/mockExeca.ts`

mocks external cli invocations:

```typescript
import { execa } from 'execa';

jest.mock('execa');
const mockExeca = execa as jest.MockedFunction<typeof execa>;

given('[case1] op cli returns success', () => {
  beforeEach(() => {
    mockExeca.mockResolvedValueOnce({
      stdout: 'secret-value',
      stderr: '',
      exitCode: 0,
    } as any);
  });

  when('[t0] get is called', () => {
    then('returns secret from op', async () => {
      const result = await vaultAdapter.get({ slug, exid: 'op://vault/item/field' });
      expect(result).toEqual('secret-value');
      expect(mockExeca).toHaveBeenCalledWith('op', ['read', 'op://vault/item/field']);
    });
  });
});
```

**needed changes:** add execa mocks for 1password tests. cover success, auth failure, invalid exid cases.
