# blocker.1: Complex boolean expression for exit code extraction inline in orchestrator

**rule**: rule.forbid.inline-decode-friction.md

**locations**:
- src/contract/cli/invoke.ts

The orchestrator '_invoke' contains a complex boolean expression to extract the exit code from an error object: 'code' in error && typeof error.code === 'object' && error.code !== null && 'exit' in error.code && typeof error.code.exit === 'number'. This requires mental simulation to understand what it produces (the exit code), violating the rule against inline decode-friction in orchestrators. Orchestrators must read as narrative with only named operation calls. Extract this logic to a named transformer, such as 'getExitCodeFromError(error)', to encapsulate the computation and make the orchestrator narrative-readable.

**snippet**:
```ts
      const exitCode =
        'code' in error &&
        typeof error.code === 'object' &&
        error.code !== null &&
        'exit' in error.code &&
        typeof error.code.exit === 'number'
          ? error.code.exit
          : 1;
```
