# q2: apiKeyHelper scope — yield

## answer

**TTL interval (default 5 min) OR on HTTP 401**

## source

[Claude Code Authentication](https://code.claude.com/docs/en/authentication)

> Refresh intervals: by default, `apiKeyHelper` is called after 5 minutes or on HTTP 401 response. Set `CLAUDE_CODE_API_KEY_HELPER_TTL_MS` environment variable for custom refresh intervals.

## implications

- apiKeyHelper is NOT called on every request (not hot path)
- apiKeyHelper IS called periodically (5 min default)
- apiKeyHelper IS called on 401 (auth error)
- apiKeyHelper is NOT called on 429 (rate limit)
- rotation latency = up to TTL (configurable via `CLAUDE_CODE_API_KEY_HELPER_TTL_MS`)
- design validated: apiKeyHelper is the integration point

## critical nuance

429 (rate limit) does NOT trigger apiKeyHelper. rotation happens on next TTL check.
