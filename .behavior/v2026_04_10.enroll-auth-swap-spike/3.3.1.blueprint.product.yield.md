# blueprint: enroll-auth-swap

## summary

build auth pool rotation for brain APIs (generic, via suppliers):

1. **brains auth namespace** — new CLI subcommand for brain auth operations
2. **enrollment extension** — `--auth` flag to configure apiKeyHelper
3. **stateless parallel probe** — try all tokens in parallel, first healthy wins
4. **brains auth get operation** — returns best available token from pool

### component responsibilities

| layer | component | responsibility |
|-------|-----------|----------------|
| contract | `invokeBrainsAuth` | parse CLI args, dispatch to orchestrator, format output |
| orchestrator | `getOneBrainAuthCredentialBySpec` | coordinate parse → capacity query → token fetch |
| transformer | `asBrainAuthSpecShape` | parse spec words into structured shape |
| communicator | `asBrainAuthTokenSlugs` | expand spec source to credential slugs (keyrack i/o) |
| communicator | `BrainAuthAdapter.capacity` | query supplier for token capacity |
| communicator | `keyrack.get` | fetch actual token value |
| domain object | `BrainAuthCredential` | credential with slug and token value |
| domain object | `BrainAuthCapacity` | capacity status for a specific credential |
| domain object | `BrainAuthSupplied<TFormat>` | formatted credential output tagged with brain format |
| domain object | `BrainAuthAdapter` | adapter contract interface |

### design principles

1. **stateless** — no local state file; capacity queried fresh on each call
2. **inversion of control** — brain suppliers implement adapter, rhachet consumes
3. **fail fast** — parse errors at boundary, exhaustion errors propagate up
4. **idempotent** — same input produces same output; no side effects in queries

### domain model relationships

```
BrainAuthSpec
├─ specifies which credentials to use
├─ strategy: 'default' | 'solo' | 'pool'
└─ source: keyrack:// URI pattern
         │
         │ expands to
         ▼
BrainAuthCredential[]
├─ each credential has a slug (keyrack key)
├─ each credential has a token value
└─ credentials are stored in keyrack
         │
         │ each has
         ▼
BrainAuthCapacity
├─ capacity for a specific credential
├─ used, limit, left, refreshAt
└─ queried from brain supplier
         │
         │ selected credential becomes
         ▼
BrainAuthSupplied
├─ the credential selected by spec
├─ formatted for the specific brain
└─ ready for stdout output
```

| domain object | what it represents | relationship |
|---------------|-------------------|--------------|
| `BrainAuthSpec` | "which credentials to use" | 1 spec → N credentials (via glob) |
| `BrainAuthCredential` | "a specific auth token" | 1 credential → 1 capacity |
| `BrainAuthCapacity` | "usage status of credential" | owned by credential |
| `BrainAuthSupplied` | "selected + formatted credential" | adapter transforms credential → choice |
| `BrainAuthError` | "typed error with exit code" | thrown by operations |

### BrainAuthError.ts

```typescript
import { DomainLiteral } from 'domain-objects';

/**
 * .what: typed error for brain auth operations
 * .why: enables exit code mapping, structured error handling, and actionable hints
 */
export interface BrainAuthError {
  code: 'ALL_EXHAUSTED' | 'KEYRACK_LOCKED' | 'NO_SPEC' | 'INVALID_SPEC' | 'NO_BRAIN_ENROLLED';
  message: string;
  exitCode: 1 | 2;  // 1 = malfunction, 2 = constraint
  hint?: string;
  metadata?: Record<string, unknown>;
}

export class BrainAuthError extends DomainLiteral<BrainAuthError> implements BrainAuthError {
  /**
   * throw as early return with proper exit code
   */
  static throw(input: BrainAuthError): never {
    throw new BrainAuthError(input);
  }

  static keyrackLocked(): never {
    return BrainAuthError.throw({
      code: 'KEYRACK_LOCKED',
      message: 'keyrack is locked',
      exitCode: 2,
      hint: 'unlock with: rhx keyrack unlock',
    });
  }

  static allExhausted(metadata: { refreshAt: string | null }): never {
    return BrainAuthError.throw({
      code: 'ALL_EXHAUSTED',
      message: 'all tokens exhausted',
      exitCode: 2,
      hint: metadata.refreshAt
        ? `wait for refresh at ${metadata.refreshAt}`
        : 'add more tokens to pool',
      metadata,
    });
  }

  static noBrainSlugProvided(): never {
    return BrainAuthError.throw({
      code: 'NO_BRAIN_SLUG',
      message: 'required flag absent: --for <brainSlug>',
      exitCode: 1,
      hint: 'usage: rhx brains auth get --from <spec> --for anthropic/claude',
    });
  }

  static noAuthSpec(): never {
    return BrainAuthError.throw({
      code: 'NO_SPEC',
      message: 'no auth spec configured',
      exitCode: 1,
      hint: 'provide --from or run: rhx enroll --auth $spec',
    });
  }

  static invalidSpec(metadata: { reason: string; input: string }): never {
    return BrainAuthError.throw({
      code: 'INVALID_SPEC',
      message: `invalid auth spec: ${metadata.reason}`,
      exitCode: 1,
      metadata,
    });
  }
}
```

### asBrainAuthSpecShape.ts

```typescript
/**
 * .what: parse spec words into structured shape
 * .why: transform boundary input into typed domain object
 */
export const asBrainAuthSpecShape = (input: {
  from: BrainAuthSpecWords;
}): BrainAuthSpecShape => {
  const { from: specWords } = input;

  // parse strategy prefix: default(), solo(), pool()
  const strategyMatch = specWords.match(/^(default|solo|pool)\((.*)\)$/);
  if (!strategyMatch) {
    throw new Error(
      `invalid BrainAuthSpecWords: expected 'strategy(source)' format, got '${specWords}'`,
    );
  }

  const [, strategy, source] = strategyMatch;

  // validate source format for pool strategy
  if (strategy === 'pool') {
    if (!source.startsWith('keyrack://')) {
      throw new Error(
        `invalid pool source: expected 'keyrack://...' URI, got '${source}'`,
      );
    }
  }

  return {
    strategy: strategy as 'default' | 'solo' | 'pool',
    source: source || null,
  };
};
```

### asBrainAuthTokenSlugs.ts

```typescript
/**
 * .what: extract token slugs from spec shape
 * .why: expand glob pattern to concrete slugs for capacity lookup
 */
export const asBrainAuthTokenSlugs = async (
  input: { spec: BrainAuthSpecShape },
  context: { keyrack: ContextKeyrack },
): Promise<string[]> => {
  const { spec } = input;
  const { keyrack } = context;

  if (spec.strategy === 'default') {
    // default strategy: no explicit tokens
    return [];
  }

  if (spec.strategy === 'solo') {
    // solo strategy: single token from source
    return [spec.source!];
  }

  // pool strategy: expand keyrack:// URI glob
  // keyrack://owner/env/KEY_* → ['owner/env/KEY_1', 'owner/env/KEY_2', ...]
  const uriPath = spec.source!.replace('keyrack://', '');
  const slugs = await keyrack.glob({ pattern: uriPath });

  if (slugs.length === 0) {
    throw new Error(
      `no tokens found for pattern '${spec.source}' in keyrack`,
    );
  }

  return slugs;
};
```

### getOneBrainAuthCredentialBySpec.ts

```typescript
/**
 * .what: get one credential from pool based on spec
 * .why: orchestrate parse → capacity query → credential fetch with exhaustion handle
 */
export const getOneBrainAuthCredentialBySpec = async (
  input: { spec: BrainAuthSpecWords },
  context: {
    keyrack: ContextKeyrack;
    brainAuthAdapter: BrainAuthAdapter;
  },
): Promise<BrainAuthCredential | { exhausted: true; refreshAt: IsoTimeStamp }> => {
  const { spec: specWords } = input;
  const { keyrack, brainAuthAdapter } = context;

  // 1. parse spec words → shape
  const specShape = asBrainAuthSpecShape({ from: specWords });

  // 2. extract credential slugs from shape
  const slugs = await asBrainAuthTokenSlugs({ spec: specShape }, { keyrack });

  // 3. query capacity for all credentials
  const capacities = await brainAuthAdapter.capacity.get.all();

  // 4. filter to credentials in our pool
  const poolCapacities = capacities.filter((cap) => slugs.includes(cap.credential.slug));

  // 5. filter to credentials with left capacity
  const available = poolCapacities.filter((cap) => cap.tokens.left > 0);

  // 6. if all exhausted, return earliest refresh time
  if (available.length === 0) {
    const exhausted = poolCapacities.filter((cap) => cap.refreshAt !== null);
    const earliest = exhausted.sort((a, b) =>
      (a.refreshAt ?? '').localeCompare(b.refreshAt ?? ''),
    )[0];

    return {
      exhausted: true,
      refreshAt: earliest?.refreshAt ?? asIsoTimeStamp(new Date()),
    };
  }

  // 7. pick credential with highest left capacity (longest runway)
  const best = available.sort((a, b) => b.tokens.left - a.tokens.left)[0]!;

  // 8. fetch actual token value from keyrack
  const token = await keyrack.get({ slug: best.credential.slug });

  // 9. return credential domain object
  return new BrainAuthCredential({ slug: best.credential.slug, token });
};
```

### invokeBrainsAuth.ts

```typescript
/**
 * .what: CLI dispatcher for brains auth commands
 * .why: expose getOneBrainAuthCredentialBySpec via CLI for apiKeyHelper integration
 */
export const invokeBrainsAuth = async (
  input: { args: string[] },
  context: { stdout: NodeJS.WriteStream; stderr: NodeJS.WriteStream },
): Promise<number> => {
  const { args } = input;
  const { stdout, stderr } = context;

  // parse subcommand
  const [subcommand, ...rest] = args;

  if (subcommand !== 'get') {
    stderr.write(`unknown subcommand: ${subcommand}\n`);
    stderr.write(`usage: rhx brains auth get --from <spec> --for <brainSlug>\n`);
    return 1;
  }

  // parse --from flag (required)
  const fromIndex = rest.indexOf('--from');
  if (fromIndex === -1 || fromIndex === rest.length - 1) {
    stderr.write(`required flag absent: --from <spec>\n`);
    stderr.write(`usage: rhx brains auth get --from <spec> --for <brainSlug>\n`);
    return 1;
  }
  const specWords = rest[fromIndex + 1]!;

  // parse --for flag (required, must be after --from)
  const forIndex = rest.indexOf('--for');
  if (forIndex === -1 || forIndex === rest.length - 1) {
    stderr.write(`required flag absent: --for <brainSlug>\n`);
    stderr.write(`usage: rhx brains auth get --from <spec> --for anthropic/claude\n`);
    return 1;
  }
  const brainSlug = rest[forIndex + 1]!;

  // get adapter by explicit brain slug (no implicit discovery)
  const brainAuthAdapter = getBrainAuthAdapter({ brainSlug });

  // get keyrack context
  const keyrack = await genContextKeyrack();

  // execute
  const result = await getOneBrainAuthCredentialBySpec({ spec: specWords }, { keyrack, brainAuthAdapter });

  // output
  if ('exhausted' in result) {
    stderr.write(`all tokens exhausted, refresh at ${result.refreshAt}\n`);
    return 2; // exit 2 = constraint error
  }

  // format credential for this brain's expected output
  const choice = brainAuthAdapter.format.credential({ credential: result });
  stdout.write(choice.formatted);
  return 0;
};
```

## error handling

### error taxonomy

| error | layer | exit code | message pattern | recovery |
|-------|-------|-----------|-----------------|----------|
| invalid spec format | `asBrainAuthSpecShape` | 1 | `invalid BrainAuthSpecWords: expected 'strategy(source)' format` | fix spec syntax |
| invalid pool source | `asBrainAuthSpecShape` | 1 | `invalid pool source: expected 'keyrack://...' URI` | use keyrack:// URI |
| no tokens found | `asBrainAuthTokenSlugs` | 1 | `no tokens found for pattern '...' in keyrack` | add tokens to keyrack |
| all exhausted | `getOneBrainAuthCredentialBySpec` | 2 | `all tokens exhausted, refresh at ...` | wait for refresh or add tokens |
| keyrack locked | `keyrack.get` | 2 | `keyrack is locked` | unlock keyrack |
| token not found | `keyrack.get` | 1 | `token not found: ...` | add token to keyrack |

### exit code semantics

| code | meaning | examples |
|------|---------|----------|
| 0 | success | token returned |
| 1 | user error | invalid spec, token not found |
| 2 | constraint | exhausted (wait), locked (unlock) |

### error propagation

```
asBrainAuthSpecShape throws → invokeBrainsAuth catches → exit 1
                                                      ↓
asBrainAuthTokenSlugs throws → invokeBrainsAuth catches → exit 1
                                                       ↓
getOneBrainAuthCredentialBySpec returns exhausted → invokeBrainsAuth → exit 2
                                                      ↓
keyrack.get throws → getOneBrainAuthCredentialBySpec propagates → invokeBrainsAuth → exit 2
```

## edge cases

### edge case 1: single token pool

**scenario**: pool pattern matches exactly one token

**behavior**: works like solo — single token selected if has capacity

**test**: `asBrainAuthTokenSlugs` returns `['KEY_1']`, selection picks it

### edge case 2: equal capacity tokens

**scenario**: multiple tokens have same `left` value

**behavior**: deterministic selection (first in sort order)

**implementation**: stable sort ensures predictable choice

### edge case 3: capacity query returns subset

**scenario**: adapter.capacity.get.all() returns fewer tokens than keyrack glob

**behavior**: only consider tokens with known capacity

**rationale**: unknown capacity = cannot determine if healthy

### edge case 4: refreshAt in past

**scenario**: token.refreshAt < now but left === 0

**behavior**: return it anyway — adapter is source of truth

**rationale**: supplier may not have updated capacity yet; let supplier handle

## adapter implementation guide

### inversion of control pattern

each brain supplier implements `BrainAuthAdapter` with three responsibilities:

1. **dao** — CRUD for auth spec (stored in brain's local config)
2. **capacity** — query rate limit status from brain's API
3. **format** — transform credential to brain's expected output format

the key insight: **rhachet does not know how to format credentials for each brain**. each adapter supplies its own formatter because only the brain supplier knows the expected output format.

### claude-code adapter example

```typescript
// src/adapters/brain.auth/claudeCode.ts
export const claudeCodeBrainAuthAdapter: BrainAuthAdapter<'anthropic/claude-code'> = {
  slug: 'anthropic/claude-code',

  dao: {
    get: { one: async () => { /* read from ~/.claude.json */ } },
    set: {
      findsert: async (input) => { /* write to ~/.claude.json if absent */ },
      upsert: async (input) => { /* write to ~/.claude.json */ },
    },
    del: async () => { /* remove from ~/.claude.json */ },
  },

  capacity: {
    get: {
      one: async (input) => {
        const response = await fetch('https://api.anthropic.com/v1/rate_limits', {
          headers: { 'x-api-key': await getTokenFromKeyrack(input.by.slug) },
        });
        const data = await response.json();
        const usedPercent = Math.round((data.requests_used / data.requests_limit) * 100);
        return new BrainAuthCapacity({
          credential: { slug: input.by.slug },
          tokens: {
            unit: 'percentage',
            used: usedPercent,
            limit: 100,
            left: 100 - usedPercent,
          },
          refreshAt: usedPercent >= 100 ? data.reset_at : null,
        });
      },
      all: async () => {
        const slugs = await keyrack.glob({ pattern: 'claude-code/*/*' });
        return Promise.all(slugs.map((slug) => this.capacity.get.one({ by: { slug } })));
      },
    },
  },

  // claude-code expects raw token string
  format: {
    credential: (input): BrainAuthSupplied<'anthropic/claude-code'> =>
      new BrainAuthSupplied({
        credential: input.credential,
        brainSlug: 'anthropic/claude-code',
        formatted: input.credential.token,  // just the token
      }),
  },
};
```

```typescript
// src/domain.operations/keyrack/globKeyrackHostKeys.ts
import { minimatch } from 'minimatch';

/**
 * .what: expand glob pattern to list of slugs from host manifest
 * .why: pool strategy needs to discover tokens via pattern
 * .grain: transformer (operates on in-memory manifest, no i/o)
 */
export const globKeyrackHostKeys = (
  input: { pattern: string },
  context: { hostManifest: KeyrackHostManifest },
): string[] => {
  const allSlugs = Object.keys(context.hostManifest.hosts);
  return allSlugs.filter((slug) => minimatch(slug, input.pattern));
};

// usage in getOneBrainAuthCredentialBySpec:
const slugs = globKeyrackHostKeys(
  { pattern: 'ehmpathy/prod/KEY_*' },
  { hostManifest: context.keyrack.hostManifest },
);
// returns: ['ehmpathy/prod/KEY_1', 'ehmpathy/prod/KEY_2', ...]
```

## capacity API

### capacity source

capacity is NOT queried from a public API endpoint. it comes from:

1. **claude-code**: `claude api usage` command or internal SDK
2. **opencode**: provider-specific usage API
3. **fallback**: assume infinite capacity if API unavailable

### claude-code capacity implementation

```typescript
// claude-code adapter queries capacity via claude CLI
capacity: {
  get: {
    one: async (input) => {
      // option 1: parse claude api usage output
      const usage = await execAsync(`claude api usage --key ${input.by.slug} --json`);
      const data = JSON.parse(usage.stdout);

      // option 2: if no usage API, probe with minimal request
      // and check response headers for rate limit info

      const limit = data.requests_limit ?? 100;
      const used = data.requests_used ?? 0;
      const usedPercent = limit > 0 ? Math.round((used / limit) * 100) : 0;
      return new BrainAuthCapacity({
        credential: { slug: input.by.slug },
        tokens: {
          unit: 'percentage',
          used: usedPercent,
          limit: 100,
          left: 100 - usedPercent,
        },
        refreshAt: usedPercent >= 100 ? (data.reset_at ?? null) : null,
      });
    },
    all: async () => {
      const slugs = await keyrack.glob({ pattern: 'claude-code/*/*' });
      return Promise.all(slugs.map((slug) => this.capacity.get.one({ by: { slug } })));
    },
  },
},
```

```typescript
// if capacity API fails, assume full capacity
```

```typescript
export const setBrainConfig = async (input: {
```

```typescript
  const sorted = capacities.sort((a, b) => b.tokens.left - a.tokens.left);

  for (const cap of sorted) {
    const status = cap.tokens.left > 0
      ? `${cap.tokens.left}% left`
      : `exhausted, refresh ${cap.refreshAt}`;
    stdout.write(`${cap.credential.slug}  ${status}\n`);
  }
};
```

## notes

- all CLI outputs follow treestruct turtle vibes format
- stateless design: no local state file, capacity queried via BrainAuthAdapter on each call
- BrainAuthAdapter.capacity DAO is readonly — brain suppliers implement capacity queries

### scope decisions

- **CLI-only for spike**: CLI contract (`invokeBrainsAuth.ts`) is sufficient; SDK extension deferred to post-spike. apiKeyHelper calls CLI directly via `npx rhachet brains auth get --from $spec --for $brainSlug`.
- **BrainAuthAdapter pattern**: follows BrainHooksAdapter pattern (inversion of control). brain suppliers implement both `dao` (spec CRUD) and `capacity` (readonly queries).
- **no local state**: capacity is queried from the brain adapter, not persisted locally.
- **graceful capacity degradation**: if capacity API unavailable, assume infinite capacity and proceed.

### test infrastructure

- **fixture required**: acceptance tests use `withBrainAuthFixture` with 3 pre-configured tokens
- **reuse extant infra**: daemon cleanup utility, isolated HOME pattern, CLI binary invocation — all from keyrack test infrastructure
- **mock adapter for unit tests**: `createMockBrainAuthAdapter` with configurable capacity responses

### decision 1: keyrack.glob verification

**question**: does keyrack support glob pattern expansion?

**status**: research required before implementation

**action**: verify keyrack.glob exists. if absent:
- option A: implement keyrack.glob (expand `KEY_*` to list of matching slugs)
- option B: enumerate all keyrack keys then filter client-side
- option C: require explicit slug list in spec (no glob)

**implementation note**: until verified, assume keyrack.glob extant. if absent at implementation time, fall back to option B (enumerate + filter).

### decision 2: asBrainAuthTokenSlugs is communicator, not transformer

**correction**: `asBrainAuthTokenSlugs` performs keyrack i/o (glob lookup), so it is a **communicator**, not a transformer.

**refactored signature**:

```typescript
/**
 * .what: expand spec source to list of credential slugs
 * .why: glob patterns need keyrack i/o to expand
 * .grain: communicator (has keyrack i/o)
 */
export const asBrainAuthTokenSlugs = async (
  input: { spec: BrainAuthSpecShape },
  context: { keyrack: { glob: (pattern: string) => Promise<string[]> } },
): Promise<string[]> => {
  if (input.spec.strategy === 'solo') {
    // solo: no glob needed, extract slug from source
    return [asKeyrackSlug({ from: input.spec.source })];
  }

  if (input.spec.strategy === 'pool') {
    // pool: expand glob via keyrack
    const pattern = asKeyrackPattern({ from: input.spec.source });
    const slugs = await context.keyrack.glob(pattern);
    if (slugs.length === 0) {
      throw new Error(`no tokens found for pattern: ${pattern}`);
    }
    return slugs;
  }

  // default: no credentials
  return [];
};
```

**test classification**: integration test (not unit) due to keyrack i/o.

```typescript
// src/adapters/brain.auth/claudeCode.ts
import { BrainAuthAdapter, BrainAuthCapacity, BrainAuthCredential, BrainAuthSupplied, BrainAuthSpecShape } from '@src/domain.objects';

export const claudeCodeBrainAuthAdapter: BrainAuthAdapter<'anthropic/claude-code'> = {
  slug: 'anthropic/claude-code',

  dao: {
    get: {
      one: async (): Promise<BrainAuthSpecShape | null> => {
        // read from ~/.claude.json → auth section
        const config = await readClaudeConfig();
        return config?.auth?.spec ?? null;
      },
    },
    set: {
      findsert: async (input: { spec: BrainAuthSpecShape }): Promise<BrainAuthSpecShape> => {
        const extant = await claudeCodeBrainAuthAdapter.dao.get.one();
        if (extant) return extant;
        await writeClaudeConfig({ auth: { spec: input.spec } });
        return input.spec;
      },
      upsert: async (input: { spec: BrainAuthSpecShape }): Promise<BrainAuthSpecShape> => {
        await writeClaudeConfig({ auth: { spec: input.spec } });
        return input.spec;
      },
    },
    del: async (): Promise<void> => {
      await writeClaudeConfig({ auth: undefined });
    },
  },

  capacity: {
    get: {
      one: async (input: { by: { slug: string } }): Promise<BrainAuthCapacity | null> => {
        // query claude API for token capacity
        const response = await fetch('https://api.anthropic.com/v1/usage', {
          headers: { Authorization: `Bearer ${await getTokenValue(input.by.slug)}` },
        });
        if (!response.ok) return null;
        const data = await response.json();
        const usedPercent = Math.round((data.requests_used / data.requests_limit) * 100);
        return new BrainAuthCapacity({
          credential: { slug: input.by.slug },
          tokens: {
            unit: 'percentage',
            used: usedPercent,
            limit: 100,
            left: 100 - usedPercent,
          },
          refreshAt: usedPercent >= 100 && data.refreshes_at
            ? asIsoTimeStamp(new Date(data.refreshes_at))
            : null,
        });
      },
      all: async (input: { slugs: string[] }): Promise<BrainAuthCapacity[]> => {
        return Promise.all(
          input.slugs.map((slug) => claudeCodeBrainAuthAdapter.capacity.get.one({ by: { slug } })),
        ).then((caps) => caps.filter((c): c is BrainAuthCapacity => c !== null));
      },
    },
  },

  format: {
    credential: (input: { credential: BrainAuthCredential }): BrainAuthSupplied => {
      return new BrainAuthSupplied({
        credential: input.credential,
        brainSlug: 'anthropic/claude-code',
        formatted: input.credential.token, // raw token for apiKeyHelper
      });
    },
  },
};
```

### decision 11: multi-brain scenario

**decision**: auth spec is per-brain. each brain adapter stores its own spec.

each brain's apiKeyHelper is self-contained with the `--for` flag, so each brain calls the CLI with its own brain slug — no implicit discovery needed.
