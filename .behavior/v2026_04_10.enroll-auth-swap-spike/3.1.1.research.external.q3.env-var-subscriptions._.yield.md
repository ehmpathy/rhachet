# q3: env var for subscriptions — yield

## answer

**`CLAUDE_CODE_OAUTH_TOKEN`** via `claude setup-token` (1-year validity)

## source

[Claude Code Authentication](https://code.claude.com/docs/en/authentication)

> Generate a long-lived token: For CI pipelines or other environments where interactive browser login isn't available, generate a one-year OAuth token with `claude setup-token`

## precedence order

| precedence | env var | header | use case |
|------------|---------|--------|----------|
| 1 | cloud provider vars | varies | bedrock, vertex, foundry |
| 2 | `ANTHROPIC_AUTH_TOKEN` | `Authorization: Bearer` | LLM gateway/proxy |
| 3 | `ANTHROPIC_API_KEY` | `X-Api-Key` | direct API access |
| 4 | `apiKeyHelper` output | depends on format | **dynamic/rotated credentials** |
| 5 | `CLAUDE_CODE_OAUTH_TOKEN` | OAuth | CI/pipelines |
| 6 | subscription OAuth | OAuth | default `/login` |

## implications

- `apiKeyHelper` sits at precedence 4
- can override subscription OAuth (precedence 6)
- `claude setup-token` generates 1-year OAuth tokens per subscription
- store each subscription's token in keyrack under distinct slugs
