# q4: setup-token exists — yield

## answer

**yes, `claude setup-token` exists** — generates 1-year OAuth token

## source

[Claude Code Authentication](https://code.claude.com/docs/en/authentication)

> Generate a long-lived token: For CI pipelines or other environments where interactive browser login isn't available, generate a one-year OAuth token with `claude setup-token`

## workflow

```
claude setup-token  # generates token, copy to keyrack
```

## implications

- each subscription can have its own long-lived token
- store tokens in keyrack under distinct slugs (e.g., `AUTH_TOKEN_SUB1`, `AUTH_TOKEN_SUB2`)
- rotation selects which token to use; tokens are not refreshed
- no need to extract from `~/.claude/.credentials.json`
