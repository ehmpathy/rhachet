## pattern.2 = daemon cleanup utility

### what

`killKeyrackDaemonForTests` kills daemon processes and removes socket files between tests.

### citations

**[3]** `blackbox/.test/infra/killKeyrackDaemonForTests.ts:1-79`
```typescript
export const killKeyrackDaemonForTests = (input?: { owner?: string | null }) => {
  const sessionId = readFileSync('/proc/${PID}/sessionid');
  const socketPath = `${runtimeDir}/keyrack.${sessionId}.${owner}.sock`;
  process.kill(pid, 'SIGTERM');
  unlinkSync(socketPath);
};
```

### relation to wish

### verdict

[REUSE] — reuse cleanup utility; call before rotation scenarios.

## pattern.5 = CLI binary invocation

### what

`invokeRhachetCliBinary` spawns the CLI as a subprocess for black-box tests.

### citations

**[6]** `blackbox/.test/infra/invokeRhachetCliBinary.ts:36-68`
```typescript
export const invokeRhachetCliBinary = (input: {
  binary?: 'rhachet' | 'rhx';
  args: string[];
  cwd: string;
  stdin?: string;
  logOnError?: boolean;
  env?: Record<string, string | undefined>;
}): SpawnSyncReturns<string> => {
  const binPath = input.binary === 'rhx' ? RHX_BIN : RHACHET_BIN;
  return spawnSync(binPath, input.args, {
    cwd: input.cwd,
    input: input.stdin,
    encoding: 'utf-8',
    env: { ...process.env, ...input.env },  // merge HOME override
  });
};
```

### relation to wish

auth pool CLI commands (`rhx auth-pool get-best`, `rhx enroll --creds`) need acceptance tests via this pattern.

### verdict

[REUSE] — reuse binary invocation for auth-pool command tests.

  args: ['keyrack', 'get', '--key', 'SUDO_SECURE_KEY', '--env', 'sudo', '--json'],
  cwd: repo.path,

## pattern.9 = test home with SSH key

### what

`withTestHomeWithSshKey` creates isolated HOME with SSH key for identity-based vault tests.

### citations

**[10]** `src/.test/infra/withTestHomeWithSshKey.ts:14-95`
```typescript
export const withTestHomeWithSshKey = async <T>(
  input: { name: string },
  fn: (home: string) => Promise<T>,
): Promise<T> => {
  const tempHome = join(os.tmpdir(), 'rhachet-test', input.name);
  mkdirSync(join(tempHome, '.ssh'), { recursive: true });
  copyFileSync(TEST_SSH_KEY_PATH, join(tempHome, '.ssh', 'id_ed25519'));
  copyFileSync(TEST_SSH_PUBKEY_PATH, join(tempHome, '.ssh', 'id_ed25519.pub'));

  process.env.HOME = tempHome;  // override HOME
  return await fn(tempHome);
};
```

### relation to wish

auth pool tokens stored in os.secure require SSH key for encode. test isolation ensures pool tests don't interfere with each other.

### verdict

[REUSE] — reuse isolated HOME pattern for pool rotation integration tests.

## key insights

1. **daemon is critical**: rotation state lives in daemon. tests must verify daemon correctly tracks active token and rate-limited set.

2. **fixture strategy**: need `with-auth-pool` fixture with 3 pre-configured tokens in os.secure vault.

3. **status granularity**: pool adds new statuses beyond granted/locked/absent — need exhausted (all rate-limited) and rate-limited (specific token).

4. **TTL for cooldown**: rate-limit cooldown (e.g., 2 hours) can reuse TTL verification patterns.
