# q8: proactive rotation — yield

## answer

**headers in response** — rate limit headers provide visibility

## source

[Anthropic Rate Limits](https://platform.claude.com/docs/en/api/rate-limits)

response headers include:

| header | description |
|--------|-------------|
| `anthropic-ratelimit-requests-remaining` | requests left before rate limited |
| `anthropic-ratelimit-tokens-remaining` | tokens left (rounded to nearest thousand) |
| `anthropic-ratelimit-input-tokens-remaining` | input tokens left |
| `anthropic-ratelimit-output-tokens-remaining` | output tokens left |
| `retry-after` | seconds to wait on 429 |

## implications

- proactive rotation IS possible via header inspection
- however, Claude CLI (not raw API) may not expose these headers
- for spike: reactive rotation (on 429) is acceptable
- future: could add proactive rotation if CLI exposes headers

## spike scope

reactive rotation acceptable for MVP:
1. use token until 429
2. apiKeyHelper called on next TTL check (5 min default)
3. auth-pool returns next available token

proactive rotation is enhancement for later.
