# blackbox criteria: enroll-auth-swap

## usecase.1 = configure auth pool

### episode: first-time setup

```
given(developer has 3 claude subscription tokens)
  when(developer runs keyrack set for each token)
    then(each token is stored in keyrack under specified owner/env)
      sothat(tokens are available for enrollment)
    then(tokens are encrypted at rest)
      sothat(credentials are secure)

given(tokens are stored in keyrack)
  when(developer runs rhx enroll with --auth spec)
    then(enrollment succeeds)
      sothat(claude is wrapped with auth rotation)
    then(auth spec is validated)
      sothat(invalid specs fail fast with clear error)
```

### episode: session start

```
given(developer has enrolled with auth pool)
  when(developer starts new terminal session)
    when(developer unlocks keyrack with glob pattern)
      then(all tokens in pool are unlocked)
        sothat(rotation can access any token)
      then(unlock is idempotent)
        sothat(re-unlock does not error)
```

## usecase.2 = automatic rotation on rate limit

### episode: single rotation

```
given(enrolled brain with 3-token pool)
  given(token A is active)
    when(token A hits rate limit)
      then(system detects rate limit response)
        sothat(rotation triggers)
      then(system rotates to token B)
        sothat(work continues)
      then(rotation is invisible to user)
        sothat(flow state is preserved)
      then(state file records A as rate-limited)
        sothat(future rotations skip A until refresh)
```

### episode: cascade rotation

```
given(enrolled brain with 3-token pool)
  given(token A is rate-limited)
    given(token B is active)
      when(token B hits rate limit)
        then(system rotates to token C)
          sothat(work continues)
        then(state file records B as rate-limited)

given(enrolled brain with 3-token pool)
  given(tokens A and B are rate-limited)
    given(token C is active)
      when(token C hits rate limit)
        then(system attempts rotation to A)
          sothat(round-robin continues)
        when(A is still rate-limited)
          then(system attempts B)
            when(B is still rate-limited)
              then(system reports all tokens exhausted)
                sothat(user knows to wait or add tokens)
```

### episode: rate limit refresh

```
given(token A was rate-limited 2 hours ago)
  given(token B is active)
    when(token B hits rate limit)
      then(system attempts token A)
        when(token A responds successfully)
          then(A becomes active token)
            sothat(rate-limited tokens recover over time)
          then(state file clears A from rate-limited set)
```

## usecase.3 = per-brain isolation

### episode: multiple enrolled brains

```
given(brain X enrolled with pool [A, B, C])
  given(brain Y enrolled with pool [D, E, F])
    when(brain X rotates from A to B)
      then(brain Y is unaffected)
        sothat(each brain manages its own pool)
      then(global auth state is unchanged)
        sothat(non-enrolled processes are unaffected)
```

### episode: same-machine terminals

```
given(3 terminals on same machine)
  given(all use same enrolled brain with shared pool)
    when(terminal 1 triggers rotation from A to B)
      then(terminals 2 and 3 see B as active)
        sothat(cross-terminal state is consistent)
      then(state file is source of truth)
        sothat(no race conditions between terminals)
```

## usecase.4 = capacity-based token selection

### episode: select token with most capacity

```
given(enrolled brain with 3-token pool)
  given(pool tokens have different capacity)
    given(token A has 10 calls left)
    given(token B has 50 calls left)
    given(token C has 30 calls left)
      when(system selects best token)
        then(system queries capacity via BrainAuthAdapter.capacity.get.all())
          sothat(capacity is provided by brain supplier)
        then(system selects token B)
          sothat(token with most capacity is used first)
        then(selection is stateless)
          sothat(no local state file tracks capacity)
```

### episode: exhaustion detection via capacity

```
given(enrolled brain with 3-token pool)
  given(all tokens have 0 capacity left)
    when(system attempts to select best token)
      then(system queries capacity via BrainAuthAdapter.capacity.get.all())
      then(system detects all tokens exhausted)
        sothat(exhaustion is determined by capacity, not trial-and-error)
      then(system returns exhaustion error with refresh times)
        sothat(user knows when capacity will recover)
```

### episode: capacity query per brain

```
given(brain X with BrainAuthAdapter)
  given(brain Y with different BrainAuthAdapter)
    when(system selects token for brain X)
      then(system queries brainAuthAdapterX.capacity.get.all())
        sothat(capacity lookup uses brain-specific adapter)
      then(brainAuthAdapterY is not queried)
        sothat(adapter isolation is maintained)
```

## usecase.5 = error states

### episode: all tokens exhausted

```
given(enrolled brain with 3-token pool)
  given(all tokens are rate-limited)
    when(brain attempts API call)
      then(system returns exhaustion error)
        sothat(user knows all tokens are depleted)
      then(error includes hint to wait or add tokens)
        sothat(user has actionable guidance)
      then(error is distinct from auth failure)
        sothat(user can distinguish rate limit from bad token)
```

### episode: invalid token

```
given(enrolled brain with pool that includes expired token)
  when(system attempts to use expired token)
    then(system detects auth failure)
      sothat(bad token is identified)
    then(system rotates to next token)
      sothat(work continues if valid tokens remain)
    then(state file records token as invalid)
      sothat(future rotations skip it)
```

### episode: keyrack locked

```
given(enrolled brain with pool)
  given(keyrack session expired)
    when(system attempts to get token)
      then(system returns keyrack-locked error)
        sothat(user knows to unlock keyrack)
      then(error includes unlock command hint)
        sothat(user can quickly recover)
```

## exchange boundaries

### input: enroll command

```
given(valid auth spec 'pool(keyrack://ehmpath/prep/CLAUDE_OAUTH_TOKEN_*)')
  when(rhx enroll --role mechanic --auth $spec)
    then(enrollment succeeds)

given(invalid auth spec 'pool(keyrack://bad)')
  when(rhx enroll --role mechanic --auth $spec)
    then(enrollment fails with spec parse error)

given(auth spec with no matches 'pool(keyrack://ehmpath/prep/NONEXISTENT_*)')
  when(rhx enroll --role mechanic --auth $spec)
    then(enrollment fails with no-matches error)
```

### input: brains auth get command

```
given(pool with healthy token)
  when(npx rhachet brains auth get --from $spec --into claude.apiKeyHelper)
    then(returns active token value)
    then(exit code 0)

given(pool with all rate-limited tokens)
  when(npx rhachet brains auth get --from $spec --into claude.apiKeyHelper)
    then(returns exhaustion error)
    then(exit code 2)

given(keyrack locked)
  when(npx rhachet brains auth get --from $spec --into claude.apiKeyHelper)
    then(returns keyrack-locked error)
    then(exit code 2)
```

### output: state file

```
given(rotation occurs)
  then(state file contains)
    then(activeTokenKey: current token key)
    then(lastRotation: ISO timestamp)
    then(rateLimited: map of key to timestamp)
```
