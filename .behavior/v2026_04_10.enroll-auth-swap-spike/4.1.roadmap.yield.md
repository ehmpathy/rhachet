## phase 3: orchestrator

### 3.1 getOneBrainAuthCredentialBySpec orchestrator

- [ ] create `src/domain.operations/brain.auth/getOneBrainAuthCredentialBySpec.ts`
- [ ] parse spec → expand tokens → filter by capacity → fetch credential
- [ ] return BrainAuthSupplied on success, BrainAuthError on failure
- [ ] create integration test

**acceptance criteria:**
```
given(spec with tokens that have capacity)
  when(getOneBrainAuthCredentialBySpec is called)
    then(returns BrainAuthSupplied with credential and capacity)
    then(selects token with most capacity left)

given(spec with all tokens exhausted)
  when(getOneBrainAuthCredentialBySpec is called)
    then(returns BrainAuthError with code = 'EXHAUSTED')
    then(error includes refreshAt for soonest refresh)

given(keyrack locked)
  when(getOneBrainAuthCredentialBySpec is called)
    then(returns BrainAuthError with code = 'KEYRACK_LOCKED')
```

**verification:** `rhx git.repo.test --what integration --scope getOneBrainAuthCredentialBySpec`

## phase 4: CLI contract

### 4.1 invokeBrainsAuth CLI handler

- [ ] create `src/contract/cli/invokeBrainsAuth.ts`
- [ ] implement `brains auth get --from $spec --for $brainSlug`
- [ ] output only the secret on success (for apiKeyHelper)
- [ ] exit 0 on success, exit 2 on constraint error
- [ ] create acceptance test

**acceptance criteria:**
```
given(valid spec with available token)
  when(rhx brains auth get --from $spec --for $brainSlug)
    then(outputs secret to stdout)
    then(exits 0)

given(all tokens exhausted)
  when(rhx brains auth get --from $spec --for $brainSlug)
    then(outputs error message to stderr)
    then(exits 2)

given(keyrack locked)
  when(rhx brains auth get --from $spec --for $brainSlug)
    then(outputs unlock hint to stderr)
    then(exits 2)
```

**verification:** `rhx git.repo.test --what acceptance --scope invokeBrainsAuth`
