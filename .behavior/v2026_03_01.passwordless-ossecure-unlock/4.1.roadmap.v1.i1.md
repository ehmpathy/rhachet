## phase 1: core logic changes

### prereqs
- read `.behavior/v2026_03_01.passwordless-ossecure-unlock/3.3.blueprint.v1.i1.md` sections "detailed changes #2, #3, #4, #4.1"
- read `.behavior/v2026_03_01.passwordless-ossecure-unlock/3.1.research.patterns._.code.prod.v1.i1.md` (if declared)

### tasks

- [ ] **1.1** update vaultAdapterOsSecure.ts unlock logic
  - check `getActiveIdentity() !== null` first → return early
  - check `input.identity !== null` → set sessionIdentity, return
  - remove all passphrase logic (input check, env var, prompt)
  - throw UnexpectedCodePathError if no identity

- [ ] **1.2** update unlockKeyrackKeys.ts
  - change `passphrase: input.passphrase` to `identity: null`
  - add comment that explains session identity flow

- [ ] **1.3** update setKeyrackKey.ts
  - remove env=all expansion logic
  - always use single slug: `${input.org}.${input.env}.${input.key}`

- [ ] **1.4** update getAllKeyrackSlugsForEnv.ts
  - change filter from `spec.env === input.env`
  - to `spec.env === input.env || spec.env === 'all'`

### acceptance
- `npm run test:types` passes
- `npm run test:unit` passes (may need updates for new behavior)

### verification
```sh
npm run test:types && npm run test:unit
```

## phase 3: test coverage

### prereqs
- read `.behavior/v2026_03_01.passwordless-ossecure-unlock/2.1.criteria.blackbox.md`
- read `.behavior/v2026_03_01.passwordless-ossecure-unlock/3.3.blueprint.v1.i1.md` section "test coverage"
- read `.behavior/v2026_03_01.passwordless-ossecure-unlock/3.1.research.patterns._.code.test.v1.i1.md` (if declared)

### tasks

#### unit tests

- [ ] **3.1** setKeyrackKey.test.ts
  - env=all stores single slug $org.all.$key
  - (negative) env=all does NOT store under $org.prod.$key

- [ ] **3.2** getAllKeyrackSlugsForEnv.test.ts
  - env=prod includes $org.all.$key slugs
  - env=prod includes $org.prod.$key slugs
  - (negative) env=prod does NOT include $org.test.$key slugs
  - env=all returns all slugs (unchanged behavior)

- [ ] **3.3** vaultAdapterOsSecure.test.ts
  - unlock with session identity set returns early
  - (negative) unlock without identity throws error

#### integration tests

- [ ] **3.4** vaultAdapterOsSecure.integration.test.ts
  - set with prikey + unlock with prikey succeeds
  - (negative) unlock without identity throws error

- [ ] **3.5** unlockKeyrackKeys.integration.test.ts
  - identity flows from manifest to vault unlock
  - unlock --env prod includes keys from env=all
  - (negative) unlock --env prod does NOT include keys from env=test

#### acceptance tests

- [ ] **3.6** keyrack.vault.osSecure.acceptance.test.ts
  - init → set → unlock → get with --prikey works
  - (negative) unlock without identity fails with clear error

- [ ] **3.7** keyrack.envAll.acceptance.test.ts (new file)
  - set --env all stores once, list shows once
  - get --env prod returns value from env=all key
  - get --env test returns value from env=all key
  - (negative) set --env all does NOT create $org.prod.$key
  - (negative) get --env prod does NOT return $org.test.$key

- [ ] **3.8** keyrack.validation.acceptance.test.ts
  - --passphrase flag returns unrecognized error
  - (negative) unlock without --prikey and no default identity fails

- `npm run test:acceptance` passes

## final verification

after all phases complete:

```sh
# full test suite
npm run test

# manual smoke test
npx tsx ./bin/run keyrack init --owner testorg --prikey ~/.ssh/test
npx tsx ./bin/run keyrack set TEST_KEY --value testvalue --env all
npx tsx ./bin/run keyrack unlock --owner testorg --prikey ~/.ssh/test --env prod
npx tsx ./bin/run keyrack get TEST_KEY --env prod
```

expected: unlock succeeds, get returns value from env=all key
