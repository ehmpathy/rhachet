# blueprint: passwordless os.secure unlock

## summary

three fixes for three issues:

1. **identity must flow to vault unlock** — vaultAdapterOsSecure.unlock() ignores already-set sessionIdentity
2. **env=all must not expand on SET, but must include on GET** — setKeyrackKey() expands to all matched slugs instead of single $org.all.$key; getAllKeyrackSlugsForEnv() must include env='all' slugs when queried for specific env
3. **passphrase must be deprecated** — remove CLI flag, update adapter interface, remove interactive prompt

## codepaths

### R1: identity flows to unlock

```
CLI: keyrack unlock --prikey ~/.ssh/ehmpath
  └─ genContextKeyrackGrantUnlock({ prikey })
       └─ daoKeyrackHostManifest.get({ prikey })
            └─ decrypts manifest with prikey
            └─ caches identity in manifestIdentity
       └─ [○] genKeyrackHostContext()
            └─ [○] setOsSecureSessionIdentity(manifestIdentity)   # already sets session identity
  └─ unlockKeyrackKeys({ ... }, context)
       └─ for each slug
            └─ adapter.unlock({ passphrase, exid })
                 │
                 ├─ [~] change to: adapter.unlock({ identity, exid })
                 │
                 └─ vaultAdapterOsSecure.unlock()
                      ├─ [~] check getActiveIdentity() first
                      ├─ [~] if identity available, return early (already unlocked)
                      └─ [-] remove passphrase prompt fallback
```

### R2: env=all behavior

**R2.1: SET stores once (no expansion)**

```
CLI: keyrack set --key X --env all
  └─ setKeyrackKey({ key, env: 'all', ... })
       │
       ├─ [current] if env=all && manifest:
       │    └─ expand to all matched slugs from manifest  👎 wrong
       │
       └─ [~] fix: always use single slug: $org.all.$key
            └─ no expansion
            └─ $org.all.X stored once
```

**R2.2: GET/UNLOCK includes env='all' keys**

```
CLI: keyrack get --key X --env prod
  └─ getAllKeyrackSlugsForEnv({ manifest, env: 'prod' })
       │
       ├─ [current] filter: spec.env === input.env
       │    └─ returns $org.prod.$key only  👎 wrong
       │
       └─ [~] fix: include env='all' slugs
            └─ filter: spec.env === input.env || spec.env === 'all'
            └─ returns both $org.prod.$key AND $org.all.$key
```

### R3: passphrase deprecated

```
KeyrackHostVaultAdapter
  └─ unlock(input: { passphrase?: string; ... })
       │
       └─ [~] change to: unlock(input: { identity: string | null; ... })
            └─ [-] remove passphrase from signature
            └─ callers must explicitly pass identity or null

invokeKeyrack.ts
  └─ keyrack unlock command
       └─ [-] remove .option('--passphrase <passphrase>', ...)
       └─ [-] remove passphrase from unlockKeyrackKeys() call

vaultAdapterOsSecure.ts
  └─ unlock()
       └─ [~] remove input.passphrase checks
       └─ [-] remove promptHiddenInput() call
       └─ [~] if identity available (session or input): return (already unlocked)
       └─ [~] if no identity: throw UnexpectedCodePathError('identity required')
```

## detailed changes

### 1. KeyrackHostVaultAdapter.ts

```diff
  unlock: (input: {
-   passphrase?: string;
+   identity: string | null;
    silent?: boolean;
    exid?: string | null;
  }) => Promise<void>;
```

**rationale**: `identity: string | null` forces callers to consciously decide whether identity is available (vs optional which allows callers to forget)

### 2. vaultAdapterOsSecure.ts

```diff
  unlock: async (input: {
-   passphrase?: string;
-   identity?: string;
+   identity: string | null;
  }) => {
-   // check identity input param first (preferred path — set from manifest decryption)
-   if (input.identity) {
-     sessionIdentity = input.identity;
-     return;
-   }
+   // check if identity already available (set via setOsSecureSessionIdentity)
+   if (getActiveIdentity() !== null) return;
+
+   // check input identity
+   if (input.identity !== null) {
+     sessionIdentity = input.identity;
+     return;
+   }

-   // check passphrase input param (legacy path)
-   if (input.passphrase) { ... }
-
-   // check passphrase env var (legacy)
-   if (process.env.KEYRACK_PASSPHRASE) { ... }
-
-   // prompt interactively with hidden input (human interactive, legacy)
-   const passphrase = await promptHiddenInput({ ... });
-
-   // validate passphrase was provided
-   if (!passphrase) {
-     throw new UnexpectedCodePathError(
-       'os.secure unlock requires identity or passphrase',
-       ...
-     );
-   }
-
-   sessionPassphrase = passphrase;
+   // no identity available
+   throw new UnexpectedCodePathError(
+     'os.secure unlock requires identity',
+     { hint: 'use --prikey to specify ssh key or run keyrack init' },
+   );
  },
```

### 3. unlockKeyrackKeys.ts

```diff
  // unlock vault if needed
  const isUnlocked = await adapter.isUnlocked({ exid: hostConfig.exid });
  if (!isUnlocked) {
    await adapter.unlock({
-     passphrase: input.passphrase,
+     identity: null,  // identity already set in session via genKeyrackHostContext
      exid: hostConfig.exid,
    });
  }
```

**note**: we pass `identity: null` because session identity is already set by `setOsSecureSessionIdentity()` in `genKeyrackHostContext()`. the adapter checks `getActiveIdentity()` first.

### 4. setKeyrackKey.ts

```diff
  // compute target slugs based on env
  const targetSlugs: string[] = (() => {
-   // env=all with manifest: expand to all envs that declare this key
-   if (input.env === 'all' && input.repoManifest) {
-     const expanded = getAllKeyrackSlugsForEnv({
-       manifest: input.repoManifest,
-       env: 'all',
-     }).filter((s) => asKeyrackKeyName({ slug: s }) === input.key);
-     if (expanded.length > 0) return expanded;
-   }
-   // specific env or no manifest: use simple slug
    return [`${input.org}.${input.env}.${input.key}`];
  })();
```

**rationale**: env=all should store under `$org.all.$key` only. expansion was wrong — it created duplicates.

### 4.1. getAllKeyrackSlugsForEnv.ts

```diff
  // env=all returns all slugs
  if (input.env === 'all') return Object.keys(input.manifest.keys);

  // filter to slugs where spec.env matches
  return Object.entries(input.manifest.keys)
-   .filter(([, spec]) => spec.env === input.env)
+   .filter(([, spec]) => spec.env === input.env || spec.env === 'all')
    .map(([slug]) => slug);
```

**rationale**: when user queries for specific env (e.g., `--env prod`), keys set with `--env all` should also be included. env=all keys are accessible from any environment.

## risk analysis

| change | risk | mitigation |
|--------|------|------------|
| remove passphrase from interface | breaks callers that passed passphrase | type error at compile time; easy to find and fix |
| remove env=all expansion on SET | changes behavior for users who relied on expansion | clear win — old behavior was wrong |
| include env=all on GET/UNLOCK | returns more keys than before | correct behavior — env=all should match any env |
| require identity in os.secure | users without prikey get error | clear error message guides to fix |

---

## migration

users with passphrase-encrypted credentials (legacy):
1. run `keyrack init --prikey ~/.ssh/key` to add identity to manifest
2. run `keyrack set --key X --prikey ~/.ssh/key` to re-encrypt with identity
3. `keyrack unlock --prikey ~/.ssh/key` now works

no automated migration — users must re-set credentials with identity.
