# test coverage

## unit tests: brief structure compliance

### what

validate each brief follows the required template structure.

### how

```ts
// .behavior/.../tests/unit/briefStructure.test.ts

describe('postlabor briefs', () => {
  const briefs = glob('.agent/repo=.this/role=ecologist/briefs/postlabor/*.md');

  briefs.forEach(brief => {
    it(`${brief} has required sections`, () => {
      const content = read(brief);
      expect(content).toContain('## .tldr');
      expect(content).toContain('## .what');
      expect(content).toContain('## .why');
    });

    it(`${brief} has citations when claims are made`, () => {
      const content = read(brief);
      if (content.includes('## .claims')) {
        expect(content).toMatch(/> \[\d+\]/); // numbered citation format
      }
    });
  });
});
```

## integration tests: cross-reference integrity

### what

validate links between briefs resolve and readme links work.

### how

```ts
// .behavior/.../tests/integration/crossReferences.test.ts

describe('cross-references', () => {
  it('readme links to postlabor briefs dir that exists', () => {
    const readme = read('readme.md');
    const linkMatch = readme.match(/\[postlabor briefs\]\(([^)]+)\)/);
    expect(linkMatch).toBeTruthy();
    expect(exists(linkMatch[1])).toBe(true);
  });

  it('soph001.overview links to all other briefs', () => {
    const overview = read('.agent/.../postlabor/soph001.overview.[article].md');
    const otherBriefs = glob('.agent/.../postlabor/soph*.md')
      .filter(b => !b.includes('soph001'));
    otherBriefs.forEach(brief => {
      const filename = basename(brief);
      expect(overview).toContain(filename);
    });
  });

  it('briefs reference research citations that exist', () => {
    const briefs = glob('.agent/.../postlabor/*.md');
    const research = read('.behavior/.../3.1.research.references._.v1.i1.md');
    briefs.forEach(brief => {
      const content = read(brief);
      const citations = content.match(/ref\.\d+/g) || [];
      citations.forEach(cite => {
        expect(research).toContain(`# ${cite}`);
      });
    });
  });
});
```

# notes

## on rhachet's role

rhachet is **not** "the brain that builds brains".

rhachet **enables** the distribution of brain-build capability:
- durable enrollment of any brain.atom to execute any skill
- skills are externalized, shareable, composable
- anyone can create/share/compose actors

this distinction is critical. rhachet is the enabler, not the capability itself.

## on test coverage interpretation

since deliverables are primarily documentation:
- "unit tests" = structural validation (sections, format)
- "integration tests" = reference integrity (links, citations)
- "acceptance tests" = criteria satisfaction (human review)

automated tests verify structure; human review verifies sense.
