# playtest: boot.yml for repo=.this/role=any

## prerequisites

- rhachet installed (`npx rhachet` works)
- current branch has the boot.yml file at `.agent/repo=.this/role=any/boot.yml`

## sandbox

this playtest uses read-only operations (no file writes needed). all commands inspect boot output without file modification.

---

## happy paths

### step 1: verify boot.yml reduces token count

**action:**
```bash
npx rhachet roles boot --repo .this --role any
```

**expected outcome:**
- stats block shows `briefs = 19` total
- stats block shows `say = 7` and `ref = 12`
- tokens ≈ 8000 (around 8k, not 20k)

**pass if:** stats show say=7, ref=12, tokens under 10k
**fail if:** say=19 (all briefs inline) or tokens above 15k

---

### step 2: verify say briefs appear inline

**action:**
scroll through the boot output from step 1

**expected outcome:**
these 7 briefs appear with `<brief.say>` tags and full content:
- `define.rhachet.v3.md`
- `define.agent-dir.md`
- `howto.test-local-rhachet.md`
- `bin.dispatcher.pattern.md`
- `run.executable.lookup.pattern.md`
- `code.test.accept.blackbox.md`
- `rule.require.shared-test-fixtures.md`

**pass if:** all 7 briefs appear as `<brief.say>` with full content
**fail if:** any of these appear as `<brief.ref/>` pointers

---

### step 3: verify ref briefs appear as pointers

**action:**
scroll through the boot output from step 1

**expected outcome:**
these briefs appear with `<brief.ref/>` tags (pointer only, no content):
- `cli.repo.introspect.md`
- `rule.forbid.rhachet-use-ts.md`
- `rule.require.fixture-gitignore-negation.md`
- `postlabor.soph000.summary.[article].md`
- all briefs in `domain.thought/` subdirectory
- all briefs in `infra.composition/` subdirectory

**pass if:** unmatched briefs appear as `<brief.ref path="..."/>` (no content)
**fail if:** these briefs appear inline with full content

---

### step 4: verify symlinked directories are followed

**action:**
check the boot output from step 1 for domain.thought/ briefs

**expected outcome:**
- 6 briefs from `domain.thought/` appear as refs
- 2 briefs from `infra.composition/` appear as refs
- total briefs = 19 (includes briefs from symlinked directories)

**pass if:** symlinked briefs are discovered and appear as refs
**fail if:** symlinked briefs are absent from output

---

## edgey paths

### edge 1: boot without boot.yml (default behavior preserved)

**action:**
```bash
# temporarily move boot.yml aside
mv .agent/repo=.this/role=any/boot.yml .agent/repo=.this/role=any/boot.yml.bak

# boot without boot.yml
npx rhachet roles boot --repo .this --role any

# restore boot.yml
mv .agent/repo=.this/role=any/boot.yml.bak .agent/repo=.this/role=any/boot.yml
```

**expected outcome:**
- without boot.yml: all briefs appear as `<brief.say>` (inline)
- stats show `say = 19`, `ref = 0`
- tokens ≈ 20k (full token cost)

**pass if:** absent boot.yml causes all briefs to be said (default behavior)
**fail if:** absent boot.yml causes error or refs

---

### edge 2: boot.yml with typo in glob (no match)

**action:**
```bash
# create test boot.yml with typo
cat > .temp/boot.yml.typo << 'EOF'
briefs:
  say:
    - briefs/define.rhachet.md  # the `.v3` segment is absent
EOF

# test with typo version (manual step: copy to role, boot, restore)
# this edge case is covered by unit tests; manual verification optional
```

**expected outcome:**
- typo glob matches zero files
- the intended brief appears as ref instead of say
- no error (graceful degradation)

**pass if:** typo causes brief to appear as ref (graceful)
**fail if:** typo causes error or crash

---

## pass/fail criteria

### overall pass

- [ ] stats show say=7, ref=12, briefs=19
- [ ] tokens ≈ 8k (reduced from ~20k)
- [ ] say briefs appear with full content
- [ ] ref briefs appear as pointers only
- [ ] symlinked directories are followed
- [ ] absent boot.yml preserves default behavior (say all)

### overall fail

- stats show say=19 (no curation effect)
- tokens above 15k (curation not applied)
- say briefs appear as refs
- ref briefs appear inline
- symlinked briefs are absent
- absent boot.yml causes error
