### mechAdapterGithubApp

**location:** `src/domain.operations/keyrack/adapters/mechanisms/mechAdapterGithubApp.ts`

**[EXTEND]** currently validates and translates json blob, but lacks guided setup.

```ts
interface GithubAppCredentials {
  appId: string | number;
  privateKey: string;
  installationId: string | number;
}

export const mechAdapterGithubApp: KeyrackGrantMechanismAdapter = {
  validate: (input) => {
    // validates json structure for source, ghs_ prefix for cached
    const result = parseGithubAppCredentials(input.source);
    return result.valid ? { valid: true } : { valid: false, reasons: result.reasons };
  },
  translate: async (input) => {
    const { creds } = parseGithubAppCredentials(input.secret);
    const auth = createAppAuth({
      appId: creds.appId,
      privateKey: creds.privateKey,
      installationId: Number(creds.installationId),
    });
    const { token } = await auth({ type: 'installation' });
    const expiresAt = addDuration(asIsoTimeStamp(new Date()), { minutes: 55 });
    return { secret: token, expiresAt };
  },
};
```

**implications for blueprint:**
- needs new `promptForSet` method: guided setup (org → app → pem path)
- uses `@octokit/auth-app` for token generation — reusable
- expiresAt set to 55 min (buffer for 1hr token lifetime) — reusable

### mechAdapterAwsSso

**location:** `src/domain.operations/keyrack/adapters/mechanisms/mechAdapterAwsSso.ts`

**[REUSE]** already has validate and translate for sso profile → credentials.

```ts
export const mechAdapterAwsSso: KeyrackGrantMechanismAdapter = {
  validate: (input) => {
    // source: profile name format, cached: json with AWS_ACCESS_KEY_ID
    if (input.cached) return isValidCachedCredentials(input.cached);
    if (input.source) return isValidProfileName(input.source) ? { valid: true } : { valid: false, ... };
  },
  translate: async (input) => {
    const profile = input.secret;
    const { stdout } = await execAsync(`aws configure export-credentials --profile "${profile}" --format env-no-export`);
    // parse env output to json
    return { secret: JSON.stringify(credentials), expiresAt };
  },
};
```

**implications for blueprint:**
- needs new `promptForSet` method: **MOVE** guided setup from vault adapter
- translate uses `aws configure export-credentials` — reusable

## new operations needed

### inferVault

**purpose:** infer --vault from key name when not supplied

**logic:**
- `AWS_PROFILE`, `AWS_*` → `aws.config`
- other patterns → null (require explicit --vault)

### inferMech

**purpose:** prompt for mech selection when vault supports multiple

**logic:**
- input: vault adapter, key name
- if vault.supportedMechs.length === 1 → auto-select
- if vault.supportedMechs.length > 1 → prompt via stdin

## composition flow (target)

```
keyrack set --key GITHUB_TOKEN --vault os.secure

1. inferVault (if --vault absent) → returns null for GITHUB_TOKEN
2. vault lookup → os.secure adapter
3. checkMechCompat → EPHEMERAL_VIA_GITHUB_APP is valid
4. inferMech (if --mech absent) → prompt if multiple valid mechs
5. mech adapter promptForSet → org → app → pem → json blob
6. vault adapter set(secret from mech) → encrypt + store
7. roundtrip validation: unlock → get → relock
8. host manifest update
```
