# playtest: mech adapters own their prompts

playtest for mech adapters own their prompts.

---

## prerequisites

- `gh` cli authenticated (`gh auth status` shows logged in)
- access to at least one github org with an installed github app
- ability to mint a fresh private key for a github app (or have one available)
- aws credentials configured for aws sso access (for aws sso tests)
- repo has been built (`npm run build` passes)
- all tests pass (`npm run test` green)

---

## sandbox

all file operations must target `@gitroot/.temp/`:
- temp HOME directories for vault isolation
- mock pem files for github app tests
- no writes to actual `~/.aws/config` or `~/.keyrack/`

---

## happy paths

### path 1: aws sso set with aws.config vault

verify the aws sso guided setup works unchanged (internal restructure only).

**step 1.1: set up temp environment**

from repo root:

```bash
mkdir -p .temp
cd .temp
mkdir -p test-home/.keyrack test-home/.aws
export HOME=$(pwd)/test-home
```

**step 1.2: run keyrack set for aws profile**

```bash
npx rhx keyrack set --key AWS_PROFILE --vault aws.config
```

**expected outcome:**
- prompted for mech selection (EPHEMERAL_VIA_AWS_SSO vs PERMANENT_VIA_AWS_KEY)
- after mech selection, guided setup runs
- prompts for sso domain, account, role
- treestruct output shows progress
- success message shows `vault: aws.config`

**pass criterion:** output format matches prior behavior — same prompts, same tree structure

---

### path 2: aws sso unlock returns credentials

verify the mech adapter delivers credentials on unlock.

**step 2.1: unlock the key**

```bash
npx rhx keyrack unlock --key AWS_PROFILE --env all
```

**expected outcome:**
- vault retrieves profile name
- mech transforms profile name to sso credentials
- daemon stores credentials

**step 2.2: get the key**

```bash
npx rhx keyrack get --key AWS_PROFILE --env all
```

**expected outcome:**
- returns credentials json (not profile name)
- credentials contain access key, secret, session token
- format resembles:
  ```json
  {
    "secret": "{\"accessKeyId\":\"ASIA...\",\"secretAccessKey\":\"...\",\"sessionToken\":\"...\"}",
    "expiresAt": "2026-04-03T12:00:00Z"
  }
  ```

**pass criterion:** `keyrack get` returns usable credentials, not the raw profile name

---

### path 3: vault inference from key name

verify AWS_PROFILE infers `--vault aws.config`.

**step 3.1: run set without --vault**

```bash
npx rhx keyrack set --key AWS_PROFILE
```

**expected outcome:**
- does NOT fail with "vault required" error
- proceeds to mech inference

**pass criterion:** vault inference works — no `--vault` required for AWS_PROFILE

---

### path 4: mech inference when ambiguous

verify mech selection prompt appears when vault supports multiple mechs.

**step 4.1: run set with inferred vault**

```bash
npx rhx keyrack set --key AWS_PROFILE --vault aws.config
```

**expected outcome:**
- mech inference prompt appears:
  ```
  🔐 keyrack set AWS_PROFILE
     │
     ├─ which mechanism?
     │  ├─ options
     │  │  ├─ 1. aws sso (EPHEMERAL_VIA_AWS_SSO) — short-lived tokens via browser login
     │  │  └─ 2. aws key (PERMANENT_VIA_AWS_KEY) — long-lived access key + secret
     │  └─ choice
  ```
- user selects mech by number

**pass criterion:** mech selection prompt appears with numbered options

---

### path 5: explicit --mech skips inference

verify explicit --mech flag skips mech selection prompt.

**step 5.1: run set with explicit mech**

```bash
npx rhx keyrack set --key AWS_PROFILE --vault aws.config --mech EPHEMERAL_VIA_AWS_SSO
```

**expected outcome:**
- NO mech selection prompt
- proceeds directly to sso guided setup

**pass criterion:** mech inference skipped when --mech supplied

---

### path 6: incompatible vault/mech fails fast

verify incompatible combinations fail with clear error.

**step 6.1: try os.secure with aws sso mech**

```bash
npx rhx keyrack set --key AWS_PROFILE --vault os.secure --mech EPHEMERAL_VIA_AWS_SSO
```

**expected outcome:**
- fails fast with clear error
- error explains incompatibility
- error suggests alternative: `use --vault aws.config`

**pass criterion:** fail-fast error with alternatives shown

---

### path 7: os.direct fails fast for ephemeral mechs

verify os.direct rejects ephemeral mechs that require source key security.

**step 7.1: try os.direct with ephemeral mech**

```bash
npx rhx keyrack set --key GITHUB_TOKEN --vault os.direct --mech EPHEMERAL_VIA_GITHUB_APP
```

**expected outcome:**
- fails fast with clear error
- error explains os.direct cannot secure source keys
- error suggests alternatives: `os.secure`, `1password`

**pass criterion:** fail-fast error protects users from insecure storage

### path 8: github app set with os.secure vault

verify the github app guided setup works with os.secure vault.

**step 8.1: set up temp environment**

from repo root:

```bash
mkdir -p .temp
cd .temp
mkdir -p test-home/.keyrack test-home/.ssh
export HOME=$(pwd)/test-home
```

**step 8.2: initialize keyrack**

```bash
npx rhx keyrack init
```

**step 8.3: create repo manifest**

```bash
mkdir -p .agent
echo 'org: testorg

env.test:
  - GITHUB_TOKEN' > .agent/keyrack.yml
```

**step 8.4: run keyrack set for github token with github app mech**

```bash
npx rhx keyrack set --key GITHUB_TOKEN --env test --vault os.secure --mech EPHEMERAL_VIA_GITHUB_APP
```

**expected outcome:**
- prompted for org selection (shows list of orgs from `gh api /user/orgs`)
- prompted for app selection (shows list of apps for selected org)
- prompted for private key path (relative path to .pem file)
- treestruct output shows progress:
  ```
  🔐 keyrack set testorg.test.GITHUB_TOKEN via EPHEMERAL_VIA_GITHUB_APP
     │
     ├─ which org?
     │  ├─ options
     │  │  ├─ 1. org1
     │  │  ├─ 2. org2
     │  └─ choice: N
     │     └─ orgN ✓
     │
     ├─ which app?
     │  ├─ options
     │  │  ├─ 1. app1 (id: ...)
     │  │  ├─ 2. app2 (id: ...)
     │  └─ choice: N
     │     └─ appN ✓
     │
     └─ private key path (.pem): ./path/to/key.pem
  🔐 keyrack set (org: testorg, env: test)
     └─ testorg.test.GITHUB_TOKEN
        ├─ mech: EPHEMERAL_VIA_GITHUB_APP
        └─ vault: os.secure
  ```
- success message shows `vault: os.secure` and `mech: EPHEMERAL_VIA_GITHUB_APP`

**pass criterion:** guided setup prompts for org → app → pem path and stores credentials

## edgey paths

### edge 1: vault inference fails for unknown keys

**command:**
```bash
npx rhx keyrack set --key STRIPE_KEY
```

**expected behavior:**
- fails fast with clear error
- error lists available vaults
- error instructs: `--vault required for STRIPE_KEY`

---

### edge 2: single mech auto-selects

when vault supports only one mech for a key, skip mech selection.

**scenario:** if a vault only supports PERMANENT_VIA_REPLICA for a given key

**expected behavior:**
- no mech selection prompt
- auto-selects the single valid mech
- proceeds to mech guided setup

---

### edge 3: empty org list degrades gracefully

**scenario:** gh cli authenticated but user has no org access

**expected behavior:**
- fallback to manual input
- prompts for raw json blob or per-field input
- does not crash

---

### edge 4: pem file not found

**command:**
```bash
# in github app guided setup, provide nonexistent path
./nonexistent.pem
```

**expected behavior:**
- fails fast with clear error
- error shows which path was tried
- allows retry

---

### edge 5: pem file malformed

**command:**
```bash
# in github app guided setup, provide invalid file
echo "not a pem" > .temp/bad.pem
./bad.pem
```

**expected behavior:**
- fails fast with clear error
- error explains expected format
- allows retry

## pass/fail criteria

### pass if:

1. aws sso guided setup output format unchanged from prior behavior
2. `keyrack get` returns credentials (not raw profile name)
3. vault inference works for AWS_PROFILE → aws.config
4. mech inference prompts when vault supports multiple mechs
5. explicit --mech skips inference
6. incompatible vault/mech combos fail fast with alternatives
7. os.direct rejects ephemeral mechs with clear error
8. all error messages guide users to success
9. github app guided setup prompts for org → app → pem path
10. github app single org/app auto-selects when only one available
11. github app works with os.secure vault (stores json blob)
12. github app guided setup follows treestruct output format

### fail if:

1. aws sso prompts or output structure changed unexpectedly
2. `keyrack get` returns profile name instead of credentials
3. vault inference requires --vault for AWS_PROFILE
4. mech inference skipped when should prompt
5. explicit --mech still prompts for mech
6. incompatible combos silently proceed or give cryptic error
7. os.direct accepts ephemeral mechs
8. any error message leaves user without guidance
9. github app setup skips org/app/pem prompts
10. github app output does not follow treestruct format
11. github app with incompatible vault (os.direct) does not fail fast
