# distill: experience reproductions

distill for mech adapters own their prompts.

---

## experience reproductions

| experience | entry point | user actions | expected outcome | test type |
|------------|-------------|--------------|------------------|-----------|
| github app set with os.secure | cli: keyrack set | select org, select app, provide pem path | json blob stored, roundtrip verified | integration |
| github app set with 1password | cli: keyrack set | select org, select app, provide pem path | json blob stored to 1password exid | integration |
| aws sso set with aws.config | cli: keyrack set | select mech, guided sso setup | profile written to ~/.aws/config | integration |
| github app unlock transforms | cli: keyrack unlock | unlock key | ghs_ token in daemon | integration |
| incompatible vault/mech fails fast | cli: keyrack set | attempt incompatible combo | clear error with alternatives | unit |
| single org auto-selects | cli: keyrack set | (auto) | skips prompt, shows selection | unit |
| single app auto-selects | cli: keyrack set | (auto) | skips prompt, shows selection | unit |
| gh cli unavailable fallback | cli: keyrack set | (auto detect) | fallback to per-field prompts | unit |
| explicit --mech skips inference | cli: keyrack set | provide --mech flag | direct to mech setup | unit |
| vault inference from key | cli: keyrack set | provide AWS_PROFILE | infers --vault aws.config | unit |

## reproduction feasibility

### available test utilities

- `withTempHome` — isolated HOME directory
- `mockPromptHiddenInput` — queue-based stdin mock
- `genMockVaultAdapter` — mock vault adapter
- `genMockMechAdapter` — mock mech adapter (needs promptForSet)
- `jest.mock` — mock gh cli, fs, child_process

### setup required

1. temp home for integration tests
2. mock gh cli responses for org/app lists
3. mock fs for pem file reads
4. mock stdin for user selections

### test sketch: github app set

```ts
describe('mechAdapterGithubApp.promptForSet.play.integration', () => {
  const tempHome = withTempHome({ name: 'github-app-test' });
  beforeAll(tempHome.setup);
  afterAll(tempHome.teardown);

  beforeEach(() => {
    // mock gh cli
    (execSync as jest.Mock)
      .mockReturnValueOnce(JSON.stringify([{ login: 'ehmpathy' }]))
      .mockReturnValueOnce(JSON.stringify({ installations: [{ app_id: 123, id: 456 }] }));
    // mock stdin
    setMockPromptValues(['1', '1', './test.pem']);
    // mock fs
    (readFileSync as jest.Mock).mockReturnValue('-----BEGIN RSA PRIVATE KEY-----\n...');
  });

  given('[case1] user sets github app key', () => {
    when('[t0] before any changes', () => {
      then('key is absent', async () => {
        const status = await keyrackStatus({ key: 'GITHUB_TOKEN', env: 'all' });
        expect(status.state).toBe('absent');
      });
    });

    when('[t1-t4] guided setup completes', () => {
      then('json blob is stored', async () => {
        const result = await keyrackSet({
          key: 'GITHUB_TOKEN',
          vault: 'os.secure',
          mech: 'EPHEMERAL_VIA_GITHUB_APP',
        });
        expect(result.slug).toBe('ehmpathy.all.GITHUB_TOKEN');
      });
      then('output matches snapshot', () => {
        expect(stdout).toMatchSnapshot();
      });
    });

    when('[t5] unlock is called', () => {
      then('ghs_ token is available', async () => {
        const grant = await keyrackUnlock({ key: 'GITHUB_TOKEN', env: 'all' });
        expect(grant.secret).toMatch(/^ghs_/);
      });
    });
  });
});
```
