### pattern 1: slug format `$org.$env.$key`

**[REUSE]** — the slug format is the source of truth for org, env, and key name.

**relation to wish**: the slug already contains the correct org. the bug ignores this and uses `repoManifest.org` instead.

#### citation [1] — asKeyrackKeyName.ts:5-9

```ts
/**
 * .what = extract raw key name from env-scoped slug
 * .why = export layer needs raw names (AWS_PROFILE, not ehmpathy.prep.AWS_PROFILE)
 */
export const asKeyrackKeyName = (input: { slug: string }): string => {
  // split on dot, take all parts after org.env prefix
  const parts = input.slug.split('.');
  return parts.slice(2).join('.');
};
```

#### citation [2] — getKeyrackKeyGrant.ts:70-72


## the fix

Extract org from slug instead of use of `repoManifest.org`:

```ts
// before (line 257)
org: repoManifest.org,

// after
const orgFromSlug = slug.split('.')[0]!;
// ...
org: orgFromSlug,
```

This aligns fillKeyrackKeys with the correct pattern (citation [12]) where org is extracted from the slug.
