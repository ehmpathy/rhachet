## new files

### detectInvocationMethod.ts

**purpose:** detect if invoked via npx or global install

**approach:** check `npm_execpath` env var — set when via npm/npx

```typescript
export const detectInvocationMethod = (): 'npx' | 'global' => {
  // npm_execpath is set when invoked via npm/npx
  const npmExecPath = process.env.npm_execpath;
  if (npmExecPath) return 'npx';
  return 'global';
};
```

**tests:**
- given npm_execpath set → returns 'npx'
- given npm_execpath not set → returns 'global'

### execNpmInstallGlobal.ts

**purpose:** execute `npm install -g rhachet@latest`

**approach:**
1. invoke npm for global install (note: npm may be aliased to pnpm on some machines)
2. fail fast on EACCES/EPERM — no sudo hints
3. return simple result

```typescript
export const execNpmInstallGlobal = (
  input: { packages: string[] },
): { upgraded: boolean } => {
  const packagesLatest = input.packages.map((p) => `${p}@latest`);

  const result = spawnSync('npm', ['install', '-g', ...packagesLatest], {
    stdio: 'pipe',
    shell: true,
  });

  if (result.status !== 0) {
    const stderr = result.stderr?.toString() || '';
    throw new Error(`global install failed: ${stderr}`);
  }

  return { upgraded: true };
};
```

**tests:**
- given npm install succeeds → returns { upgraded: true }
- given EACCES error → throws error
- given EPERM error (Windows) → throws error

**npm-aliased-to-pnpm test strategy:**

the function invokes `npm` which may actually execute `pnpm` on some machines. both produce different stdout/stderr formats. tests must verify we handle both:

```typescript
// npm success output format
{ status: 0, stdout: 'added 1 package...', stderr: '' }

// pnpm success output format (when aliased as npm)
{ status: 0, stdout: 'Packages: +1\n...', stderr: '' }

// npm EACCES format
{ status: 1, stderr: 'npm ERR! code EACCES\nnpm ERR! syscall...' }

// pnpm EACCES format (when aliased as npm)
{ status: 1, stderr: 'ERR_PNPM_EACCES  EACCES: permission denied...' }
```

test cases:
- given stdout matches npm format, status 0 → returns { upgraded: true }
- given stdout matches pnpm format, status 0 → returns { upgraded: true }
- given stderr contains 'EACCES' (npm format) → throws error
- given stderr contains 'EACCES' (pnpm format) → throws error
- given stderr contains 'EPERM' (npm format) → throws error
- given stderr contains 'EPERM' (pnpm format) → throws error

### getGlobalRhachetVersion.ts

**purpose:** detect current global rhachet version (if installed)

**approach:** use `npm list -g rhachet --depth=0 --json` to get installed version

```typescript
export const getGlobalRhachetVersion = (): string | null => {
  const result = spawnSync('npm', ['list', '-g', 'rhachet', '--depth=0', '--json'], {
    stdio: 'pipe',
    shell: true,
  });

  if (result.status !== 0) return null;

  try {
    const output = JSON.parse(result.stdout.toString());

    // npm format: { dependencies: { rhachet: { version: '1.39.11' } } }
    if (output.dependencies?.rhachet?.version) {
      return output.dependencies.rhachet.version;
    }

    // pnpm format (when aliased as npm): [{ name: 'rhachet', version: '1.39.11' }]
    if (Array.isArray(output)) {
      const rhachet = output.find((p: { name: string }) => p.name === 'rhachet');
      return rhachet?.version ?? null;
    }

    return null;
  } catch {
    return null;
  }
};
```

**tests:**
- given rhachet installed globally → returns version string
- given rhachet not installed globally → returns null
- given npm list fails → returns null

**npm-aliased-to-pnpm test strategy:**

`npm list -g --json` produces different output when npm is aliased to pnpm:

```typescript
// npm list -g --json output
{ "dependencies": { "rhachet": { "version": "1.39.11" } } }

// pnpm list -g --json output (when aliased as npm)
[{ "name": "rhachet", "version": "1.39.11", "path": "..." }]
```

test cases:
- given stdout matches npm json format → returns version
- given stdout matches pnpm json format → returns version
- given stdout is malformed json → returns null
