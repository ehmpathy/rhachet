# blackbox criteria matrix: upgrade global rhachet

## matrix 1: invocation × --which flag

| ind: invocation | ind: --which | dep: local | dep: global | dep: exit |
|-----------------|--------------|------------|-------------|-----------|
| rhx | (default) | upgraded | upgraded | success |
| rhx | local | upgraded | skip | success |
| rhx | global | skip | upgraded | success |
| rhx | both | upgraded | upgraded | success |
| npx | (default) | upgraded | skip | success |
| npx | local | upgraded | skip | success |
| npx | global | skip | error | error |
| npx | both | upgraded | error | error |

**notes:**
- npx + global = error because no global install to upgrade
- npx + both = should this error or warn? (gap flagged below)

**gap flagged:** npx + --which both — should this:
- error immediately (global not available)?
- warn and proceed with local only?

recommendation: warn and proceed with local, consistent with rhx permission failure behavior.

## matrix 2: rhx invocation × permission state × version state

| ind: permissions | ind: local version | ind: global version | dep: local | dep: global | dep: exit |
|------------------|-------------------|---------------------|------------|-------------|-----------|
| ok | outdated | outdated | upgraded | upgraded | success |
| ok | outdated | current | upgraded | already current | success |
| ok | current | outdated | already current | upgraded | success |
| ok | current | current | already current | already current | success |
| elevated needed | outdated | outdated | upgraded | warn+hint | success |
| elevated needed | outdated | current | upgraded | already current | success |
| elevated needed | current | outdated | already current | warn+hint | success |
| elevated needed | current | current | already current | already current | success |

**notes:**
- permission failure only affects global upgrade
- local upgrade always succeeds (no elevated permissions needed)
- exit is always success — global failure does not block

## matrix 3: version mismatch scenarios

| ind: local | ind: global | dep: local action | dep: global action |
|------------|-------------|-------------------|-------------------|
| older | older | upgrade to latest | upgrade to latest |
| older | latest | upgrade to latest | no-op |
| latest | older | no-op | upgrade to latest |
| latest | latest | no-op | no-op |
| older | newer than latest | upgrade to latest | no downgrade |
| newer than latest | older | no downgrade | upgrade to latest |

**notes:**
- "newer than latest" = edge case where local/global has unreleased version
- never downgrade either install

## decomposition analysis

**dimensions count:** 3 independent variables max per matrix — acceptable

**no decomposition needed:** the matrices are clean and enumerable. the upgrade behavior has clear orthogonal dimensions:
1. invocation method (how rhachet was called)
2. --which flag (user intent)
3. system state (permissions, versions)

these are naturally separate concerns and do not require further decomposition.

## gaps identified

1. **npx + --which both:** behavior not specified. recommend: warn about global, proceed with local.

2. **npx + --which global:** error message content not specified. recommend: "global upgrade not available via npx — install globally first: npm i -g rhachet"
