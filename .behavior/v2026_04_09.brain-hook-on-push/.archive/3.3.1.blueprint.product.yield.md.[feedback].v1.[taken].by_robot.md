# response to feedback v1

## summary

the feedback restructures the CLI and removes `.agent/.hooks/`:

1. **CLI consolidation** — single command with `--when` context:
   - `rhx hook brain.onPush --when hook.onStop` — forces brain to start poll via `--start poll`
   - `rhx hook brain.onPush --when bash.onBackground` — does the poll, manages heartbeat
   - `rhx hook brain.onPush --when hook.onTool` — validates poll was started in background

2. **no `.agent/.hooks/` directory** — use BrainHooks DAO instead:
   - BrainHooks DAO writes each onPush hook to brain config
   - poll task reads onPush declarations back FROM config via DAO
   - per-enrollment configs are respected
   - zero new directory structure needed

## execution notes

### key changes made

1. **CLI renamed**: `rhx poll --for hook.onPush` → `rhx hook brain.onPush`

2. **new `--when` contexts:**
   - `--when hook.onStop` — checks heartbeat, exit(2) if absent
   - `--when hook.onTool` — validates poll was started in background
   - `--when bash.onBackground` — does the poll, manages heartbeat

3. **BrainHooks DAO replaces `.agent/.hooks/`:**
   - onPush hooks stored in brain's native config
   - poll task reads declarations via BrainHooks DAO
   - per-enrollment configs respected
   - zero `.agent/.hooks/` directory needed

4. **heartbeat replaces pid file:**
   - `--when bash.onBackground` starts heartbeat on entry, clears on exit
   - `--when hook.onStop` checks heartbeat to decide exit(0) vs exit(2)

## contract verification

| file | contract |

### compatibility analysis

| contract | current state | needed for onPush | status |
|----------|---------------|-------------------|--------|
| `BrainHookEvent` | `'onBoot' \| 'onTool' \| 'onStop'` | `'onPush'` | **requires extension** |
| `BrainHookFilter.what` | tool pattern | topic pattern | **repurpose** |
| `BrainHookFilter.roles` | does not exist | `'+mechanic'` syntax | **requires extension** |
| `BrainHooksAdapterDao.get.all({ by: { event } })` | supported | filter by `'onPush'` | **compatible** |

### verdict

design will work. contracts need extension as part of implementation phase 0, which is already documented in the blueprint filediff tree.

### codepath for active brain detection

verified the codepath for poll task to get the BrainHooks DAO:

```
detectBrainReplsInRepo(context)
  → .claude/settings.json → 'anthropic/claude/code'
  → .opencode/ → 'anomaly/opencode'

getBrainHooksAdapterByConfigImplicit({ brain }, context)
  → scans package.json for rhachet-brains-*
  → calls getBrainHooks({ brain, repoPath })
  → returns BrainHooksAdapter { slug, dao }

adapter.dao.get.all({ by: { event: 'onPush' } })
  → returns all onPush hooks
```

the DAO contracts work as-is once the event type is added.
