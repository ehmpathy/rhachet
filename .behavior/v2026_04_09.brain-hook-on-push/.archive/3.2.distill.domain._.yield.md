# distill.domain: brain-hook-on-push

## summary

distillation of domain objects and operations for brain-hook-on-push. defines the declastruct structure that enables push notifications to brain CLIs.

---

## section 1: usecases

### usecase 1: clone receives push notification

```
given(clone enrolled with onPush hook for topic 'radio' with filter '+mechanic')
  given(message dropped into .agent/.comms/radio/)
    when(poll fires)
      then(message matched by filter)
      then(hook command invoked with message payload)
      then(message deleted from directory)
```

### usecase 2: clone filters messages by role

```
given(clone with roles ['mechanic', 'driver'])
  given(message with filter '+foreman')
    when(poll fires)
      then(message not matched)
      then(message skipped)
```

### usecase 3: clone wakes from idle on message

```
given(clone idle)
  given(message arrives)
    when(onStop fires)
      then(poll checks .comms/)
      then(message found)
      then(exit(2) forces continuation)
      then(clone wakes to process message)
```

### usecase 4: malformed message quarantined

```
given(malformed message in .comms/)
  when(poll fires)
    then(parse fails)
    then(message moved to .errors/)
    then(poll continues with other messages)
```

## section 2: domain objects

### 2.1 entities

#### BrainHook (EXTEND)

**changes:** add 'onPush' to BrainHookEvent union

```typescript
// extant
type BrainHookEvent = 'onBoot' | 'onTool' | 'onStop';

// extended
type BrainHookEvent = 'onBoot' | 'onTool' | 'onStop' | 'onPush';
```

#### BrainHookPushMessage (NEW)

**purpose:** represents a message in `.agent/.comms/$topic/`

```typescript
interface BrainHookPushMessage {
  /**
   * unique identifier for this message (uuid v4)
   */
  id: string;

  /**
   * route attributes for message delivery
   */
  for: {
    /**
     * topic this message belongs to (e.g., 'radio', 'slack', 'github')
     */
    topic: string;

    /**
     * role filter expression (e.g., '+mechanic', '-foreman', 'mechanic,driver')
     */
    roles: string;
  };

  /**
   * ISO timestamp when message was created
   */
  timestamp: string;

  /**
   * arbitrary payload data
   */
  payload: unknown;
}

class BrainHookPushMessage extends DomainEntity<BrainHookPushMessage> implements BrainHookPushMessage {
  public static primary = ['id'] as const;
  public static unique = ['id'] as const;
}
```

## section 4: relationships

### treestruct of domain objects

```
BrainHook (entity)
├── BrainHookEvent (literal) - includes 'onPush'
├── BrainHookFilter (literal)
│   ├── tool? (for onTool)
│   ├── topic? (for onPush)
│   └── roles? (for onPush)
└── author (literal) - namespace ownership

BrainHookPushMessage (entity)
├── id (primary key)
├── for (literal)
│   ├── topic
│   └── roles
├── timestamp
└── payload

BrainHookPushFilter (literal)
├── includes[]
├── excludes[]
└── exact[]
```

### treestruct of operations

```
poll operations
├── getAllMessagesForTopic
├── getOneMessage
├── delMessage
└── setMessageError

filter operations
├── parseBrainHookPushFilter
└── matchesBrainHookPushFilter

hook operations (extended)
├── translateHook.toClaudeCode (onPush case)
├── translateHook.fromClaudeCode (poll detect)
└── syncOneRoleHooksIntoOneBrainRepl (unchanged)
```

### dependencies

```
onPush hook registration
  → translateHook (to create poll command)
    → parseBrainHookPushFilter (to validate filter syntax)

poll execution
  → getAllMessagesForTopic
    → getOneMessage
      → matchesBrainHookPushFilter
        → (invoke hook command)
          → delMessage OR setMessageError
```

## section 5: composition for wish fulfillment

### how objects compose to fulfill wish

1. **onPush hook declared** in role briefs
2. **BrainHook with event='onPush'** synced via syncOneRoleHooksIntoOneBrainRepl
3. **translateHook.toClaudeCode** produces:
   - poll command as onStop hook
   - command reads `.comms/$topic/`, filters by roles
   - command invokes original hook command with payload
   - command exits(2) if messages found to wake clone

4. **poll command execution:**
   - getAllMessagesForTopic scans directory
   - getOneMessage parses each file
   - matchesBrainHookPushFilter checks role match
   - original hook command invoked
   - delMessage removes consumed message
   - setMessageError quarantines failures

5. **wake mechanism:**
   - onStop hook checks for messages
   - if found, exit(2) forces continuation
   - stop_hook_active guard prevents loop
