## proposed templates for onPush

### template 1: BrainHookPushMessage entity

```typescript
export interface BrainHookPushMessage {
  id: string;
  for: BrainHookPushMessageFor;
  timestamp: string;
  payload: unknown;
}

export class BrainHookPushMessage extends DomainEntity<BrainHookPushMessage> implements BrainHookPushMessage {
  public static primary = ['id'] as const;
  public static unique = ['id'] as const;
  public static nested = { for: BrainHookPushMessageFor };
}
```

### template 2: BrainHookPushMessageFor literal

```typescript
export interface BrainHookPushMessageFor {
  topic: string;
  roles: string;
}

export class BrainHookPushMessageFor extends DomainLiteral<BrainHookPushMessageFor> implements BrainHookPushMessageFor {}
```

### template 3: OnPushHook entity

```typescript
export interface OnPushHook {
  author: string;
  filter: OnPushFilter;
  command: string;
  timeout: IsoDuration;
}

export class OnPushHook extends DomainEntity<OnPushHook> implements OnPushHook {
  public static unique = ['author', 'filter', 'command'] as const;
}
```

### template 4: poll adapter

```typescript
export const genPushAdapterForClaudeCode = (input: {
  repoPath: string;
}): PushAdapter => {
  return {
    slug: 'claude-code',
    dao: {
      get: { one, all },
      set: { findsert, upsert },
      del,
    },
  };
};
```

### template 5: onStop poll hook

```typescript
const pollOnStop = async (input: { sessionId: string; stopHookActive: boolean }) => {
  // guard infinite loop
  if (input.stopHookActive) process.exit(0);

  // poll for messages
  const messages = await scanCommsDirectory({ topics: ['slack', 'radio'] });

  // if messages found, output and force continuation
  if (messages.length > 0) {
    console.log(JSON.stringify({ messages }));
    process.exit(2);
  }

  process.exit(0);
};
```

### strong convergence (3+ sources agree)

1. **DomainEntity for identity** — sources [1], [13], [14] agree on entity pattern

## anti-patterns identified

| anti-pattern | sources | why avoid |
|--------------|---------|-----------|
| inline domain logic | [13], [14] | use domain objects for encapsulation |
| direct fs.readFile without error handle | [22], [29] | wrap in validation |
| throw in validation | [29], [30] | use safeParse for graceful handle |
| forget stop_hook_active check | [18] | causes infinite loops |
