## section 2: mock adapter patterns

### 2.1 createMockAdapter pattern

**classification:** [EXTEND] — extend for poll adapter

**citation [3]** [internal, src/domain/hooks/syncOneRoleHooksIntoOneBrainRepl.test.ts]
```typescript
const createMockAdapter = (
  hooksFound: BrainHook[] = [],
): BrainHooksAdapter & { calls: { upsert: BrainHook[]; del: BrainHook[] } } => {
  const calls = { upsert: [] as BrainHook[], del: [] as BrainHook[] };
  return {
    slug: 'test-adapter',
    calls,
    dao: {
      get: {
        one: jest.fn().mockResolvedValue(null),
        all: jest.fn().mockResolvedValue(hooksFound),
      },
      set: {
        findsert: jest.fn().mockImplementation(async ({ hook }) => {
          calls.upsert.push(hook);
          return hook;
        }),
        upsert: jest.fn().mockImplementation(async ({ hook }) => {
          calls.upsert.push(hook);
          return hook;
        }),
      },
      del: jest.fn().mockImplementation(async ({ by }) => {
        const hookToRemove = hooksFound.find(
          (h) => h.event === by.unique.event && h.command === by.unique.command,
        );
        if (hookToRemove) calls.del.push(hookToRemove);
      }),
    },
  };
};
```

**usage for onPush:**
- extend pattern for createMockPollAdapter
- track poll operations: { poll: [], consume: [], error: [] }

## section 3: temp directory patterns

### 3.1 genTestTempDir infrastructure

**classification:** [REUSE] — temp dir management

**citation [5]** [internal, src/.test/infra/genTestTempDir.ts]
```typescript
export const genTestTempDir = (input: {
  base: string;
  name: string;
}): {
  path: string;
  setup: () => void;
  teardown: () => void;
  rm: (relativePath: string) => void;
} => {
  const path = join(input.base, input.name);
  return {
    path,
    setup: () => {
      mkdirSync(path, { recursive: true });
    },
    teardown: () => {
      rmSync(path, { recursive: true, force: true });
    },
    rm: (relativePath: string) => {
      rmSync(join(path, relativePath), { recursive: true, force: true });
    },
  };
};
```

**usage for onPush:**
- create temp `.agent/.comms/` directories
- write test message files
- cleanup after tests

    encoding: 'utf-8',

  return {
    stdout: result.stdout,
    stderr: result.stderr,

| anti-pattern | why avoid | alternative |
|--------------|-----------|-------------|
| mocking fs module | hides real file ops | use temp directories |
| inline test data | clutters tests | use genTestTempDir |
| let mutations | breaks immutability | use useBeforeAll |
| missing [caseN] labels | hard to trace failures | label all given blocks |
