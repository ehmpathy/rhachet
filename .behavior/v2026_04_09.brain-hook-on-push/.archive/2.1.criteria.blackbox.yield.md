# criteria.blackbox: brain-hook-on-push

## v1 scope

**onPush only responds to NEW messages after the poll starts.**

- wake clone from idle when new message arrives
- filter by topic and role
- broadcast to multiple consumers (same message, all receive)
- detect files added after poll started

**out of scope (v1):** pre-poll messages, state management, per-consumer acknowledgment, persistence across sessions.

---

## episode experience

### usecase.1 = clone receives notification and wakes

```
given(clone is active but idle, awaited input)
  given(clone has onPush hook for topic 'slack' with filter '+mechanic')
  given(clone is enrolled with mechanic role)
    when(message drops into .agent/.comms/slack/)
      then(clone wakes from idle)
        sothat(clone responds to external events without human intervention)
      then(hook command is invoked with message)
        sothat(clone can process the notification)
```

### usecase.2 = clone ignores notification that doesn't match filter

```
given(clone is active but idle)
  given(clone has onPush hook for topic 'slack' with filter '+mechanic')
  given(clone is enrolled with driver role only)
    when(message drops into .agent/.comms/slack/ with for.roles='+mechanic')
      then(clone does not wake)
        sothat(notifications reach only intended recipients)
```

### usecase.3 = clone ignores notification for different topic

```
given(clone is active but idle)
  given(clone has onPush hook for topic 'slack')
    when(message drops into .agent/.comms/radio/)
      then(clone does not wake)
        sothat(topic route isolates notification streams)
```

### usecase.4 = multiple clones receive broadcast

```
given(clone A is active with mechanic role)
  given(clone B is active with mechanic role)
  given(both have onPush hook for topic 'radio' with filter '+mechanic')
    when(message drops into .agent/.comms/radio/ with for.roles='+mechanic')
      then(clone A wakes and processes)
      then(clone B wakes and processes)
        sothat(broadcast reaches all matched clones)
```

### usecase.5 = clone on standby wakes when notification arrives

```
given(clone was started with instruction "wait for PR feedback")
  given(clone is idle, on standby)
  given(clone has onPush hook for topic 'github' with filter '+reviewer')
    when(PR review comment drops into .agent/.comms/github/)
      then(clone wakes)
      then(clone processes the review feedback)
        sothat(clones can be left on standby for async events)
```

## exchange experience

### usecase.6 = message shape

```
given(external system wants to notify clones)
  when(message is written to .agent/.comms/$topic/$id.json)
    then(message contains id)
      sothat(message can be deduplicated)
    then(message contains for.topic)
      sothat(route knows which topic)
    then(message contains for.roles)
      sothat(filter knows which roles)
    then(message contains timestamp)
      sothat(order is possible)
    then(message contains payload)
      sothat(arbitrary data can be delivered)
```

### usecase.7 = hook declaration

```
given(role wants to receive notifications)
  when(role declares onPush hook)
    then(hook specifies filter.topic)
      sothat(hook receives only relevant topics)
    then(hook specifies filter.roles)
      sothat(hook receives only when role matches)
    then(hook specifies command)
      sothat(hook knows what to invoke)
    then(hook optionally specifies timeout)
      sothat(runaway commands are killed)
```

### usecase.8 = role filter syntax

```
given(filter.roles = '+mechanic')
  when(clone has mechanic role)
    then(filter matches)
      sothat(inclusion works)

given(filter.roles = '+mechanic')
  when(clone does not have mechanic role)
    then(filter does not match)
      sothat(inclusion excludes non-members)

given(filter.roles = '-foreman')
  when(clone has foreman role)
    then(filter does not match)
      sothat(exclusion works)

given(filter.roles = '-foreman')
  when(clone does not have foreman role)
    then(filter matches)
      sothat(exclusion includes non-members)

given(filter.roles = '+mechanic,-foreman')
  when(clone has mechanic role but not foreman)
    then(filter matches)
      sothat(combined filters work)

given(filter.roles = '+mechanic,-foreman')
  when(clone has both mechanic and foreman roles)
    then(filter does not match)
      sothat(exclusion takes precedence in combined filters)
```

### usecase.9 = poll detects new messages only

```
given(poll task starts at time T0)
  given(message msg_old.json was written before T0)
  given(message msg_new.json is written after T0)
    when(poll scans .agent/.comms/slack/)
      then(poll ignores msg_old.json)
        sothat(onPush only responds to NEW messages after poll starts)
      then(poll processes msg_new.json)
        sothat(new messages wake the clone)
```

note: v1 scope is wake-from-idle only. no state management, no persistence across sessions.

### usecase.10 = hook command invocation

```
given(notification matches filter)
  when(hook command is invoked)
    then(command receives message path as argument)
      sothat(command can read the message)
    then(command runs in clone's context)
      sothat(command has access to clone's tools and state)
```

## edge cases

### usecase.11 = malformed message

```
given(file in .agent/.comms/slack/ is not valid json)
  when(poll detects the file)
    then(error is logged)
    then(file is moved to .comms/.errors/)
      sothat(malformed messages don't block queue)
```

### usecase.12 = hook command fails

```
given(notification matches filter)
  when(hook command exits with error)
    then(error is logged)
    then(message is moved to .comms/.errors/)
      sothat(failed messages can be inspected)
```

### usecase.13 = hook command times out

```
given(notification matches filter)
  given(hook has timeout of PT30S)
  when(hook command runs longer than 30 seconds)
    then(command is killed)
    then(timeout error is logged)
      sothat(runaway commands don't block queue)
```
