# blueprint.product: brain-hook-on-push

## summary

add `onPush` hook event to enable push notifications into brain CLIs via file-based message poll.

**v1 scope:** onPush only responds to NEW messages after poll starts. no state management, no consumer considerations — just filters and "what's new?". wake-from-idle only (notify-while-active is subsequent PR).

**core mechanism:**
1. producer drops message into `.agent/.comms/$topic/`
2. background poll task scans directory, matches role filter, invokes hook
3. background task completion wakes idle clone (empirically validated 2026-04-11)

**note:** onStop hook with exit(2) ensures poll task is active, not for wake. background task completion alone wakes the CLI.

---

## flowpath: who runs what, when

this section specifies exactly which actor runs which command at which moment.

### actors

| actor | description |
|-------|-------------|
| **human** | runs `rhx init --hooks` once to set up hooks |
| **claude code harness** | runs onTool/onStop hooks automatically |
| **background poll process** | single process polls ALL topics, invokes matched hook commands |
| **clone (claude)** | the AI that receives the wake signal and processes notifications |

### example hook declarations

```typescript
// in rhachet-roles-ehmpathy/src/roles/mechanic/index.ts
onPush: [
  {
    filter: { topic: 'slack', roles: '+mechanic' },
    command: 'npx rhachet run --skill slack.notification.get --when hook.onPush',
  },
  {
    filter: { topic: 'github', roles: '+mechanic' },
    command: 'npx rhachet run --skill github.notification.get --when hook.onPush',
  },
]
```

### phase 1: setup (once, by human)

**when:** human runs `rhx init --hooks` after roles are enrolled

**who runs:** human (terminal)

**what happens:**

```
human$ rhx init --hooks
         │
         ├─[1] writes onPush hooks to brain config via BrainHooks DAO
         │     → each onPush declaration stored in brain's native config
         │     → e.g., .claude/settings.local.json for claude code
         │
         └─[2] writes poll lifecycle hooks to brain config
               → Stop: rhx hook brain.onPush --when hook.onStop
               → PreToolUse: rhx hook brain.onPush --when hook.onTool
```

**onPush hooks stored in brain config (via BrainHooks DAO):**

the poll task reads onPush declarations back FROM the brain config. zero `.agent/.hooks/` directory needed. per-enrollment configs are respected.

**translation example:**

rhachet roles declare onPush hooks:
```typescript
// in rhachet-roles-ehmpathy/src/roles/mechanic/index.ts
export const mechanicRole = {
  hooks: {
    onPush: [
      {
        filter: { what: 'slack', roles: '+mechanic' },
        command: 'npx rhachet run --skill slack.notification.get --when hook.onPush',
        timeout: 'PT30S',
      },
    ],
  },
};

// in rhachet-roles-ehmpathy/src/roles/driver/index.ts
export const driverRole = {
  hooks: {
    onPush: [
      {
        filter: { what: 'slack', roles: '+driver' },
        command: 'npx rhachet run --skill slack.driver.notify --when hook.onPush',
        timeout: 'PT30S',
      },
    ],
  },
};
```

BrainHooksAdapter translates to claude code format (.claude/settings.local.json):
```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "rhx hook brain.onPush --when hook.onStop",
            "timeout": 5000,
            "author": "repo=ehmpathy/role=mechanic, repo=ehmpathy/role=driver"
          }
        ]
      }
    ],
    "PreToolUse": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "rhx hook brain.onPush --when hook.onTool",
            "timeout": 5000,
            "author": "repo=ehmpathy/role=mechanic, repo=ehmpathy/role=driver"
          }
        ]
      }
    ],
    "Push": [
      {
        "matcher": "slack",
        "hooks": [
          {
            "type": "command",
            "command": "npx rhachet run --skill slack.notification.get --when hook.onPush",
            "timeout": 30000,
            "author": "repo=ehmpathy/role=mechanic",
            "filter": { "roles": "+mechanic" }
          },
          {
            "type": "command",
            "command": "npx rhachet run --skill slack.driver.notify --when hook.onPush",
            "timeout": 30000,
            "author": "repo=ehmpathy/role=driver",
            "filter": { "roles": "+driver" }
          }
        ]
      }
    ]
  }
}
```

**multi-role collision resolution:**
- when multiple roles register onPush for the same topic, their hooks append to the same entry's `hooks` array
- each hook preserves its own `author` field for traceability
- the poll lifecycle hooks (Stop, PreToolUse) are shared — author field contains comma-separated list of all roles that registered onPush

**note:** the `Push` section is NOT a native claude code event — it's a custom section that the poll task reads via `adapter.dao.get.all({ by: { event: 'onPush' } })`. the Stop and PreToolUse hooks manage the poll lifecycle.

**poll lifecycle:**

- `--when hook.onStop` — checks heartbeat, exit(2) if poll not active (forces `--start poll`)
- `--when hook.onTool` — validates poll was started in background, exit(2) if foreground
- `--when bash.onBackground` — does the poll, manages heartbeat (started via `--start poll`)

**key insight:** ONE poller command, reads ALL onPush hooks via BrainHooks DAO. clone's enrolled roles read from session context. per-enrollment configs respected. zero `.agent/.hooks/` directory needed.

### phase 2: session start (automatic, by harness)

**when:** clone starts a session, uses first tool

**who runs:** claude code harness (automatic)

**what happens:**

```
clone uses tool (e.g., Read, Bash)
         │
         ▼
claude code harness fires PreToolUse hook
         │
         ▼
harness runs: rhx hook brain.onPush --when hook.onTool
         │
         ├─[1] checks if poll was started in background mode
         │     → if yes: exit 0 (all good)
         │     → if no (foreground): exit 2 (must run in background)
         │
         └─[2] on first tool use, Stop will have started poll via --start poll
```

---

### phase 3: clone goes idle (automatic, by harness)

**when:** clone finishes work, awaits human input

**who runs:** claude code harness (automatic)

**what happens:**

```
clone finishes work, stops (awaits input)
         │
         ▼
claude code harness fires Stop hook
         │
         ▼
harness runs: rhx hook brain.onPush --when hook.onStop
         │
         ├─[1] checks heartbeat (is poll active?)
         │
         ├─ if active → exit 0
         │     → harness sees exit 0, no action
         │     → clone remains idle, poll continues in background
         │
         └─ if NOT active → exit 2
               → forces clone to run: rhx hook brain.onPush --start poll
               → starts: rhx hook brain.onPush --when bash.onBackground &
               │
               └─ background process:
                    ├─ starts heartbeat
                    ├─ records start time T0
                    ├─ reads clone's enrolled roles from session context
                    ├─ reads ALL onPush hooks via BrainHooks DAO
                    └─ begins poll loop on ALL topics (.agent/.comms/$topic/)
```

**the poll registration loop:**

```
Stop fires
    │
    └→ rhx hook brain.onPush --when hook.onStop
           │
           ├→ heartbeat active? exit 0 (all good, stay idle)
           │
           └→ heartbeat absent? exit 2 (force re-engage)
                  │
                  └→ re-engage runs: rhx hook brain.onPush --start poll
                         │
                         └→ starts: rhx hook brain.onPush --when bash.onBackground &
                                │
                                ├→ heartbeat started
                                └→ poll loop active
```

---

### phase 4: message arrives (automatic, by poll process)

**when:** external system writes message to any `.agent/.comms/$topic/`

**who runs:** background poll process (`--when bash.onBackground`, polls ALL topics)

**what happens:**

```
external system writes: .agent/.comms/slack/msg_123.json
         │
         ▼
background poll process detects new file in slack/
         │
         ├─[1] checks file modification time > T0 (start time)
         │     → if older: skip (prior message, before poll started)
         │
         ├─[2] looks up onPush hook for this topic
         │     → reads via BrainHooks DAO
         │
         ├─[3] parses message JSON, extracts for.roles (e.g., '+mechanic')
         │
         ├─[4] checks: do clone's roles (from session context) satisfy BOTH filters?
         │     → hook filter: '+mechanic' (from hook declaration)
         │     → message filter: '+mechanic' (from message)
         │     → clone has roles: ['mechanic', 'driver']
         │     → match? YES
         │
         └─[5] invokes command from hook with message path:
               │
               ▼
               npx rhachet run --skill slack.notification.get \
                   --when hook.onPush \
                   --message .agent/.comms/slack/msg_123.json
               │
               ▼
               command outputs to stdout
               │
               ▼
               stdout output wakes clone from idle
               │
               ▼
               clone (claude) receives notification, processes it
```

---

### phase 5: clone processes notification

**when:** clone wakes from idle due to stdout output

**who runs:** clone (claude)

**what happens:**

```
clone wakes, sees notification in context
         │
         ▼
clone processes the notification
         │
         ├─ reads message payload
         ├─ decides action
         └─ acts (responds, creates PR, broadcasts to peers, etc.)
         │
         ▼
clone finishes, goes idle again
         │
         ▼
→ back to phase 3 (onStop fires, checks poll active)
```

## cli contract

**command:** `rhx hook brain.onPush`

```bash
# Stop hook — checks heartbeat, forces poll start if not active
rhx hook brain.onPush --when hook.onStop

# PreToolUse hook — validates poll was started in background
rhx hook brain.onPush --when hook.onTool

# background poll — started via --start poll
rhx hook brain.onPush --when bash.onBackground &
```

**args:**
| arg | required | description |
|-----|----------|-------------|
| `--when` | yes | context: `hook.onStop`, `hook.onTool`, or `bash.onBackground` |
| `--start` | no | action: `poll` to start background poll |

**what the poll reads at runtime:**
- **topics**: via BrainHooks DAO (onPush declarations in brain config)
- **clone's roles**: from session context (enrolled roles)
- **hook commands**: from each onPush declaration's `command` field

**how to get the BrainHooks DAO for the active brain:**

```
detectBrainReplsInRepo(context)
  → .claude/settings.json present → 'anthropic/claude/code'
  → .opencode/ directory present → 'anomaly/opencode'

getBrainHooksAdapterByConfigImplicit({ brain }, context)
  → scans package.json for rhachet-brains-* packages
  → calls getBrainHooks({ brain, repoPath }) on each
  → returns matched BrainHooksAdapter { slug, dao }

adapter.dao.get.all({ by: { event: 'onPush' } })
  → returns all onPush hooks for that brain
```

**codepath for poll task:**
1. detect active brain: `detectBrainReplsInRepo(context)` → `['anthropic/claude/code']`
2. get adapter: `getBrainHooksAdapterByConfigImplicit({ brain: 'anthropic/claude/code' }, context)`
3. read hooks: `adapter.dao.get.all({ by: { event: 'onPush' } })`
4. for each hook: extract topic from `filter.what`, roles from `filter.roles`

**extensions needed for onPush:**

| component | current | needed | status |
|-----------|---------|--------|--------|
| `BrainHookEvent` | `'onBoot' \| 'onTool' \| 'onStop'` | add `'onPush'` | extend |
| `BrainHookFilter` | `{ what, when? }` | add `roles?: string` | extend |
| `translateHookToClaudeCode` | handles onBoot, onTool, onStop | handle onPush | extend |
| `translateHookFromClaudeCode` | handles onBoot, onTool, onStop | handle onPush | extend |

the extant DAO contracts (`get.all`, `set.upsert`, `del`) work as-is for onPush once the event type is added.

**behavior (`--when hook.onStop`):**
1. checks heartbeat (is poll active?)
2. if active: exit 0 (all good, stay idle)
3. if not active: exit 2 (forces clone to run `--start poll`)

**behavior (`--when hook.onTool`):**
1. checks if poll was started in background mode
2. if yes: exit 0
3. if no (foreground): exit 2 (must run in background)

**behavior (`--when bash.onBackground`):**
1. starts heartbeat (so `--when hook.onStop` knows poll is active)
2. reads clone's enrolled roles from session context
3. reads ALL onPush hooks via BrainHooks DAO
4. records start time T0
5. poll loop detects NEW files (after T0) in ALL `.agent/.comms/$topic/` directories
6. for each new file:
   - looks up hook for that topic
   - parses message's `for.roles` filter
   - checks if clone's roles satisfy BOTH filters (hook filter AND message filter)
   - if match, invokes command from hook with `--message $path` appended
7. outputs to stdout (wakes clone from idle)
8. on exit: clears heartbeat

**heartbeat location:**
- path: `.agent/.cache/for.actor=$sessionId/hook.brain.onpush.heartbeat`
- content: timestamp of last heartbeat (ISO 8601)
- liveness check: file exists AND timestamp < 30s ago
- session-scoped: each actor session has its own heartbeat, no collision

// translateHook.integration.test.ts
given('[case1] two roles register onPush for same topic', () => {
  const mechanicHook: BrainHook = {
    author: 'repo=ehmpathy/role=mechanic',
    event: 'onPush',
    command: 'npx rhachet run --skill slack.notification.get',
    timeout: 'PT30S',
    filter: { what: 'slack', roles: '+mechanic' },
  };
  const driverHook: BrainHook = {
    author: 'repo=ehmpathy/role=driver',
    event: 'onPush',
    command: 'npx rhachet run --skill slack.driver.notify',
    timeout: 'PT30S',
    filter: { what: 'slack', roles: '+driver' },
  };

  when('[t0] both hooks are translated to claude code config', () => {
    then('Push entry has matcher "slack"', () => {
      expect(result.Push[0].matcher).toEqual('slack');
    });

    then('Push entry hooks array contains both commands', () => {
      expect(result.Push[0].hooks).toHaveLength(2);
      expect(result.Push[0].hooks[0].command).toContain('slack.notification.get');
      expect(result.Push[0].hooks[1].command).toContain('slack.driver.notify');
    });

    then('each hook preserves its author', () => {
      expect(result.Push[0].hooks[0].author).toEqual('repo=ehmpathy/role=mechanic');
      expect(result.Push[0].hooks[1].author).toEqual('repo=ehmpathy/role=driver');
    });

    then('Stop hook author is comma-separated list', () => {
      expect(result.Stop[0].hooks[0].author).toEqual(
        'repo=ehmpathy/role=mechanic, repo=ehmpathy/role=driver'
      );
    });

    then('PreToolUse hook author is comma-separated list', () => {
      expect(result.PreToolUse[0].hooks[0].author).toEqual(
        'repo=ehmpathy/role=mechanic, repo=ehmpathy/role=driver'
      );
    });
  });
});
