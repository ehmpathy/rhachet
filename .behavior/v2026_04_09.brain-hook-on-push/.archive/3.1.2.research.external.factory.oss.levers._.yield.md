## summary

research on open source tools that can be leveraged for brain-hook-on-push implementation. focus on production codepaths for message poll, validation, and filter operations.

## section 1: file system operations

### 1.1 native node.js fs module

**assessment:** recommended — no external dependency needed

**citation [1]** [official, Node.js]
> "fs.readdir(path[, options], callback): Reads the contents of a directory. The optional options argument can be a string or object."
— [Node.js fs Documentation](https://nodejs.org/api/fs.html#fsreaddirpath-options-callback)

**pros:**
- zero dependencies
- stable API (LTS support)
- async/await via fs/promises

**cons:**
- no glob support (need separate library)
- no recursive watch (not needed for poll)

**usage for onPush:**
```typescript
import { readdir, readFile, unlink, rename } from 'node:fs/promises';

// poll .comms/$topic/ directory
const files = await readdir(`.agent/.comms/${topic}`);
for (const file of files) {
  if (file.endsWith('.json')) {
    const content = await readFile(join(dir, file), 'utf-8');
    // process message
  }
}
```

### 1.2 chokidar (file system watcher)

**assessment:** not recommended — poll is more reliable

**citation [2]** [official, npm]
> "Minimal and efficient cross-platform file watch library. Uses native fsevents on macOS, otherwise falls back to recursive fs.watch."
— [npm chokidar](https://www.npmjs.com/package/chokidar)

**citation [3]** [practitioner, GitHub]
> "chokidar is used by many projects: webpack, karma, pm2, browserify, and others."
— [GitHub chokidar](https://github.com/paulmillr/chokidar)

**why not recommended:**
- poll is more reliable (no event loss)
- inotify limitations affect chokidar on Linux
- adds dependency for no benefit in poll architecture

## section 2: schema validation

### 2.1 zod

**assessment:** recommended for message validation

**citation [4]** [official, npm]
> "TypeScript-first schema validation with static type inference. 3.5M+ weekly downloads."
— [npm zod](https://www.npmjs.com/package/zod)

**citation [5]** [practitioner, 2024]
> "Zod has become the de facto standard for runtime validation in TypeScript projects, used by Prisma, tRPC, and many others."
— [TypeScript Schema Validation Comparison](https://blog.logrocket.com/typescript-schema-validation-libraries-compared/)

**usage for onPush:**
```typescript
import { z } from 'zod';

const BrainHookPushMessageSchema = z.object({
  id: z.string().uuid(),
  for: z.object({
    topic: z.string(),
    roles: z.string(),
  }),
  timestamp: z.string().datetime(),
  payload: z.unknown(),
});

// validate message
const message = BrainHookPushMessageSchema.parse(JSON.parse(content));
```

**pros:**
- TypeScript inference
- excellent error messages
- 3.5M+ weekly downloads
- active maintenance

**cons:**
- adds ~50KB to bundle
- may be overkill for simple validation

### 2.2 native JSON.parse + type guards

**assessment:** acceptable alternative

**citation [6]** [official, MDN]
> "JSON.parse() parses a JSON string. Can throw SyntaxError if the string to parse is not valid JSON."
— [MDN JSON.parse](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/JSON/parse)

**usage for onPush:**
```typescript
const isBrainHookPushMessage = (obj: unknown): obj is BrainHookPushMessage => {
  if (typeof obj !== 'object' || obj === null) return false;
  const msg = obj as Record<string, unknown>;
  return (
    typeof msg.id === 'string' &&
    typeof msg.for === 'object' &&
    typeof msg.timestamp === 'string' &&
    'payload' in msg
  );
};

const parsed = JSON.parse(content);
if (!isBrainHookPushMessage(parsed)) throw new Error('invalid message');
```

**pros:**
- zero dependencies
- minimal bundle size

**cons:**
- manual type guards
- less comprehensive error messages

## section 7: role filter parse

### 7.1 custom parser (recommended)

**assessment:** recommended — simple enough to implement

role filter syntax is simple: `+role`, `-role`, comma-separated. no external library needed.

**proposed implementation:**
```typescript
const parseBrainHookPushFilter = (filter: string): BrainHookPushFilter => {
  const parts = filter.split(',').map(p => p.trim());
  const includes: string[] = [];
  const excludes: string[] = [];
  const exact: string[] = [];

  for (const part of parts) {
    if (part.startsWith('+')) includes.push(part.slice(1));
    else if (part.startsWith('-')) excludes.push(part.slice(1));
    else exact.push(part);
  }

  return { includes, excludes, exact };
};

const matchesFilter = (
  cloneRoles: string[],
  filter: BrainHookPushFilter,
): boolean => {
  // exact match: clone has exactly these roles
  if (filter.exact.length > 0) {
    const sorted = [...cloneRoles].sort();
    const expected = [...filter.exact].sort();
    return JSON.stringify(sorted) === JSON.stringify(expected);
  }

  // inclusion check
  for (const role of filter.includes) {
    if (!cloneRoles.includes(role)) return false;
  }

  // exclusion check
  for (const role of filter.excludes) {
    if (cloneRoles.includes(role)) return false;
  }

  return true;
};
```

```typescript
import { execa } from 'execa';

const result = await execa(hook.command, {
  timeout: hook.timeout ?? 30_000,
  input: JSON.stringify(message),

  // hook blocked, take action
}
```
