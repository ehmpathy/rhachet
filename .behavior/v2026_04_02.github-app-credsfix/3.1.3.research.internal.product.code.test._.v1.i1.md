**file**: `src/contract/cli/invokeRun.stdin.integration.test.ts`

**relation to wish**: tests stdin at skill level. we need to extend to test multiline stdin.

```ts

when('rhachet run --skill is invoked with piped stdin', () => {
  then('the skill executable should receive the stdin data', () => {
    const stdinData = JSON.stringify({
      input: 'test data',
      value: 42,
    });

    const result = invokeRhachetRun({
      skill: 'read-stdin',
      cwd: testDir.path,
      stdin: stdinData,
    });
    // ...
```

**note**: uses single-line JSON. does NOT test multi-line stdin.

## pattern 2: keyrack set acceptance test [EXTEND]

**file**: `blackbox/cli/keyrack.set.acceptance.test.ts`

**what it does**: acceptance tests for `keyrack set` command.

**relation to wish**: tests keyrack set via stdin. we need to extend with multi-line stdin test.

### citation 2: extant keyrack set test with stdin

```ts
// blackbox/cli/keyrack.set.acceptance.test.ts:18-41
when('[t0] keyrack set --key NEW_KEY --mech PERMANENT_VIA_REPLICA --vault os.direct --json', () => {
  const result = useBeforeAll(async () =>
    invokeRhachetCliBinary({
      args: [
        'keyrack',
        'set',
        '--key',
        'NEW_KEY',
        '--env',
        'test',
        '--mech',
        'PERMANENT_VIA_REPLICA',
        '--vault',
        'os.direct',
        '--json',
      ],
      cwd: repo.path,
      env: { HOME: repo.path },
      stdin: 'new-key-test-value\n',  // <-- single line stdin
    }),
  );

  then('exits with status 0', () => {
    expect(result.status).toEqual(0);
  });
  // ...
});
```

**note**: uses single-line stdin (`'new-key-test-value\n'`). does NOT test multi-line stdin.

## pattern 3: invokeRhachetCliBinary helper [REUSE]

**file**: `blackbox/.test/infra/invokeRhachetCliBinary.ts`

**what it does**: invokes rhachet CLI as subprocess with stdin support.

**relation to wish**: can use extant helper to pipe multi-line stdin in new tests.

### citation 3: helper supports stdin

```ts
// usage pattern from keyrack.set.acceptance.test.ts:34-37
invokeRhachetCliBinary({
  args: ['keyrack', 'set', ...],
  cwd: repo.path,
  env: { HOME: repo.path },
  stdin: 'new-key-test-value\n',  // stdin is supported
}),
```
