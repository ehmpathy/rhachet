## code changes

### promptHiddenInput.ts — piped stdin branch

**before** (lines 13-36):
```ts
if (!process.stdin.isTTY) {
  // read from stdin as a line
  const rl = readline.createInterface({
    input: process.stdin,
    output: process.stdout,
    terminal: false,
  });
  return new Promise((promiseCallback) => {
    let done = false;
    rl.once('line', (line) => {        // <-- BUG: only reads ONE line
      if (done) return;
      done = true;
      rl.close();
      promiseCallback(line);
    });
    // handle stdin close without input (e.g., CI environment)
    rl.once('close', () => {
      if (done) return;
      done = true;
      promiseCallback('');
    });
  });
}
```

**after**:
```ts
if (!process.stdin.isTTY) {
  // read ALL stdin content, not just first line
  const chunks: string[] = [];
  process.stdin.setEncoding('utf8');
  for await (const chunk of process.stdin) {
    chunks.push(chunk as string);
  }
  const content = chunks.join('');
  // trim trailing newline if present (stdin often ends with \n)
  return content.endsWith('\n') ? content.slice(0, -1) : content;
}
```


**after**:
```ts
if (!process.stdin.isTTY) {
  // read ALL stdin content, not just first line
```

## dependencies

## notes

- the fix function must be `async` to use `for await`
- both functions already return `Promise<string>`, so async is compatible
- the trailing newline trim preserves extant behavior where single-line input ending in `\n` returns without the newline
