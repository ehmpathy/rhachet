# research: internal prod code patterns

## pattern 1: promptHiddenInput [REPLACE]

**file**: `src/infra/promptHiddenInput.ts`

**what it does**: prompts user for secret input with hidden echo (passwords).

**relation to wish**: this is the PRIMARY bug location. when stdin is piped, it reads only the first line.

### citation 1: the bug

```ts
// src/infra/promptHiddenInput.ts:13-35
if (!process.stdin.isTTY) {
  // read from stdin as a line
  const rl = readline.createInterface({
    input: process.stdin,
    output: process.stdout,
    terminal: false,
  });
  return new Promise((promiseCallback) => {
    let done = false;
    rl.once('line', (line) => {        // <-- BUG: only reads ONE line
      if (done) return;
      done = true;
      rl.close();
      promiseCallback(line);
    });
    // handle stdin close without input (e.g., CI environment)
    rl.once('close', () => {
      if (done) return;
      done = true;
      promiseCallback('');
    });
  });
}
```

**problem**: `rl.once('line', ...)` reads exactly one line. multi-line input is truncated.

### citation 2: interactive path (works correctly)

```ts
// src/infra/promptHiddenInput.ts:38-95
return new Promise((promiseCallback, promiseReject) => {
  // write the prompt

});
```

**note**: interactive path accumulates chars until enter. works correctly.

## fix approach

replace the piped-stdin branch in both files to read ALL stdin content:

```ts
// read ALL stdin, not just first line
const chunks: string[] = [];
process.stdin.setEncoding('utf8');
for await (const chunk of process.stdin) {
  chunks.push(chunk);
}
return chunks.join('');
```

or use stream consumption pattern:

```ts
import { text } from 'node:stream/consumers';
const content = await text(process.stdin);
return content;
```
