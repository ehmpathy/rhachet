# blocker.1: failhide via bare catch block

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/contract/cli/invokeKeyrack.ts (firewall action JSON parse)

In the firewall command, a try/catch around JSON.parse uses bare catch {} to swallow any error (including unexpected) and unconditionally throw a new BadRequestError without rethrowing originals or using error wrapping. Per maintenance hazards rule (and refs on failhide), catch must allowlist only expected errors, rethrow unexpected, and avoid silent swallows. This hides real errors, causing debug hours or silent defects. Use HelpfulError.wrap or explicit allowlist + rethrow instead.

**snippet**:
```ts
        try {
          secrets = JSON.parse(rawJson);
        } catch {
          throw new BadRequestError('malformed secrets JSON', {
            source,
            hint: 'ensure the input is valid JSON object',
          });
        }
```
