# blackbox criteria: onTalk hook

## episode experience

### usecase.1 = declare onTalk in role and link

```
given(role declares onTalk hook)
  when(developer runs `rhachet roles link --role myrole`)
    then(settings.json contains UserPromptSubmit hook for claude code)
      sothat(hook fires when user submits prompt)
    then(opencode plugin contains chat.message hook)
      sothat(hook fires when user submits prompt in opencode)
    then(hook command matches declared command)
    then(hook timeout matches declared timeout)
    then(hook author is namespaced to role)
      sothat(hook can be identified for cleanup)
```

### usecase.2 = unlink removes onTalk hook

```
given(role with onTalk hook is linked)
  when(developer runs `rhachet roles unlink --role myrole`)
    then(settings.json no longer contains the hook)
      sothat(orphan hooks do not accumulate)
    then(opencode plugin is removed)
    then(other roles' hooks remain untouched)
```

### usecase.3 = hook fires on prompt submission

```
given(role with onTalk hook is linked)
  given(claude code session is active)
    when(user submits a prompt)
      then(hook command executes)
      then(hook receives prompt content via stdin)
        sothat(hook can read what user asked)
      then(hook can emit output to influence context)
      then(hook can block prompt via non-zero exit)
```

### usecase.4 = multiple onTalk hooks

```
given(role declares multiple onTalk hooks)
  when(developer runs `rhachet roles link --role myrole`)
    then(all hooks are synced to settings.json)
    then(each hook has unique author namespace)
  when(user submits a prompt)
    then(all hooks execute)
```

### usecase.5 = onTalk alongside other hook types

```
given(role declares onBoot, onTool, onStop, and onTalk hooks)
  when(developer runs `rhachet roles link --role myrole`)
    then(all hook types are synced)
    then(each hook type maps to correct brain event)
      sothat(hooks fire at appropriate moments)
```

## exchange experience

### input: role declaration

```typescript
hooks: {
  onBrain: {
    onTalk: [{
      command: string;       // path to executable
      timeout: IsoDuration;  // e.g., 'PT5S'
    }];
  };
}
```

**constraints:**
- command is required
- timeout is required
- filter.what is not supported (single event, no sub-events)

### output: claude code settings.json

```json
{
  "hooks": {
    "UserPromptSubmit": [{
      "matcher": "*",
      "hooks": [{
        "type": "command",
        "command": "<declared command>",
        "timeout": <timeout in ms>
      }]
    }]
  }
}
```

### output: opencode plugin

```json
{
  "author": "repo=<repo>/role=<role>",
  "event": "onTalk",
  "command": "<declared command>",
  "timeout": <timeout in ms>
}
```

### error: invalid timeout

```
given(role declares onTalk with invalid timeout format)
  when(developer runs `rhachet roles link --role myrole`)
    then(link fails with clear error message)
      sothat(developer knows what to fix)
```

### error: command not found

```
given(role declares onTalk with non-existent command path)
  when(hook fires)
    then(hook fails)
    then(error is visible in brain output)
      sothat(developer can diagnose)
```

## boundary conditions

### empty onTalk array

```
given(role declares empty onTalk array)
  when(developer runs `rhachet roles link --role myrole`)
    then(no UserPromptSubmit hooks are added)
    then(link succeeds)
```

### onTalk not declared

```
given(role does not declare onTalk)
  when(developer runs `rhachet roles link --role myrole`)
    then(no UserPromptSubmit hooks are added)
    then(other declared hooks are synced normally)
```

### idempotent link

```
given(role with onTalk is already linked)
  when(developer runs `rhachet roles link --role myrole` again)
    then(hooks remain unchanged)
    then(no duplicate hooks are created)
      sothat(link is safe to run multiple times)
```
