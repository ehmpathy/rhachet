# blueprint: rhachet init --keys

## summary

**implementation status: needs update**

extant components require modification:
- domain operations: discoverRoleKeyracks needs `roles` input, initKeyrackRepoManifest needs validation
- CLI flags: composable (no early returns), `--keys` requires `--roles`
- acceptance tests: init.keys.acceptance.test.ts needs new test cases
- test fixtures: with-role-keyrack (extant), need fixture for role-without-keyrack case

**scope (per wish):**
- repo keyrack only (`.agent/keyrack.yml`) — no `--for` flag (use `keyrack init` for target selection)
- `--roles` flag required — explicit control over which role keyracks to extend
- org auto-detected from package.json only (fallback chain: organization → scoped name → repository field)
- no `--org` flag — for custom org, use `keyrack init` instead
- extends array populated from specified role keyracks only (roles without keyracks are skipped)
- composable flags: `--roles`, `--keys`, `--hooks`, `--prep`, `--config` all execute in order (no early returns)

## contracts

### cli contract

```sh
# basic usage (--roles required)
npx rhachet init --keys --roles mechanic

# multiple roles
npx rhachet init --keys --roles mechanic dispatcher

# flag composition (all flags execute, no early returns)
npx rhachet init --keys --roles mechanic --hooks

# errors:
# npx rhachet init --keys                    # error: --roles required
```

**note:** org is auto-detected from package.json. no `--org` flag. for custom org or host keyrack, use `keyrack init` instead.

## test coverage

### acceptance tests (blackbox)

| test case | fixture | command | assertion |
|-----------|---------|---------|-----------|
| fresh repo setup | with-role-keyrack | `init --keys --roles tester` | creates .agent/keyrack.yml with org + extends |
| org from organization field | custom package.json | `init --keys --roles tester` | org matches organization |
| org from scoped name | @org/pkg package.json | `init --keys --roles tester` | org extracted from scope |
| org from repository field | github:org/repo package.json | `init --keys --roles tester` | org extracted from owner |
| org not detectable | unscoped package.json | `init --keys --roles tester` | error, redirect to keyrack init |
| extends append | extant manifest + new keyrack | `init --keys --roles tester newrole` | new keyrack appended |
| extends preserve | extant extends | `init --keys --roles tester` | prior entries preserved |
| scoped extends | multi-role fixture | `init --keys --roles mechanic` | only mechanic keyrack in extends |
| --roles required | any | `init --keys` | error: --roles required |
| role lacks keyrack | role without keyrack | `init --keys --roles norack` | creates manifest with empty extends (role skipped) |
| flag composition | any | `init --keys --roles tester --hooks` | all flags execute (composable) |
| idempotent re-run | up-to-date | `init --keys --roles tester` | no changes, found effect |

## output format

tree format with emoji prefix (matches extant CLI patterns):

```
🔑 keyrack init
   ├─ org: ehmpathy
   ├─ extends:
   │  ├─ .agent/repo=bhuild/role=dispatcher/keyrack.yml
   │  └─ .agent/repo=ehmpathy/role=mechanic/keyrack.yml
   └─ created .agent/keyrack.yml
```

effect messages:
- `created {path}` — manifest was absent
- `updated {path}` — extends were merged
- `found {path} (no changes)` — manifest was up-to-date
