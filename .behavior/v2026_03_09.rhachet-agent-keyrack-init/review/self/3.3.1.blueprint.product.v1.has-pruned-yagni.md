## analysis of questionable items

### org fallback chain (organization → scoped name → repository)

**was this requested?** partially.

wish says: "org based on github repo owner" — maps to repository field.

**is organization field YAGNI?** no.
- many packages have explicit `organization` field
- if user declares it, we should honor it
- one field check is trivial

**is scoped name YAGNI?** no.
- `@org/pkg` is npm convention
- most ehmpathy packages are scoped
- org extract from scope is trivial

**decision:** keep fallback chain. minimal code, maximum compatibility.

### `--org` flag

**was this requested?** not explicitly.

**is it YAGNI?** no.
- what happens when org can't be detected?
- without `--org`, user is stuck
- this is error recovery, not feature creep

**decision:** keep `--org` flag for error recovery.

### org-related test cases

**are org fallback tests YAGNI?** no.
- if we ship fallback chain, we must test it
- untested code paths are liability
- these tests verify pit-of-success works

**decision:** keep all org tests.
