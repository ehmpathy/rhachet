### rule: idempotent mutations (rule.require.idempotent-procedures)

**rule quote from boot context:**
> procedures idempotent unless marked; handle twice no double effects
> allowed verbs: findsert (find extant or insert), upsert (update or insert), delete

**blueprint return type (line 111):**
```ts
Promise<{ effect: 'created' | 'found' | 'updated'; manifestPath: string; org: string; extends: string[] }>
```

**idempotency check:**

| effect | semantics | idempotent? |
|--------|-----------|-------------|
| `found` | manifest extant, no changes | ✅ yes |
| `created` | manifest absent → create | ✅ yes (findsert) |
| `updated` | extends merged | ✅ yes (append-only) |

**retry safety:**
- first call: creates manifest → effect = `created`
- second call: manifest extant → effect = `found`
- no duplicate manifests, no data loss

**verdict:** ✅ operation is idempotent (findsert semantics)
