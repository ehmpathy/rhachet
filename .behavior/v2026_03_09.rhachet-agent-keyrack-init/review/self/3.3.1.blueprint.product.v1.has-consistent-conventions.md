## output emoji conventions

### investigation

two commands output "keyrack init" text:

| command | file | emoji | line |
|---------|------|-------|------|
| `keyrack init` | invokeKeyrack.ts | 🔐 | 133 |
| `init --keys` | invokeInit.ts | 🔑 | 127 |

### is this inconsistent?

**no — intentional differentiation.**

| command | scope | emoji meaning |
|---------|-------|---------------|
| `keyrack init` | full keyrack (host manifest + encryption) | 🔐 locked = security |
| `init --keys` | repo manifest only (no encryption) | 🔑 key = config |

the emojis signal different operations:
- 🔐 = security-sensitive (encryption, private keys)
- 🔑 = configuration (manifest setup, no secrets)

### extant patterns

other init commands in invokeInit.ts:
- `initRolesFromPackages` uses `🔧`
- `execRoleInits` uses `💪`

each init type has its own emoji. `--keys` using 🔑 fits this pattern.

**verdict:** consistent — emojis differentiate scope, not inconsistency
