## what was questioned and kept

1. **auto-detect vs explicit org** — kept auto-detect. the wish says "based on github repo owner" which implies auto-detection. `--org` override addresses edge cases.

2. **extends discovery convention** — kept `.agent/repo=*/role=*/keyrack.yml` pattern. this matches the established .agent directory structure.

3. **upsert vs findsert** — the wish says "upsert" but the behavior described is findsert-like (add absent, preserve extant). clarified in vision as "upsert: add absent, preserve extant entries".

## what should be validated with wisher

2. **env.prep between prod and test** — the wish shows `env.prod`, `env.prep`, `env.test`. is there a semantic order? (prod = production, prep = preview/preprod, test = local/ci?)
