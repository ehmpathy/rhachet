### Q1: should we support non-github remotes (gitlab, bitbucket)?

| source | answer method |
|--------|---------------|
| vision open questions | self-answerable via logic |

**research:**
git remote URLs have standard formats across providers:
- GitHub SSH: `git@github.com:owner/repo.git`
- GitHub HTTPS: `https://github.com/owner/repo.git`
- GitLab: `git@gitlab.com:owner/repo.git`
- Bitbucket: `git@bitbucket.org:owner/repo.git`

the org extraction pattern is provider-agnostic: extract the owner/org segment from the path after the domain.

**self-answer:** yes, support non-github remotes via generic pattern:
- SSH: extract segment after `:` and before `/`
- HTTPS: extract path segment after domain

**action:** no wisher input needed. vision can note this as implementation detail.
