## architecture clarity

### the two init paths

```
keyrack init (invokeKeyrack.ts)
├─ initKeyrack
│  ├─ creates host manifest (encrypted)
│  ├─ requires private key
│  └─ optionally creates repo manifest
└─ output: host + repo status

init --keys (invokeInit.ts)
├─ initKeyrackRepoManifest
│  ├─ creates repo manifest only
│  ├─ no encryption, no key required
│  └─ auto-detects org, discovers extends
└─ output: repo status
```

these are complementary, not duplicated:
- `init --keys` is simpler — just the manifest, composable with `--roles`, `--hooks`
- `keyrack init` is fuller — sets up encryption infrastructure
