## reproduction feasibility

### test utilities

- `genTestTempRepo({ fixture })` — creates isolated temp repo with fixture files
- `invokeRhachetCliBinary({ args, cwd })` — invokes CLI as subprocess
- extant fixtures can be extended with `with-role-keyrack` pattern

### setup required

for each test:
1. create temp repo via `genTestTempRepo`
2. fixture includes:
   - package.json with scoped name or repository field
   - .agent/ directory with role keyrack symlink
3. invoke CLI via `invokeRhachetCliBinary`
4. assert on output and file contents

### test sketch

```ts
given('[case1] repo with scoped package name and role keyrack', () => {
  const repo = useBeforeAll(async () =>
    genTestTempRepo({ fixture: 'with-role-keyrack' }),
  );

  when('[t0] init --keys is run', () => {
    const result = useBeforeAll(async () =>
      invokeRhachetCliBinary({
        args: ['init', '--keys'],
        cwd: repo.path,
      }),
    );

    then('exits with status 0', () => {
      expect(result.status).toEqual(0);
    });

    then('creates .agent/keyrack.yml', async () => {
      const content = await fs.readFile(
        join(repo.path, '.agent/keyrack.yml'),
        'utf8',
      );
      expect(content).toContain('org: test-org');
    });

    then('includes role keyrack in extends', async () => {
      const content = await fs.readFile(
        join(repo.path, '.agent/keyrack.yml'),
        'utf8',
      );
      expect(content).toContain('.agent/repo=test-repo/role=tester/keyrack.yml');
    });
  });
});
```
