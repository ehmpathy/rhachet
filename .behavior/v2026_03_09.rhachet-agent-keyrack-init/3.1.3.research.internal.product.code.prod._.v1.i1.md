# research: production codepath patterns for init --keys

## summary

this research identifies extant production patterns relevant to `init --keys` feature implementation.

---

## pattern.1: package.json parse

### [REUSE] discoverRolePackages

**source:** `src/domain.operations/init/discoverRolePackages.ts` [^1]

scans `package.json` dependencies for packages matched by `rhachet-roles-*`:

```ts
const allDeps = { ...pkg.dependencies, ...pkg.devDependencies };
return Object.keys(allDeps).filter((name) => name.startsWith('rhachet-roles-'));
```

**applicability:** same pattern can extract org from scoped names (e.g., `@ehmpathy/my-project` → `ehmpathy`).

### [REUSE] discoverBrainPackages

**source:** `src/domain.operations/brains/discoverBrainPackages.ts` [^2]

same dependency scan pattern for `rhachet-brains-*` packages.

### [EXTEND] invokeRepoIntrospect

**source:** `src/contract/cli/invokeRepoIntrospect.ts` [^3]

reads `package.json` for package name:

```ts
const packageJsonPath = join(gitRoot, 'package.json');
const packageJson = JSON.parse(readFileSync(packageJsonPath, 'utf8'));
const packageName: string = packageJson.name;
```

**gap:** no extant code reads `organization` field or parses `repository` field for org extraction. the fallback chain (organization → scoped name → repository field) requires new logic.

## pattern.2: keyrack manifest init

### [EXTEND] initKeyrackRepoManifest

**source:** `src/domain.operations/keyrack/initKeyrackRepoManifest.ts` [^4]

creates `keyrack.yml` with org + env sections:

```ts
const content = `org: ${input.org}

env.prod:
  # - AWS_PROFILE

env.test:
  # - AWS_PROFILE
`;
await fs.writeFile(manifestPath, content, 'utf-8');
```

**gap:**
- requires `org` as explicit input (no auto-detection)
- does not populate `extends` array
- template omits `env.prep` section per vision

### [REUSE] daoKeyrackRepoManifest.init

**source:** `src/access/daos/daoKeyrackRepoManifest/index.ts` [^5]

findsert semantics for keyrack.yml:

```ts
if (existsSync(path)) {
  return { effect: 'found' };
}
// ... create minimal manifest
return { effect: 'created' };
```

**applicability:** same findsert pattern needed for `init --keys` idempotency.

## pattern.3: extends hydration

### [REUSE] hydrateKeyrackRepoManifest

**source:** `src/access/daos/daoKeyrackRepoManifest/hydrate/hydrateKeyrackRepoManifest.ts` [^6]

recursive extends traversal with circular detection:

```ts
if (visited.has(manifestPath)) {
  throw new BadRequestError('circular extends detected');
}
visited.add(manifestPath);

for (const extendPath of explicit.extends) {
  const absolutePath = join(context.gitroot, extendPath);
  const extendedManifest = hydrateKeyrackRepoManifest({ ... });
  mergedKeys = { ...mergedKeys, ...extendedManifest.keys };
}
```

**applicability:** extends array is already hydrated on read. `init --keys` only needs to populate the extends array on write.

## pattern.4: role keyrack link

### [REUSE] execRoleLink

**source:** `src/domain.operations/invoke/link/execRoleLink.ts` [^7]

symlinks role keyrack.yml into `.agent/repo=$repo/role=$role/keyrack.yml`:

```ts
if (input.role.keyrack?.uri) {
  const targetPath = join(repoRoleDir, 'keyrack.yml');
  symlinkFile({ sourcePath: input.role.keyrack.uri, targetPath });
}
```

**applicability:** after `roles link`, role keyracks exist at `.agent/repo=$repo/role=$role/keyrack.yml`. glob pattern `.agent/repo=*/role=*/keyrack.yml` discovers them.

## pattern.5: glob-based discovery

### [EXTEND] discoverInitExecutables / discoverSkillExecutables

**source:** `src/domain.operations/invoke/discoverSkillExecutables.ts` (inferred from filename)

discovers executables via filesystem glob in `.agent/` directory.

**gap:** no extant `discoverRoleKeyracks` function. new glob needed:
```ts
const pattern = '.agent/repo=*/role=*/keyrack.yml';
```

## pattern.6: init flag composition

### [EXTEND] invokeInit

**source:** `src/contract/cli/invokeInit.ts` [^8]

handles `--roles`, `--hooks`, `--config`, `--prep` flags:

```ts
if (options.roles && options.roles.length > 0) {
  await initRolesFromPackages({ specifiers: options.roles }, context);
  if (options.hooks !== undefined) {
    await syncHooksForLinkedRoles({ brains }, context);
  }
}
```

**gap:** no `--keys` flag. needs to be added and composed with other flags.
