# blackbox criteria: rhachet init --keys

## usecase.1 = fresh repo keyrack setup

```
given('a repo with package.json and no .agent/keyrack.yml')
  given('roles: mechanic, dispatcher have keyracks in .agent/')
  when('user runs `npx rhachet init --keys --roles mechanic dispatcher`')
    then('creates .agent/keyrack.yml')
      sothat('keyrack manifest exists for the repo')
    then('org field is set from package.json')
      sothat('user does not have to specify org manually')
    then('extends array contains only mechanic and dispatcher keyracks')
      sothat('only specified role keyracks are extended')
    then('extends entries are sorted alphabetically')
      sothat('output is deterministic for clean diffs')
    then('env.prod, env.prep, env.test sections are present')
      sothat('user has env sections ready for key declarations')
    then('outputs tree with org source and discovered extends')
      sothat('user sees what was discovered and created')
```

## usecase.2 = org detection fallback chain

```
given('a repo with package.json#organization field')
  given('roles: mechanic has keyrack in .agent/')
  when('user runs `npx rhachet init --keys --roles mechanic`')
    then('org is set from package.json#organization')
      sothat('explicit org declaration takes precedence')

given('a repo with scoped package name @org/name but no organization field')
  given('roles: mechanic has keyrack in .agent/')
  when('user runs `npx rhachet init --keys --roles mechanic`')
    then('org is extracted from scoped name')
      sothat('scoped packages work without extra config')

given('a repo with repository field but no organization or scoped name')
  given('roles: mechanic has keyrack in .agent/')
  when('user runs `npx rhachet init --keys --roles mechanic`')
    then('org is extracted from repository field owner')
      sothat('standard npm repos work without extra config')

given('a repo with no org-detectable fields in package.json')
  given('roles: mechanic has keyrack in .agent/')
  when('user runs `npx rhachet init --keys --roles mechanic`')
    then('error message directs user to `keyrack init` for custom org')
      sothat('user knows how to proceed')
```

## usecase.3 = extends append semantics

```
given('a repo with .agent/keyrack.yml that has extends entries for mechanic')
  given('roles: mechanic, dispatcher both have keyracks in .agent/')
  when('user runs `npx rhachet init --keys --roles mechanic dispatcher`')
    then('dispatcher keyrack is appended to extends')
      sothat('new specified roles are added')
    then('prior mechanic extends entry is preserved')
      sothat('extant entries are not lost')
    then('extends array is sorted alphabetically')
      sothat('output remains deterministic')

given('a repo with .agent/keyrack.yml that has extends for mechanic and dispatcher')
  given('roles: only mechanic has keyrack in .agent/ (dispatcher was removed)')
  when('user runs `npx rhachet init --keys --roles mechanic`')
    then('dispatcher extends entry is preserved')
      sothat('append-only semantics are safe by default')
    then('only mechanic keyrack is validated')
      sothat('unspecified roles are not checked')
```

## usecase.4 = role without keyrack (skip, create manifest anyway)

```
given('a repo with linked roles but none have keyracks')
  given('roles: tester is linked but has no keyrack.yml')
  when('user runs `npx rhachet init --keys --roles tester`')
    then('keyrack.yml is created')
      sothat('manifest still initializes even without role keyracks')
    then('extends array is empty or absent')
      sothat('no invalid keyracks are referenced')
    then('org, env.prod, env.prep, env.test are still present')
      sothat('manifest is valid and ready for future use')
```

## usecase.5 = repo keyrack only

```
given('a repo with linked roles')
  given('roles: mechanic has keyrack in .agent/')
  when('user runs `npx rhachet init --keys --roles mechanic`')
    then('creates .agent/keyrack.yml')
      sothat('init --keys always operates on repo keyrack')
    then('host keyrack is NOT touched')
      sothat('host and repo keyracks are separate concerns')
```

note: for host keyrack, use `rhachet keyrack init --for host`. the `init` command does not have a `--for` flag.

## usecase.6 = flag composition

```
given('a repo that needs keys, hooks, and roles init')
  given('roles: mechanic has keyrack in .agent/')
  when('user runs `npx rhachet init --keys --hooks --roles mechanic`')
    then('keyrack.yml is created with mechanic in extends')
    then('hooks are initialized for mechanic')
    then('mechanic role is initialized')
      sothat('all init tasks complete in one command')
```

## usecase.7 = idempotent re-run

```
given('a repo with up-to-date .agent/keyrack.yml')
  given('roles: mechanic has keyrack and is already in extends')
  when('user runs `npx rhachet init --keys --roles mechanic`')
    then('keyrack.yml is unchanged')
      sothat('re-runs are safe and idempotent')
    then('output indicates no changes were needed')
      sothat('user knows state is current')
```

## usecase.8 = scoped extends via --roles (explicit only)

```
given('a repo with multiple linked roles that have keyracks')
  given('roles: mechanic, dispatcher, designer all have keyracks in .agent/')
  when('user runs `npx rhachet init --keys --roles mechanic dispatcher`')
    then('extends array contains only mechanic and dispatcher keyracks')
      sothat('user controls which role keyracks are extended')
    then('extends array does NOT contain designer keyrack')
      sothat('unspecified roles are excluded')
    then('extends entries are sorted alphabetically')
      sothat('output remains deterministic')
```

## usecase.9 = --keys requires --roles (fail-fast)

```
given('a repo with linked roles')
  when('user runs `npx rhachet init --keys` without --roles')
    then('error is thrown')
      sothat('user must explicitly specify which roles to extend')
    then('error message suggests using --roles flag')
      sothat('user knows how to proceed')
```

## usecase.10 = mixed roles (some with keyrack, some without)

```
given('a repo with roles where some have keyracks and some do not')
  given('roles: mechanic has keyrack, tester does not')
  when('user runs `npx rhachet init --keys --roles mechanic tester`')
    then('keyrack.yml is created')
      sothat('manifest initializes successfully')
    then('extends array contains only mechanic keyrack')
      sothat('roles with keyracks are included')
    then('tester is silently skipped')
      sothat('roles without keyracks do not block initialization')
```
