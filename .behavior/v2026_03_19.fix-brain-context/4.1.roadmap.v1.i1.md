  - [ ] add generic to class: `class BrainAtom<TContext = Empty>`

- [ ] **2.3** update `src/domain.objects/BrainRepl.ts`
  - [ ] add generic to interface: `interface BrainRepl<TContext = Empty>`
  - [ ] change `context?: Empty` to `context?: TContext` on `ask`
  - [ ] change `context?: Empty` to `context?: TContext` on `act`
  - [ ] add generic to class: `class BrainRepl<TContext = Empty>`
  - [ ] update extends: `DomainEntity<BrainRepl<TContext>>`
  - [ ] update implements: `implements BrainRepl<TContext>`

- [ ] **2.4** update `src/domain.objects/BrainRepl.types.test.ts`
  - [ ] test: `BrainRepl<ContextBrainSupplierXai>` accepts typed context on ask
  - [ ] test: `BrainRepl<ContextBrainSupplierXai>` accepts typed context on act
  - [ ] test: `BrainRepl` (no generic) infers `BrainRepl<Empty>`
  - [ ] test: backwards compat — extant callers without context still work

### acceptance criteria

```
given BrainAtom with TContext generic
  when supplier factory returns BrainAtom<ContextBrainSupplierXai>
    then typescript accepts the return type
    then ask method accepts context of that type
    then call without context is allowed (optional param)

given BrainRepl with TContext generic
  when supplier factory returns BrainRepl<ContextBrainSupplierXai>
    then typescript accepts the return type
    then ask method accepts context of that type
    then act method accepts context of that type
    then call without context is allowed (optional param)

given backwards compatibility
  when extant code uses BrainAtom without generic
    then typescript infers BrainAtom<Empty>
    then no type errors occur
```

### verification

npm run test:types -- BrainAtom.types.test.ts BrainRepl.types.test.ts

## phase 3: actor context flow

### prereqs

- phase 0-2 complete
- read: `.behavior/v2026_03_19.fix-brain-context/3.1.3.research.internal.product.code.prod._.v1.i1.md` (patterns 5-7)
- read: `.behavior/v2026_03_19.fix-brain-context/2.1.criteria.blackbox.md` (usecase 6)

### steps

- [ ] **3.1** update `src/domain.operations/actor/actorAsk.ts`
  - [ ] add generic: `actorAsk<TContext = Empty>`
  - [ ] add input param: `context?: TContext`
  - [ ] pass to brain: `input.brain.ask(..., context)`

- [ ] **3.2** update `src/domain.operations/actor/actorAct.ts`
  - [ ] add generic: `actorAct<TContext = Empty>`
  - [ ] add input param: `context?: TContext`
  - [ ] pass to brain: `input.brain.act(..., context)`

- [ ] **3.3** update `src/domain.operations/actor/genActor.ts`
  - [ ] update `ask` method to accept context, pass to actorAsk
  - [ ] update `act` method to accept context, pass to actorAct
  - [ ] preserve all other logic

- [ ] **3.4** add/update integration tests
  - [ ] `actorAsk.integration.test.ts`: context flows to brain
  - [ ] `actorAct.integration.test.ts`: context flows to brain
  - [ ] `genActor.integration.test.ts`: context thread through ask and act

### acceptance criteria

```
given actor with enrolled brain
  when consumer calls actor.ask with context
    then context is passed to the brain
    then brain receives context at context param

  when consumer calls actor.act with context
    then context is passed to the brain
    then brain receives context at context param

  when consumer calls without context
    then brain receives undefined context
    then no errors occur (optional param)
```

### verification

```sh
npm run test:integration -- actorAsk.integration.test.ts actorAct.integration.test.ts genActor.integration.test.ts
```
