## what we need

### 1. generic context on BrainAtom and BrainRepl interfaces

current signature in rhachet:

```ts
interface BrainAtom {
  ask: <TOutput, TPlugs>(input, context?: Empty) => Promise<BrainOutput<...>>
}

interface BrainRepl {
  ask: <TOutput, TPlugs>(input, context?: Empty) => Promise<BrainOutput<...>>
  act: <TOutput, TPlugs>(input, context?: Empty) => Promise<BrainOutput<...>>
}
```

needed signature:

```ts
interface BrainAtom<TContext = Empty> {
  ask: <TOutput, TPlugs>(input, context?: TContext) => Promise<BrainOutput<...>>
}

interface BrainRepl<TContext = Empty> {
  ask: <TOutput, TPlugs>(input, context?: TContext) => Promise<BrainOutput<...>>
  act: <TOutput, TPlugs>(input, context?: TContext) => Promise<BrainOutput<...>>
}
```

**why**: brain supplier factories can bind their required context type at the return type:
```ts
export const genBrainAtom = (input: { slug: XaiBrainAtomSlug }): BrainAtom<ContextBrainSupplierXai> => { ... }
export const genBrainRepl = (input: { slug: XaiBrainReplSlug }): BrainRepl<ContextBrainSupplierXai> => { ... }
```

### 2. publish ContextBrainSupplier type

```ts
/**
 * .what = generic context type for brain suppliers
 * .why = enables typed context injection for any brain supplier
 *
 * .example
 *   ContextBrainSupplier<'xai', BrainSuppliesXai>
 *   // expands to: { 'brain.supplier.xai'?: BrainSuppliesXai }
 */
export type ContextBrainSupplier<TSlug extends string, TSupplies> = {
  /**
   * optional by mandate
   * - forces consideration of context without supplier's supplies
   * - no way to forget to handle the absent case
   */
  [K in `brain.supplier.${TSlug}`]?: TSupplies;
};
```

export from `rhachet` or `rhachet/brains`.

### 4. composition with genContextBrainChoice

`genContextBrainSupplier` composes with `genContextBrainChoice` (from `.behavior/v2026_01_23.context-brain-choice`):

```ts
// they live at different keys - zero collision:
// - ContextBrain uses:           context.brain.*
// - ContextBrainSupplier uses:   context['brain.supplier.*']

type ContextSkillXai =
  & ContextBrain<BrainAtom>                           // brain.choice, brain.atom.ask, etc
  & ContextBrainSupplier<'xai', BrainSuppliesXai>;    // brain.supplier.xai

// construction via spread
const context: ContextSkillXai = {
  ...await genContextBrainChoice({ brain: 'xai/grok/code-fast-1' }),
  ...genContextBrainSupplier('xai', {
    creds: async () => ({ XAI_API_KEY: await vault.get('XAI_API_KEY') }),
  }),
};

// usage in skill
const brain = context.brain.choice;                           // from ContextBrain
const creds = await context['brain.supplier.xai'].creds();    // from ContextBrainSupplier
```

**type safety tradeoff**:

| path | safety | when |
|------|--------|------|
| direct brain ref | compile-time | `xaiAtom.ask(input, context)` — TS enforces context type |
| dynamic choice | runtime | `brain.choice.ask(...)` — brain validates context at runtime |

via `brain.choice`, the brain implementation must validate its required context is present and throw `BadRequestError` if absent. this matches how DI containers work: dynamic resolution trades compile-time safety for flexibility.

**pit-of-success: optional by mandate**

the type enforces optionality via `[K in ...]?: TSupplies` — forces consideration of context without supplier's supplies:
- `context['brain.supplier.xai']` is always `TSupplies | undefined`
- implementers must handle the absent case — no way to forget

TypeScript forces explicit handle:

```ts
export const genBrainAtom = (input: { slug: XaiBrainAtomSlug }): BrainAtom<ContextBrainSupplierXai> => {
  return new BrainAtom({
    slug: input.slug,
    // ...
    ask: async (askInput, context?: ContextBrainSupplierXai) => {
      // option A: fail fast if supplies absent
      const creds = await (context?.['brain.supplier.xai']?.creds() ?? BadRequestError.throw('brain.supplier.xai required'));

      // option B: fallback to env var if supplies absent
      const creds = await (context?.['brain.supplier.xai']?.creds() ?? { XAI_API_KEY: process.env.XAI_API_KEY });
    },
  });
};
```

no silent failures — implementers must handle the absent-slug case explicitly.
