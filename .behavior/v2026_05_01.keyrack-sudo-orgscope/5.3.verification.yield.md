## summary

the implementation:
1. when no repo context exists and no full slug provided → only `@all` keys are accessible
2. when repo context exists → keys matching repo org OR `@all` are accessible
3. full slug syntax bypasses org filter completely

this enforces org scope for sudo keys while preserving cross-org access via `@all` wildcard and explicit full slug escape hatch.
