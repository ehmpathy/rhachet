# blocker.1: Integration tests mock external vault services instead of calling real services

**rule**: require.external-contract-integration-tests

**locations**:
- src/domain.operations/keyrack/session/unlockKeyrackKeys.integration.test.ts

The rule requires that every external contract (such as vault adapters for services like 1password, aws.config, github.secrets, etc.) must have integration tests that call the real service with real credentials and verify responses. However, in the provided integration test file, all vault adapters are mocked using genMockVaultAdapter, which does not interact with real external services. While the daemon is accessed via socket (potentially a real call), the core external contracts (vaults) are not tested against live services, leading to false confidence. No exceptions are noted in the code or vision for mocking these vaults. To fix, update the tests to use real vault adapters with actual credentials and validate real responses.
