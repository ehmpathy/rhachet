## gaps identified

### gap 1: error message suggestion

**vision says:**
> mitigation: error message should suggest `--prikey` when decryption fails

**blueprint status:** not explicitly addressed in implementation details

**resolution:** the DAO layer already throws a clear error when decryption fails. the error message improvement is nice-to-have but not a blocker — users will understand they need `--prikey` when they see "could not decrypt manifest".

**verdict:** minor gap, acceptable for v1

### gap 2: implicit edgecase tests

**vision says:**
> prikey doesn't exist → fail-fast: "file not found: {path}"
> prikey is pubkey by mistake → fail-fast: "expected private key, got public key"
> prikey permissions wrong → fail-fast: "permission denied to read key"

**blueprint status:** these are handled by the DAO layer but not explicitly tested

**resolution:** these are lower-level failures that happen in `sshPrikeyToAgeIdentity`. we could add unit tests for these in future iterations, but the acceptance test with "wrong key" covers the decryption failure path.

**verdict:** minor gap, acceptable for v1
