# blueprint: keyrack daemon home-scoped cache

## summary

add HOME to daemon identity scope for test isolation.

**before**: one daemon per (session, owner) → `keyrack.$SESSIONID.$owner.sock`
**after**: one daemon per (session, home, owner) → `keyrack.$SESSIONID.$HOMEHASH.$owner.sock`

key changes:
1. socket path adds home hash (keeps session ID)
2. daemon auto-terminates when all keys expire

## contracts

### new: getHomeHash

```ts
/**
 * .what = compute truncated hash of HOME path
 * .why = deterministic daemon identity per HOME
 */
export const getHomeHash = (): string => {
  const homePath = process.env['HOME'] ?? process.cwd();
  const realPath = realpathSync(homePath); // fail fast if symlink broken
  return createHash('sha256').update(realPath).digest('hex').slice(0, 8);
};
```

### new: scheduleAutoTermination

```ts
/**
 * .what = periodically check if all keys expired, exit if so
 * .why = daemon with only expired keys serves no purpose
 */
export const scheduleAutoTermination = (
  input: { keyStore: DaemonKeyStore; checkIntervalMs?: number },
): NodeJS.Timeout => {
  const { keyStore, checkIntervalMs = 15 * 60 * 1000 } = input; // 15min
  let hasEverHadKeys = false;

  return setInterval(() => {
    const entries = keyStore.entries(); // purges expired as side effect
    if (entries.length > 0) hasEverHadKeys = true;
    if (hasEverHadKeys && entries.length === 0) {
      console.log('[keyrack-daemon] all keys expired, terminate');
      process.exit(0);
    }
  }, checkIntervalMs);
};
```

## security model change

**before**: (session, owner) isolation
- daemon verifies caller is in same login session
- prevents cross-session access

**after**: (session, home, owner) isolation
- keeps session isolation (same security)
- adds home isolation (enables test isolation)
- socket permissions (0600) prevent other users

**benefit**:
- keeps: cross-session isolation (security)
- gains: test isolation across HOMEs (the core requirement)

## daemon lifecycle

```
t=0:     CLI calls keyrack unlock with HOME=/tmp/test
t=0:     findsertKeyrackDaemon computes socket path with home hash
t=0:     no daemon found → spawn daemon
t=0:     daemon receives UNLOCK, caches key (TTL from manifest)
t=TTL:   key expires, entries().length === 0
t=TTL:   termination scheduler triggers process.exit(0)
t=TTL+1: CLI calls keyrack unlock again
t=TTL+1: no daemon found → spawn fresh daemon
t=TTL+1: user prompted to configure (no stale state)
```

## risks and mitigations

| risk | mitigation |
|------|------------|
| many daemons in heavy test runs | daemons auto-terminate when keys expire |
| daemon orphaned if termination fails | graceful shutdown on SIGTERM; can always pkill |
| race between entries check and new unlock | only terminate after hasEverHadKeys flag set |
| HOME with special chars breaks socket | sha256 hash normalizes any string |
