## mental model

### how users describe this

> "it shows accounts alphabetically now, so prod is always in the same spot"

> "it echoes back what i picked so i can verify without re-read the options"

### analogies

like a restaurant menu that lists dishes alphabetically instead of in random order вЂ” you know where to look.

like a confirmation dialog that says "delete 'important-file.txt'?" instead of just "delete?"

### terms

| user says | we say |
|-----------|--------|
| "account name" | `accountName` |
| "account id" | `accountId` |
| "role" | `roleName` |
| "option" | numbered choice in list |
| "choice echo" | confirmation of selected value |

## evaluation

### how well does it solve the goals?

| goal | solved? |
|------|---------|
| stable sort for accounts | вњ“ sort by accountName |
| stable sort for roles | вњ“ sort by roleName |
| name before id | вњ“ `{name} В· {id}` format |
| echo choice | вњ“ add `в””в”Ђ {value}` under `в””в”Ђ N вњ“` |

### pros

- deterministic order across runs
- human-readable first (name), machine-id second
- choice confirmation reduces doubt
- no behavior changes, purely cosmetic

### cons

- slightly more vertical space used (choice echo line)
- alphabetical may not match "importance" (e.g., prod vs prep)

### edge cases

| edge case | how we handle |
|-----------|---------------|
| accounts with same name prefix | alphabetical still deterministic |
| single account | still sorted (trivially) |
| single role | still sorted (trivially) |
| account name with special chars | unchanged вЂ” just display order changes |

## what is awkward?

### alphabetical vs importance

"ahbode-prep" comes before "ahbode-prod" alphabetically. some users might expect prod first as the "primary" environment. but importance is subjective вЂ” alphabetical is predictable.

### choice echo adds a line
