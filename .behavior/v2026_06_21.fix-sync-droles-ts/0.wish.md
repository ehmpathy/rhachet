wish =

when rhachet compiles a repo for publish

it should include in the rsync any .ts files as well as the .sh files

why?

because that enable sskills to be .sh entrypoints which then run `npx tsx ./$slug.ts`

which is how oftentimes skills are written

especially in .agent/repo=.this/role=any/skills

---

that way, repos can easily create their own .ts backed skills

even if .sh is still the entrypoint

without needing compilation or buildsteps for simple skills that dispatch to ts

---

only bundle those files if they're in @gitroot/src/domain.roles/**/* though

the other dirs are expected to require comilation and be imported via `package/contract/cli`


---
