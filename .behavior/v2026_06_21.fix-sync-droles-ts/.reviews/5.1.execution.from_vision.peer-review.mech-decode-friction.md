# blocker.1: Inline decode-friction in complex sort comparator

**rule**: rule.forbid.inline-decode-friction.md

**locations**:
- src/domain.operations/compile/getAllArtifactsForRole.ts

The getAllArtifactsForRole function (an orchestrator) contains inline decode-friction in its final sorting logic. The comparator splits paths by '/', uses positional length checks to determine directory vs file status per segment (aIsDir = i < length-1), performs segment-by-segment directory-first ordering, then falls back to length. This requires mental simulation to understand the produced ordering.

Per the rule, decode-friction (array splits with positional semantics, complex boolean expressions, sort with custom comparator) must be extracted to a named transformer. Orchestrators must read as narrative using only named calls.

Extract to a transformer such as `sortArtifactsByDirectoryFirst` and call it instead.

**snippet**:
```ts
  // sort for deterministic output: directories before files at each level
  return artifacts.sort((a, b) => {
    const aParts = a.split('/');
    const bParts = b.split('/');

    // compare segment by segment
    const minLen = Math.min(aParts.length, bParts.length);
    for (let i = 0; i < minLen; i++) {
      const aIsDir = i < aParts.length - 1;
      const bIsDir = i < bParts.length - 1;

      // at this level, directories come before files
      if (aIsDir && !bIsDir) return -1;
      if (!aIsDir && bIsDir) return 1;

      // both are dirs or both are files, compare alphabetically
      const cmp = aParts[i]!.localeCompare(bParts[i]!);
      if (cmp !== 0) return cmp;
    }

    // shorter path first (shouldn't happen given the above logic)
    return aParts.length - bParts.length;
  });
```
