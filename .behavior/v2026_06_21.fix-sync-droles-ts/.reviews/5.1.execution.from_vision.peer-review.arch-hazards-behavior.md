# blocker.4: Non-determinism from hardcoded path separator in sort

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.behavior-hazards.md

**locations**:
- src/domain.operations/compile/getAllArtifactsForRole.ts:133

The final sort logic does a.split('/') and b.split('/') to determine directory vs file ordering. On Windows, path.relative() produces backslashes, so splitting on '/' fails and the entire sort falls back to inconsistent alphabetical ordering of full strings. Given identical inputs, output order (and thus behavior) differs by OS/platform. This is exactly the non-determinism behavior hazard (different outputs for same inputs, depends on external state like OS) that the rule forbids. Also risks incorrect dir-before-file semantics.

**snippet**:
```ts
    const aParts = a.split('/');
    const bParts = b.split('/');
```
