# blueprint: rhx help & list

## summary

implement two new rhx capabilities:

> **note:** per [flagged research](3.1.1.research.external.product.flagged._.yield.md), no external research needed — this feature is purely internal CLI ergonomics.

1. **help command**: `rhx --help`, `rhx -h`, `rhx help` → display commands treestruct with readme reference
2. **list command**: `rhx list [pattern] [--repo X] [--role X] [--all]` → display discovered skills in treestruct format

both commands route through `bin/rhx` short-circuits to `rhachet run --help` and `rhachet run list` respectively.

## implementation notes

### pattern match logic

```ts
/**
 * .what = checks if pattern contains glob metacharacters
 * .why = determines whether to use glob mode vs contains mode
 */
const getHasGlobChars = (input: { pattern: string }): boolean =>
  /[*?[\]]/.test(input.pattern);

/**
 * .what = checks if a skill slug matches a search pattern
 * .why = enables both glob patterns (*.sh) and contains search (radio)
 */
const getIsSlugMatch = (input: { slug: string; pattern: string }): boolean => {
  if (!input.pattern) return true;
  if (getHasGlobChars({ pattern: input.pattern })) {
    return picomatch.isMatch(input.slug, input.pattern);
  }
  return input.slug.includes(input.pattern);
};
```

### self-descriptive header

when user runs `rhx list radio` (no glob chars), the header renders as:
```
🪨 rhx list '*radio*'
```

this makes the "contains" behavior explicit and self-descriptive.

### truncation threshold

default: 10 skills per role before truncation
configurable via `--all` flag to show all

### reserved command names

`list`, `help`, `keyrack`, `upgrade`, `enroll` are reserved. if a skill has these names, it's shadowed by the command.
