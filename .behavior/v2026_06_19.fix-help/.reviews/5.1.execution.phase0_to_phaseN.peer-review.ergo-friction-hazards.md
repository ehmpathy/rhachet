**locations**:
- blackbox/cli/rhx.list.acceptance.test.ts (t2, t3)

the acceptance test for `rhx list` covers `--help` (snapped) but not the `help` subcommand variant nor the `-h` flag variant. the rule requires exhaustive snapshot coverage for every output variant. the `help` subcommand (t2) and `-h` flag (t3) both emit help output but are not snapped. without a snapshot, drift in those outputs goes undetected.

affected variants:
- rhx list help (t2) – only substring checks, no snapshot
- rhx list -h (t3) – only substring checks, no snapshot

**snippet**:
```ts
when('[t2] rhx list help is invoked', () => {

});
```

- src/contract/cli/invokeRun.ts (emitHelpOutput function)

**snippet**:
```ts
const emitHelpOutput = (): void => {
  console.log(`🪨 rhx [command] [...args]`);
  console.log(`   └─ commands`);
  console.log(`      ├─ [skill]     run a skill (default)`);
  console.log(`      ├─ list        show available skills`);
  console.log(`      ├─ keyrack     manage credentials`);
  console.log(`      ├─ enroll      enroll brain in role`);
  console.log(`      └─ upgrade     upgrade rhachet`);
  console.log(``);
  console.log(`ref: https://github.com/ehmpathy/rhachet#readme`);
};
// missing: --skill, --init, --repo, --role, --attempts options
```
