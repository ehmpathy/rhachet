## notes

### Commander.js option shadowing fix
The `--repo` and `--role` options weren't working via CLI because Commander.js
shadows parent command options on subcommands. Solution:
1. Renamed internal options to `--filter-repo` and `--filter-role` in invokeRunList.ts
2. Added shell translation in bin/rhx to translate user-facing `--repo`/`--role` to internal names
3. Restored `allowExcessArguments(true)` in invokeRun.ts for skill positional args
