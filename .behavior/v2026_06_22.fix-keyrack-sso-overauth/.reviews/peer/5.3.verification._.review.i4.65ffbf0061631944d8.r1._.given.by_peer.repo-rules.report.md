# blocker.1: vaultAdapterAwsConfig.set ignores context.identity for roundtrip verification

**rule**: .agent/repo=.this/role=keyrack/briefs/rule.require.context-for-identity.md

**locations**:
- src/domain.operations/keyrack/adapters/vaults/aws.config/vaultAdapterAwsConfig.ts:449-453
- src/domain.operations/keyrack/adapters/vaults/aws.config/vaultAdapterAwsConfig.ts:492-496

The set method of vaultAdapterAwsConfig receives `context?: ContextKeyrack` but does not use `context.identity` to discover identities for roundtrip verification. Instead, it passes `identity: null` to `vaultAdapterAwsConfig.unlock`. This bypasses the context's identity pool (prescribed + discovered), violating the rule that operations needing identity access must receive ContextKeyrack and use `context.identity.getOne`.

Specifically in the roundtrip verification path for --exid case, it calls:
```
await vaultAdapterAwsConfig.unlock({ identity: null, exid: profileName, silent: true, skipMetaCheck: true })
```
This should be:
```
const identity = await context?.identity?.getOne({ for: 'manifest' });
await vaultAdapterAwsConfig.unlock({ identity, exid: profileName, silent: true, skipMetaCheck: true })
```
The same pattern occurs in the guided setup path as well.

**snippet**:
```ts
// roundtrip verification for --exid case (silent — no console output)
// .note = proves unlock + get + relock work without TTY interaction
// 1. unlock — prove sso session is valid
await vaultAdapterAwsConfig.unlock({
  identity: null,
  exid: profileName,
  silent: true,
  skipMetaCheck: true,
});
```

# nitpick.1: Error messages suggest using `npx` instead of `pnpm exec` or `rhx`

**rule**: .agent/repo=.this/role=any/briefs/rule.require.pnpm-over-npm.md

**locations**:
- src/contract/cli/invokeKeyrack.ts:296
- src/contract/cli/invokeKeyrack.ts:367
- src/contract/cli/invokeKeyrack.ts:416
- src/contract/cli/invokeKeyrack.ts:584

Several error messages in invokeKeyrack.ts suggest running `npx rhachet keyrack init` or similar. The rule requires always using pnpm and avoiding npm. While `npx` is part of npm, the preferred approach would be to recommend `pnpm exec` or directly `rhx` (which is the installed binary). This is a user-facing hint that could promote npm usage.

Example hint in keyrack get command:
```
"run 'npx rhachet keyrack init --org <your-org>' to create one"
```
Should be:
```
"run 'pnpm exec rhachet keyrack init --org <your-org>'"
or
"run 'rhx keyrack init --org <your-org>'"
```

**snippet**:
```ts
// from keyrack set command (line 296)
console.log(
  "   └─ tip: run 'npx rhachet keyrack init --org <your-org>' to create one",
);
```
