# blocker.1: Inline IIFE for org expansion introduces decode-friction in orchestrator

**rule**: rule.forbid.decode-friction-in-orchestrators.md

**locations**:
- src/domain.operations/keyrack/setKeyrackKeyHost.ts

In `setKeyrackKeyHost.ts`, the logic to expand the org input is implemented as an immediately invoked function expression (IIFE). This forces readers to mentally simulate the IIFE to understand what `orgExpanded` will be. According to the rule, orchestrators must contain no decode-friction — each line should tell *what* happens, not *how*. Extract this logic to a named transformer (e.g., `expandOrg({ orgInput, repoManifest })`).

**snippet**:
```ts
const orgExpanded = (() => {
  if (orgInput === '@this') {
    if (!context.repoManifest?.org) {
      throw new BadRequestError(
        '@this requires repo manifest to expand org',
        { note: 'run from a repo with keyrack.yml or use @all for cross-org credentials' },
      );
    }
    return context.repoManifest.org;
  }
  return orgInput;
})();
```

# nitpick.1: Ternary for default duration should be extracted

**rule**: rule.forbid.decode-friction-in-orchestrators.md

**locations**:
- src/domain.operations/keyrack/session/unlockKeyrackKeys.ts

In `unlockKeyrackKeys.ts`, the default duration is computed inline with a ternary: `const defaultDuration = input.env === 'sudo' ? '30m' : '9h';`. While small, this is decode-friction in an orchestrator. Extract to a named transformer like `getDefaultDuration({ env })` to make the intents explicit and keep the orchestrator narrative.

**snippet**:
```ts
const defaultDuration = input.env === 'sudo' ? '30m' : '9h';
```
