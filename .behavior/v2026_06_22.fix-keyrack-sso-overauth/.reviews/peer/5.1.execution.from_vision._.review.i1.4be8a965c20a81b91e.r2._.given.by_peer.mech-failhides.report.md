# nitpick.1: catch-all try/catch in daoKeyrackHostManifest.get without allowlist

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.errors/rule.forbid.failhide.md.pt1.md

**locations**:
- src/access/daos/daoKeyrackHostManifest/index.ts

The try/catch block in `get` catches all errors from `decryptWithIdentity` and throws a new `BadRequestError`. The rule requires that catch blocks must have an allowlist of expected errors and rethrow others. While the original error is included in metadata, this pattern could mask unexpected errors (e.g., file read failures, identity format issues) that should propagate as-is for faster diagnosis. Prefer a more specific allowlist or use `HelpfulError.wrap` as shown in the rule examples.

**snippet**:
```ts
try {
  plaintext = await decryptWithIdentity({ ciphertext, identity });
} catch (error) {
  throw new BadRequestError('failed to decrypt host manifest', {
    path,
    owner,
    error: error instanceof Error ? error.message : String(error),
  });
}
```
