# blocker.1: domain.objects importing from domain.operations violates upward dependency

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.scope-leaks.md

**locations**:
- src/domain.objects/keyrack/KeyrackHostVaultAdapter.ts:1

The file `KeyrackHostVaultAdapter.ts` is a domain object (in domain.objects) but imports a type from domain.operations (`ContextKeyrack`). This violates the directional dependency rule: lower layers must not import from higher layers. Domain objects should be independent of the operations layer. The import creates a tight coupling that makes the domain object dependent on operations logic. The type `ContextKeyrack` should be defined in domain.objects or in a shared contract layer, or the interface should be restructured to avoid this dependency.

**snippet**:
```ts
import type { ContextKeyrack } from '@src/domain.operations/keyrack/genContextKeyrack';
```
