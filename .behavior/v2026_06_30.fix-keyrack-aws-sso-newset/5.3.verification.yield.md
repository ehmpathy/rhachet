# 5.3 verification — buttonup

> the wish: `keyrack set --key AWS_PROFILE --vault aws.config` completes sso auth, writes the
> profile to `~/.aws/config`, then fails `CredentialsProviderError: Profile X was not found` — yet an
> immediate retry (same name, no logout) works. root cause: the `@smithy/core/config` loader caches
> file contents in a process-lifetime module-level map (`filePromises`), keyed by path; a profile
> written earlier this same run is served stale, so the in-process `fromSSO` cannot see it (the cli, a
> fresh subprocess, can). fix: `fromSSO({ profile, ignoreCache: true })` forces a fresh disk read.
