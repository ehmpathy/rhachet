# research: internal test codepaths

## pattern.1 = unit test with mocked child_process

**[REUSE]**

aws sso requires browser auth, so unit tests mock child_process.

citation [1]: `src/domain.operations/keyrack/adapters/vaults/aws.config/vaultAdapterAwsConfig.test.ts:4-9`
```ts
/**
 * .note = mocks child_process (exec, spawn) and fs to simulate aws cli behavior
 * .why = aws sso requires browser-based oauth flow — cannot be automated in tests
 *        see: aws sso uses interactive browser auth to approve access
 *        mocks allow test of adapter logic without real aws credentials
 * .note = no snapshot coverage because aws.config is internal vault adapter, not user-faced contract
 */
```

citation [2]: `src/domain.operations/keyrack/adapters/vaults/aws.config/vaultAdapterAwsConfig.test.ts:12-29`
```ts
// mock child_process.exec and spawn before import
jest.mock('node:child_process', () => {
  const originalModule = jest.requireActual('node:child_process');
  const { EventEmitter } = require('node:events');

  return {
    ...originalModule,
    exec: jest.fn((cmd: string, callback: Function) => {
      // default: commands succeed
      callback(null, { stdout: '{}', stderr: '' });
    }),
    spawn: jest.fn(() => {
      // default: spawn succeeds
      const emitter = new EventEmitter();
      process.nextTick(() => emitter.emit('close', 0));
      return emitter;
    }),
  };
});
```

## pattern.3 = test for get() returns profile name

**[EXTEND]**

this test verifies get() returns exid — but only when NO mech is supplied. the bug is when mech IS supplied.

citation [5]: `src/domain.operations/keyrack/adapters/vaults/aws.config/vaultAdapterAwsConfig.test.ts:140-149`
```ts
  given('[case2] exid provided', () => {
    when('[t0] get called with exid', () => {
      then('returns the exid as the profile name', async () => {
        const result = await vaultAdapterAwsConfig.get({
          slug: 'acme.prod.AWS_PROFILE',
          exid: 'acme-prod',
        });
        expect(result).toEqual('acme-prod');
      });
    });
```

we need to add a test that verifies get() returns exid EVEN WHEN mech is supplied (i.e., the fix behavior).
