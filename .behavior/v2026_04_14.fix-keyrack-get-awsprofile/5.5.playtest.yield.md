# playtest: keyrack get AWS_PROFILE returns profile name

## prerequisites

- aws cli installed (`aws --version`)
- aws profile configured in `~/.aws/config` (e.g., `ehmpathy.demo`)
- keyrack configured with `AWS_PROFILE` key via `aws.config` vault

**if keyrack is not yet configured with AWS_PROFILE**, set it up first:

```bash
# set the AWS_PROFILE key to use your aws profile name
rhx keyrack set --key AWS_PROFILE --vault aws.config --exid "ehmpathy.demo" --env test
```

(replace `ehmpathy.demo` with your actual profile name from `~/.aws/config`)

## sandbox

- no file operations required for this playtest
- all verification is via CLI commands

## happy paths

### 1. verify keyrack get returns profile name (not credentials json)

**before the fix**, keyrack returned a JSON blob like:
```json
{"AWS_ACCESS_KEY_ID":"ASIA3W6J4C3WCTG6MEL4","AWS_SECRET_ACCESS_KEY":"...","AWS_SESSION_TOKEN":"..."}
```

**after the fix**, keyrack should return just the profile name:

```bash
# unlock keyrack for test env
rhx keyrack unlock --env test

# get the AWS_PROFILE value
rhx keyrack get --key AWS_PROFILE --env test
```

**expected outcome**:
```
ehmpathy.demo
```

(or whatever profile name was set via `keyrack set --key AWS_PROFILE --vault aws.config`)

**pass criterion**: output is a simple string (profile name), not a JSON object

### 2. verify the profile name works with aws cli

```bash
# use the profile name from keyrack to call aws sts
aws sts get-caller-identity --profile "$(rhx keyrack get --key AWS_PROFILE --env test)"
```

**expected outcome**: returns your aws account identity (or asks for sso login if session expired)

**pass criterion**: aws cli accepts the profile name without error about invalid json

### 3. verify adapter returns exid (profile name) when mech is set

this is the core fix - when `mech: EPHEMERAL_VIA_AWS_SSO` is set, the adapter should return the exid (profile name), not resolved credentials

```bash
# the internal behavior (already tested in unit/integration tests)
# adapter.get({ exid: 'ehmpathy.demo', mech: 'EPHEMERAL_VIA_AWS_SSO' })
# should return 'ehmpathy.demo'
```

**pass criterion**: unit tests `[case2][t0] get called with exid` and `[case2][t0.5] get called with exid and mech` pass

## edgey paths

### edge 1: profile not configured in aws config

```bash
rhx keyrack set --key AWS_PROFILE --vault aws.config --exid "nonexistent-profile" --env test
```

**expected behavior**: fails with error `aws profile 'nonexistent-profile' not found in ~/.aws/config`

### edge 2: no exid provided to get

this edge case is covered by unit test `[case1][t1] get called without exid`:

```bash
# verify via unit tests
rhx git.repo.test --what unit --scope vaultAdapterAwsConfig
# look for: [case1] no exid provided > [t1] get called without exid > returns null
```

**expected behavior**: adapter returns `null` when no exid is provided
