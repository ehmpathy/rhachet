## pattern.3 = vault.get() delegation to mech.deliverForGet()

**[REPLACE]**

this is the bug. vaultAdapterAwsConfig.get() delegates to mech.deliverForGet(), which returns JSON credentials.

citation [5]: `src/domain.operations/keyrack/adapters/vaults/aws.config/vaultAdapterAwsConfig.ts:176-187`
```ts
  get: async (input) => {
    const source = input.exid ?? null;
    if (!source) return null;

    // if no mech supplied, return source as-is
    if (!input.mech) return source;

    // transform source → usable secret via mech
    const mechAdapter = getMechAdapter(input.mech);
    const { secret } = await mechAdapter.deliverForGet({ source });
    return secret;
  },
```

citation [6]: `src/domain.operations/keyrack/adapters/mechanisms/aws.sso/mechAdapterAwsSso.ts:159`
```ts
      return { secret: JSON.stringify(credentials), expiresAt };
```

the mech returns JSON, but aws.config vault should return the profile name directly.
