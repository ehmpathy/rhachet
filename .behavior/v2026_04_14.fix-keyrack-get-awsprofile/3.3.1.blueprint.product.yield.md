# blueprint: fix keyrack get AWS_PROFILE

## summary

single-file fix in **vaultAdapterAwsConfig.get()**:
- call mech.deliverForGet() to validate SSO session
- return `input.exid` (the profile name) instead of the mech's secret

the mech is unchanged — it still validates SSO. the vault just ignores its secret and returns the profile name directly.

## rationale

for aws.config vault with SSO:
- **exid**: profile name (e.g., `ehmpathy.demo`)
- **mech work**: validate SSO session (prompts browser login if expired)
- **vault returns**: exid (the profile name)

the mech is called to validate SSO, but its secret is ignored. the vault returns the profile name directly — that's what `AWS_PROFILE` needs. the AWS SDK resolves credentials from the profile.
