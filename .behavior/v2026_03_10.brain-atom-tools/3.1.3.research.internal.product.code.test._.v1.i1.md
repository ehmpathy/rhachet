## new test requirements

### type tests (`.types.test.ts`)

| test | assertion |
|------|-----------|
| no tools → output is TOutput | `const _output: TOutput = result.output` (no null check) |
| no tools → call is null | `const _call: null = result.call` |
| with tools → output is TOutput \| null | must null-check before access |
| with tools → call is { tools } \| null | must null-check before access |
| with tools → invocation has exid, slug, input | type inference on call.tools elements |

### integration tests (`.integration.test.ts`)

| test | scenario |
|------|----------|
| brain requests tool | prompt that requires tool, verify call.tools populated |
| brain answers directly | prompt that can be answered without tools, verify output populated |
| parallel tool calls | prompt that requires multiple tools, verify multiple invocations |
| tool input validation | verify input matches schema |
