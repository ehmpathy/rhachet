### step 2: examine changes to genContextBrain.acceptance.test.ts

i ran `git diff main -- blackbox/sdk/genContextBrain.acceptance.test.ts` and found three categories of changes:

#### category 1: async/await additions

**before**:
```ts
then('brain.choice is that atom', () => {
  const context = genContextBrain({ ... });
```

**after**:
```ts
then('brain.choice is that atom', async () => {
  const context = await genContextBrain({ ... });
```

**reason**: `genContextBrain` signature changed to return `Promise<...>`. this is a mechanical update — the test still verifies the same behavior.

**intention preserved**: yes. the test still verifies that the returned context has the correct brain.choice.

---

#### category 2: non-null assertions added

**before**:
```ts
const choice: BrainRepl = context.brain.choice;
```

**after**:
```ts
const choice: BrainRepl = context.brain.choice!;
```

**reason**: the type of `context.brain.choice` now includes `null` because BrainOutput extended with TPlug generic. however, in these test scenarios:
- the test explicitly provides a `choice` parameter
- the test setup guarantees a match will be found
- the choice will never be null in these cases

the `!` assertion is valid because the test scenario guarantees non-null.

**intention preserved**: yes. the test still verifies that the choice is the expected brain.

---

#### category 3: type guard test refactor

**before**:
```ts
const choice: BrainChoice = context.brain.choice;
if (isBrainAtom(choice)) {
  expect(choice.ask).toBeDefined();
} else {
  fail('expected atom');
}
```

**after**:
```ts
const choice = context.brain.choice!;
expect(isBrainAtom(choice)).toBe(true);
expect((choice as BrainAtom).ask).toBeDefined();
```

**original intentions**:
1. verify `isBrainAtom` returns true for an atom
2. verify the choice has `.ask` method

**both intentions preserved**:
- `expect(isBrainAtom(choice)).toBe(true)` — directly asserts the guard returns true
- `expect((choice as BrainAtom).ask).toBeDefined()` — asserts the property exists

the refactor changed syntax, not intention. the if/else pattern with `fail()` was converted to explicit expects, which is more direct.
