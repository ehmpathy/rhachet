# research: references for BrainAtom tool support

> what research can we reference to ground our thoughts?

---

## primary references

### 1. anthropic tool use documentation

**url**: https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/overview

**key concepts**:

> [1] "Claude is capable of interacting with tools and functions, allowing you to extend Claude's capabilities to perform a wider variety of tasks." — anthropic docs

> [2] "Each tool defines a contract: you specify what operations are available and what they return; Claude decides when and how to call them." — anthropic docs

> [3] "Tool access is one of the highest-leverage primitives you can give an agent." — anthropic docs

**terms established**:
- `tool`: a capability that can be invoked
- `tool_use`: content block type for tool invocations
- `tool_result`: content block type for tool results
- `input_schema`: JSON Schema for tool input
- `stop_reason: tool_use`: indicates tool call intent

---

### 2. openai function call documentation

**url**: https://developers.openai.com/api/docs/guides/function-calling/

**key concepts**:

> [4] "Function call enables models to connect to external tools and systems." — openai docs

> [5] "The model may choose to call multiple functions in a single turn." — openai docs

> [6] "When enabled, strict mode ensures function calls reliably adhere to the function schema." — openai docs

**terms established**:
- `function`: openai's term for tool
- `tool_calls`: array of function invocations in response
- `parameters`: JSON Schema for function input
- `arguments`: JSON-encoded function arguments (string)
- `parallel_tool_calls`: option to enable/disable parallel calls

---

### 3. model context protocol (MCP)

**url**: https://modelcontextprotocol.io

**key concepts**:

> [7] "MCP tool definitions use a schema format that's similar to Claude's tool format. You just need to rename `inputSchema` to `input_schema`." — anthropic docs

**relevance**: MCP provides a standard for tool interoperability. rhachet's ToolDefinition should align for compatibility.

---

## secondary references

### 4. JSON Schema specification

**url**: https://json-schema.org/

**relevance**: both anthropic and openai use JSON Schema for tool input validation. rhachet should align.

### 5. zod schema library

**url**: https://zod.dev/

**relevance**: rhachet callers use zod for TypeScript schema definition. zod-to-json-schema enables conversion.

---

## lessons from references

### lesson 1: tool use is a first-class LLM capability

> [1] tool use is not a workaround — it's a native capability that LLMs are trained on.

### lesson 2: tool invocations require correlation IDs

> [3] both providers assign unique IDs to enable tool result correlation.

### lesson 3: parallel tool calls are supported

> [5] brains can request multiple tool executions in a single turn.

### lesson 4: strict mode is optional and has constraints

> [6] strict mode guarantees schema conformance but limits parallel calls (openai).

### lesson 5: JSON Schema is the industry standard

> [4] both providers use JSON Schema for tool input schemas.

### lesson 6: MCP provides interoperability standard

> [7] MCP aligns with anthropic's tool format; rhachet should maintain compatibility.
