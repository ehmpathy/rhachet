## patterns to [EXTEND]

### 1. conditional type for brain grain

**file**: `src/domain.objects/BrainOutput.ts`

**pattern**: conditional type narrows based on generic parameter

> [7] "export type AsBrainOutputSeriesFor<T extends BrainGrain> = T extends 'repl'
>   ? BrainSeries
>   : T extends 'atom'
>     ? null
>     : BrainSeries | null;" — BrainOutput.ts:12-16

**extend for**: create `AsBrainOutputCallFor<TPlug>` that narrows `call` type based on whether tools are plugged:

```ts
type AsBrainOutputCallFor<TPlug> = TPlug extends { tools: ToolDefinition[] }
  ? { tools: ToolInvocation[] } | null
  : null;
```

### 2. BrainOutput generics

**file**: `src/domain.objects/BrainOutput.ts`

**pattern**: BrainOutput has TOutput and TBrainGrain generics

> [8] "export interface BrainOutput<
>   TOutput,
>   TBrainGrain extends BrainGrain = BrainGrain,
> > {
>   output: TOutput;
>   ...
> }" — BrainOutput.ts:24-31

**extend for**: add TPlug generic to enable conditional output/call types:

```ts
export interface BrainOutput<
  TOutput,
  TBrainGrain extends BrainGrain = BrainGrain,
  TPlug extends BrainAtomPlugs = BrainAtomPlugs,
> {
  output: AsBrainOutputOutputFor<TOutput, TPlug>;  // TOutput | null when tools present
  call: AsBrainOutputCallFor<TPlug>;               // null when no tools
  ...
}
```

## composition summary

### new domain objects

| object | base class | fields |
|--------|------------|--------|
| `ToolDefinition` | DomainLiteral | `slug`, `description`, `schema.input` |
| `ToolInvocation<TInput>` | DomainLiteral | `exid`, `slug`, `input` |

### extended domain objects

| object | extension |
|--------|-----------|
| `BrainAtomPlugs` | add `tools?: ToolDefinition[]` |
| `BrainOutput` | add `TPlug` generic, add `call` field |

### type system

| type | purpose |
|------|---------|
| `AsBrainOutputCallFor<TPlug>` | narrows call based on TPlug |
| `AsBrainOutputOutputFor<TOutput, TPlug>` | narrows output to allow null when tools present |
