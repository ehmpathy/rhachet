# research: internal factory blockers for BrainAtom tool support

> what blockers exist in the current codebase that may impede implementation?

---

## blockers identified

### 1. [BLOCKER] BrainAtomPlugs is a placeholder

**file**: `src/domain.objects/BrainAtomPlugs.ts`

**issue**: BrainAtomPlugs has `output?: never` with a todo comment

> [1] "output?: never; // todo: allow configuration" — BrainAtomPlugs.ts:15

**impact**: the addition of `tools?: ToolDefinition[]` requires verification that the placeholder pattern is intentional

**resolution**: the placeholder pattern is intentional; extend with `tools?: ToolDefinition[]`

---

### 2. [BLOCKER] BrainOutput lacks TPlug generic

**file**: `src/domain.objects/BrainOutput.ts`

**issue**: BrainOutput only has TOutput and TBrainGrain generics

> [2] "export interface BrainOutput<
>   TOutput,
>   TBrainGrain extends BrainGrain = BrainGrain,
> >" — BrainOutput.ts:24-26

**impact**: cannot narrow `output` and `call` types based on whether tools are plugged

**resolution**: add TPlug generic, create conditional types `AsBrainOutputCallFor<TPlug>` and `AsBrainOutputOutputFor<TOutput, TPlug>`

---

### 3. [BLOCKER] BrainAtom.ask does not infer TPlug

**file**: `src/domain.objects/BrainAtom.ts`

**issue**: ask signature has single generic TOutput

> [3] "ask: <TOutput>(
>     input: {
>       ...
>       plugs?: BrainAtomPlugs;
>       ...
>     },
>   ) => Promise<BrainOutput<TOutput, 'atom'>>;" — BrainAtom.ts:50-59

**impact**: callers cannot get type-narrowed call/output based on plugs

**resolution**: add TPlug generic to ask signature, pass to BrainOutput

---

### 4. [BLOCKER] no ToolDefinition domain object exists

**file**: n/a

**issue**: ToolDefinition is specified in vision but does not exist in codebase

**impact**: cannot reference ToolDefinition in BrainAtomPlugs

**resolution**: create `src/domain.objects/ToolDefinition.ts`

---

### 5. [BLOCKER] no ToolInvocation domain object exists

**file**: n/a

**issue**: ToolInvocation is specified in vision but does not exist in codebase

**impact**: cannot reference ToolInvocation in BrainOutput.call.tools

**resolution**: create `src/domain.objects/ToolInvocation.ts`

---

### 6. [BLOCKER] askViaBrainAtom does not pass TPlug

**file**: `src/domain.operations/brainAtom/askViaBrainAtom.ts`

**issue**: wrapper only has TOutput generic

> [4] "export const askViaBrainAtom = async <TOutput>(
>   input: {
>     atom: BrainAtom;
>     plugs?: BrainAtomPlugs;
>     ...
>   },
> ): Promise<BrainOutput<TOutput>> =>" — askViaBrainAtom.ts:15-24

**impact**: callers via askViaBrainAtom lose type inference on call/output

**resolution**: add TPlug generic to askViaBrainAtom, forward to return type

---

## non-blockers

### 1. [NON-BLOCKER] BrainExchange uses string serialization

**file**: `src/domain.objects/BrainExchange.ts`

**issue**: exchanges have `input: string` and `output: string`

**why non-blocker**: vision explicitly chose string serialization for tool flow; tool call becomes output, tool result becomes next input; no structural change needed

---

### 2. [NON-BLOCKER] no test infrastructure for tool use

**issue**: no test assets or mocks for tool definitions

**why non-blocker**: can create test assets as part of implementation; no extant infrastructure to work around
