## matrix 1: output behavior

the core behavioral matrix for atom.ask() with tools.

| ind: tools plugged | ind: brain decision | dep: output | dep: call |
|--------------------|---------------------|-------------|-----------|
| no | answers directly | TOutput | null |
| yes | answers directly | TOutput | null |
| yes | requests tools only | null | { tools: [...] } |

**note**: when tools are not plugged, brain cannot request tools — only direct answers are possible.

## matrix 3: tool invocation edge cases

| ind: tool in plug.tools | ind: input valid | dep: included in call.tools | dep: input value |

| yes | yes | yes | parsed input |
| yes | no | yes | raw input (caller validates) |
| no | n/a | no | n/a |

**note**: brain cannot invoke tools not in plug.tools — the plugged set is the boundary.
