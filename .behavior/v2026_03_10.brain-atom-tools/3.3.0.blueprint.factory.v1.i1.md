## rationale

### test infrastructure

extant patterns are sufficient:
- type tests via `@ts-expect-error` (BrainAtom.types.test.ts)
- unit tests via test-fns given/when/then (BrainAtomPlugs is pure domain object)
- integration tests via rhachet-brains-* packages (actorAsk.integration.test.ts)

no new test utilities needed.

### credentials

tool support is a contract change within rhachet. providers (anthropic, openai) implement the actual tool calls. rhachet only:
- accepts ToolDefinition in plugs
- returns ToolInvocation in call.tools

no new API credentials required.

### feedback loops

| test type | expected duration | blocker? |
|-----------|-------------------|----------|
| type tests | < 1s | no |
| unit tests | < 5s | no |
| integration tests | < 30s | yes (requires brain) |

integration tests can run against openai/codex (cheap, fast).

### blockers

no factory blockers. all identified blockers are product code changes:
- create ToolDefinition domain object
- create ToolInvocation domain object
- extend BrainAtomPlugs with tools
- extend BrainOutput with TPlug generic
- extend BrainAtom.ask with TPlug generic

these are addressed in the product blueprint.
