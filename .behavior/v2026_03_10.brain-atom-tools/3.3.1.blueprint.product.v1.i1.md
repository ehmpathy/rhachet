# blueprint: product for BrainAtom tool support

> how will we implement the wish?

---

## summary

implement BrainAtom tool support via:
1. create BrainPlugToolDefinition domain object
2. create BrainPlugToolInvocation domain object
3. extend BrainPlug with `tools?: BrainPlugToolDefinition[]`
4. extend BrainOutput with TPlug generic and `call` field
5. extend BrainAtom.ask with TPlug generic
6. extend askViaBrainAtom with TPlug generic
7. add type tests for conditional narrower
8. add integration tests for tool flow

## contract details

### BrainPlugToolDefinition

```ts
export interface BrainPlugToolDefinition {
  /** unique identifier for the tool */
  slug: string;

  /** human-readable description of what the tool does */
  description: string;

  /** JSON Schema for tool input validation */
  schema: { input: z.Schema<unknown> };
}
```

maps to:
- anthropic: `name`, `description`, `input_schema`
- openai: `function.name`, `function.description`, `function.parameters`

### BrainPlugToolInvocation

```ts
export interface BrainPlugToolInvocation<TInput = unknown> {
  /** provider-assigned external ID for correlation */
  exid: string;

  /** which tool the brain requests */
  slug: string;

  /** parsed arguments for the tool */
  input: TInput;
}
```

maps to:
- anthropic: `id`, `name`, `input`
- openai: `id`, `function.name`, JSON.parse(`function.arguments`)

### BrainOutput.call

```ts
call: TPlug extends { tools: BrainPlugToolDefinition[] }
  ? { tools: BrainPlugToolInvocation[] } | null
  : null;
```

- when tools are plugged: `{ tools: [...] }` if brain requests tools, `null` if brain answers directly
- when no tools: always `null`

### BrainOutput.output

```ts
output: TPlug extends { tools: BrainPlugToolDefinition[] }
  ? TOutput | null
  : TOutput;
```

- when tools are plugged: `TOutput` if brain answers, `null` if brain defers to tools
- when no tools: always `TOutput`

## test coverage

### type tests (BrainAtom.tools.types.test.ts)

| test | assertion |
|------|-----------|
| no tools → output is TOutput | `const _output: TOutput = result.output` compiles |
| no tools → call is null | `const _call: null = result.call` compiles |
| with tools → output is TOutput \| null | `// @ts-expect-error` if no null check |
| with tools → call is { tools } \| null | `// @ts-expect-error` if no null check |
| invocation has exid, slug, input | type inference on call.tools elements |

### integration tests

integration tests require brain supplier support — deferred to supplier packages (rhachet-brains-anthropic, rhachet-brains-openai).

rhachet core validates:
- type inference works at compile time
- BrainOutput structure is correct
- conditional types narrow correctly
